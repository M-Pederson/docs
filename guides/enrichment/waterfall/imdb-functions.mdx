## Entry gates

The functions that can start IMDB work — the person \+ title get-add gates (#12873 / #12874), the master-person attach path (#10509), reverse IMDB-URL verification (#12677), and the cross-waterfall expertise bridge (#13185) — are documented on [IMDB Waterfall Entry Points](/guides/enrichment/waterfall/imdb-entry-points). #10509 creates/links the `imdb_person` stub before scraping, queues a retry row on optional no-markdown Firecrawl misses, stamps success metadata on direct attach, clears stale retry rows after success, materializes IMDb `official_sites` as `master_link` rows (v2.5), and delegates credit fanout to `cascade-credits-from-person(sync_top_credits: 5)` so the top seed credits enrich inline while the rest queue with `seed_imdb_person_id` provenance.

## Graph writers (FalkorDB nodes \+ edges)

<Card icon="cloud-arrow-up">
  ```text
  mvp/edge/add-node-edge-imdb-award — #2334
  ```

  **v1.12 (2026-07-09):** Writes `Film_TV` \+ `Film_TV_Award` nodes, `HAS_AWARD` edge, and (when a Person UUID resolves) `WORKED_ON` \+ `WON_AWARD` / `NOMINATED_FOR` edges to FalkorDB for each `film_tv_award` row. Two-query split so a missing Person stub never kills the whole write. Three-tier bulletproof chain on Query A: MERGE → retry → MATCH-only fallback → `NULL_UUID_FINAL` hard-fail log (v1.6). Honorary handling (v1.5): `film_tv_id == null` rows persist a Person→Award edge without a `Film_TV` anchor (`Film_TV_Award` MERGE key swaps `film_tv_imdb_url` → `person_uuid`). Stamps `film_tv_award.node_uuid` on success; both node types stamp `created_at` / `updated_at`. **2026-07-08/09 series:** title-level award rows with no `imdb_person_id` are supported — `Film_TV_Award` nodes \+ `HAS_AWARD` edges materialize without a person (v1.7); `Film_TV_Award` nodes stamp display metadata (`name`, `title`, `display_label`, `source`, `updated_at`) on create and match (v1.8); Query B edges stamp `uuid` / `updated_at`, and award-linked `Film_TV` nodes get a deterministic description fallback (v1.9); Query A returns the `Film_TV` graph uuid and writes it back to `film_tv.node_uuid` so award-created title nodes stay relationally linked (v1.10); per-award **LLM description generation is removed from this hot path** — deterministic descriptions keep graph repair bounded on high-award people (v1.11); `Film_TV_Award` and award-created `Film_TV` node UUIDs are registered in `node_registry` (v1.12). **Invocation:** the award scrapers defer graph writes, so this now runs via `tool/backfill-award-edges-imdb` #13014 batches. Edge weights per the [§3.6 weight audit](/guides/ontology/edge-weights#weight-audit--applied-adjustments) — current values on [IMDB Edges](/guides/enrichment/waterfall/imdb-edges#summary--imdb-edges-at-a-glance).
