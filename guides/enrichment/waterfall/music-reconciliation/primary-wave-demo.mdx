## Goal

Build one small demo that does two jobs:

1. Show Orbiter to Primary Wave with a graph story that is relevant to their business.
2. Seed a compact MusicBrainz data set into Sandbox and validate that the music graph creates the expected labels, artists, releases, recordings, works, and ownership edges.

The best target is **Sun Records / Sun Record Company**.

Primary Wave is mostly a music catalog and rights company, so not every Primary Wave deal is a record-label ownership case. Sun is different: Music Business Worldwide reported on January 28, 2021 that Primary Wave acquired Sun Records, including master recordings, publishing, logo, and trademarks. That makes Sun the cleanest owned-label demo target.

## Research Summary

| The deal included masters for artists including Johnny Cash, Jerry Lee Lewis, Carl Perkins, Roy Orbison, and more. | Use those artists and releases as the MusicBrainz seed set. |
| Sun's own history page centers Elvis Presley, Johnny Cash, Jerry Lee Lewis, Roy Orbison, Sam Phillips, and Carl Perkins. | The demo story is recognizable without needing a long explanation. |

## Demo Thesis

The story to show Primary Wave:

```text
Primary Wave
  owns Sun Record Company
    released early landmark recordings
      by Johnny Cash, Jerry Lee Lewis, Carl Perkins, Junior Parker, and others
        which connect to works, producers, backing musicians, and later catalog opportunities
```

The demo should open with ownership, then walk into catalog intelligence:

- What label assets does Primary Wave own?
- Which artists and recordings are attached to that label?
- Which works and recordings are highly connected?
- Which producers/session players appear across the catalog?
- Which data gaps still need reconciliation?

## Function to Build

Create one operator-only smoke function:

```text
mb/smoke/primary-wave-sun-demo
```

Inputs:

| Input | Type | Default | Notes |
| --- | --- | --- | --- |
| `data_source` | text | `sandbox` | Must not default to live. |
| `mode` | text | `seed_and_check` | `seed_only`, `check_only`, `seed_and_check`. |
| `reset_seed_rows` | bool | `false` | Optional cleanup for only this seed set. |
| `run_owner_resolution` | bool | `true` | Needed for the Primary Wave ownership test. |
| `max_wait_seconds` | int | `900` | Lets queued phases finish before checks. |

Implementation:

1. Assert `data_source == "sandbox"` unless an explicit admin override is passed.
2. Upsert seed rows into the normal `queue_mb_*` tables using the existing queue helper.
3. Prioritize Sun label and Primary Wave owner resolution first.
4. Trigger or wait for the existing queue drain.
5. Run the read-only graph checks below.
6. Return a compact result object: counts, pass/fail checks, missing MBIDs, and links to failed nodes.

Do not create a separate graph model for the smoke test. The value is that the same function exercises the real waterfall contracts documented in [MusicBrainz Nodes](/guides/ontology/nodes#musicbrainz-nodes) and [MusicBrainz Edges](/guides/ontology/edges#musicbrainz).

## Expected Graph Shape

Minimum expected after a successful run:

| Area | Expected |
| --- | --- |
| Label | One `Company:Music_Label` node for `Sun Record Company`. |
| Owner | One `Company` node for `Primary Wave`, plus `Sun Record Company -[:OWNED_BY]-> Primary Wave`. |
| Artists and groups | `Person:Music_Artist` nodes for Cash, Lewis, Perkins, Orbison, Presley, Phillips, and Clement, plus `Music_Group` for Little Junior's Blue Flames. |
| Releases | Seven `Release` nodes and seven `Release_Group` nodes. |
| Recordings | At least eight `Recording` nodes. |
| Works | At least six `Work` nodes; `Folsom Prison Blues` can be verified opportunistically. |
| Label edges | `RELEASED_ON` from each canonical release to Sun. |
| Derived label edges | `ON_RECORD_LABEL` from each seeded recording to Sun where release scope exists. |
| Work edges | `RECORDING_OF` for the known recording/work pairs. |
| Credit edges | `PERFORMED`, `PLAYED_ON`, `PRODUCED`, and/or `ENGINEERED` where MusicBrainz relation data is present. |
| Descriptions | Newly-created music edges start with semantic null description fields and are picked up by the edge describer. |

## Smoke Checks

Run these against Sandbox after the seed. Keep them read-only.

### 1. Label and owner

```cypher
MATCH (sun:Company:Music_Label {mbid: $sun_label_mbid})
OPTIONAL MATCH (sun)-[owned:OWNED_BY]->(owner:Company)
RETURN sun.name AS label,
       owner.name AS owner,
       owned.weight AS weight,
       owned.match_method AS match_method,
       owned.confidence AS confidence
```

Pass condition:

- `label = "Sun Record Company"`
- `owner` resolves to Primary Wave or a clearly equivalent Primary Wave company node
- `weight = 15`

### 2. Release chain

```cypher
MATCH (sun:Company:Music_Label {mbid: $sun_label_mbid})
MATCH (sun)<-[:RELEASED_ON]-(rel:Release)-[:RELEASE_OF]->(rg:Release_Group)<-[:RELEASED]-(artist)
RETURN artist.name AS artist,
       rg.title AS release_group,
       rel.title AS release,
       rel.mbid AS release_mbid
ORDER BY artist, release
```

Pass condition: at least seven release rows, including Johnny Cash, Jerry Lee Lewis, and Carl Perkins.

### 3. Derived recording-to-label shortcut

```cypher
MATCH (sun:Company:Music_Label {mbid: $sun_label_mbid})
MATCH (rec:Recording)-[edge:ON_RECORD_LABEL]->(sun)
RETURN rec.title AS recording,
       rec.mbid AS recording_mbid,
       edge.via_release AS via_release,
       edge.weight AS weight
ORDER BY recording
```

Pass condition: seeded recordings have `ON_RECORD_LABEL` with `weight = 55` where their canonical release exists.

### 4. Recording-to-work coverage

```cypher
MATCH (rec:Recording)-[:RECORDING_OF]->(work:Work)
WHERE rec.mbid IN $recording_mbids
RETURN rec.title AS recording,
       rec.mbid AS recording_mbid,
       work.title AS work,
       work.mbid AS work_mbid
ORDER BY recording
```

Pass condition: known pairs for `I Walk the Line`, `Blue Suede Shoes`, `Great Balls of Fire`, `Whole Lotta Shakin' Goin' On`, `Mystery Train`, `Matchbox`, and `Hey Porter!`.

### 5. Producer and session-player edges

```cypher
MATCH (person:Person:Music_Artist)-[edge]->(rec:Recording)
WHERE rec.mbid IN $recording_mbids
  AND type(edge) IN ["PLAYED_ON", "PRODUCED", "ENGINEERED", "PERFORMED"]
RETURN person.name AS person,
       type(edge) AS edge_type,
       rec.title AS recording,
       edge.weight AS weight
ORDER BY recording, edge_type, person
```

Pass condition: Jerry Lee Lewis recordings show performer/player edges, and Sam Phillips appears as producer where MusicBrainz relation data provides him.

### 6. Smoke-test count summary

```cypher
MATCH (n)
WHERE n.mbid IN $all_seed_mbids
RETURN labels(n) AS labels,
       count(n) AS count
ORDER BY count DESC
```

Pass condition: no missing seed MBIDs, except any intentionally opportunistic work IDs.

## Demo Query Bank

These are the Primary Wave-facing queries to stage after the smoke test passes.

### 1. What label does Primary Wave own?

```cypher
MATCH (label:Company:Music_Label)-[owned:OWNED_BY]->(owner:Company)
WHERE toLower(owner.name) CONTAINS "primary wave"
RETURN label.name AS owned_label,
       owner.name AS owner,
       owned.match_method AS match_method,
       owned.confidence AS confidence
ORDER BY owned_label
```

### 2. What catalog sits under Sun?

```cypher
MATCH (sun:Company:Music_Label {mbid: $sun_label_mbid})
MATCH (artist)-[:RELEASED]->(:Release_Group)<-[:RELEASE_OF]-(:Release)-[:RELEASED_ON]->(sun)
RETURN DISTINCT artist.name AS artist
ORDER BY artist
```

### 3. Which recordings connect the label to works?

```cypher
MATCH (sun:Company:Music_Label {mbid: $sun_label_mbid})
MATCH (rec:Recording)-[:ON_RECORD_LABEL]->(sun)
OPTIONAL MATCH (rec)-[:RECORDING_OF]->(work:Work)
RETURN rec.title AS recording,
       work.title AS work
ORDER BY recording
```

### 4. Who are the producer/session connectors?

```cypher
MATCH (sun:Company:Music_Label {mbid: $sun_label_mbid})
MATCH (rec:Recording)-[:ON_RECORD_LABEL]->(sun)
MATCH (person:Person:Music_Artist)-[edge]->(rec)
WHERE type(edge) IN ["PLAYED_ON", "PRODUCED", "ENGINEERED", "PERFORMED"]
RETURN person.name AS person,
       type(edge) AS role_edge,
       count(DISTINCT rec) AS recordings
ORDER BY recordings DESC, person
```

### 5. Full Sun graph visual

Paste this into FalkorDB Browser to render the demo graph around Sun:

```cypher
MATCH p = (:Company:Music_Label {mbid: $sun_label_mbid})<-[:RELEASED_ON]-(:Release)-[:RELEASE_OF]->(:Release_Group)<-[:RELEASED]-(artist)
OPTIONAL MATCH q = (artist)-[:PERFORMED|PLAYED_ON|PRODUCED|ENGINEERED]->(rec:Recording)-[:ON_RECORD_LABEL]->(:Company:Music_Label {mbid: $sun_label_mbid})
OPTIONAL MATCH r = (rec)-[:RECORDING_OF]->(:Work)
RETURN p, q, r
LIMIT 300
```

## Run Checklist

1. Confirm Sandbox has the latest music waterfall functions.
2. Run `mb/smoke/primary-wave-sun-demo` with `data_source = "sandbox"` and `mode = "seed_and_check"`.
3. Wait for the queue drain or let the function poll up to `max_wait_seconds`.
4. Review the returned missing-MBID list first.
5. Run the six smoke checks in FalkorDB Browser.
6. If the owner edge is missing, inspect `mb_label.owner_match_method`, `owner_confidence`, and the label owner resolver logs.
7. Once checks pass, stage demo queries 1, 2, 3, and 5.

## What This Catches

| Contract | Why this seed catches it |
| --- | --- |
| Label ingest | Sun is a real `Company:Music_Label` with many releases. |
| Owner resolution | Sun should resolve to Primary Wave through the Exa/LLM owner tier, not MusicBrainz label relations. |
| Release materialization | Seven releases give enough coverage for release-group and canonical release writes. |
| Derived `ON_RECORD_LABEL` | Recordings from canonical releases should get direct label shortcuts. |
| Recording/work linkage | The selected tracks have known work relationships in MusicBrainz. |
| Credit edges | Jerry Lee Lewis, Carl Perkins, Sam Phillips, and Jack Clement exercise performer/player/producer relationships. |
| Demo usefulness | The result is not a fake QA fixture; it is a Primary Wave-relevant catalog graph. |

## Cut for v1

| Cut | Reason |
| --- | --- |
| Full Sun catalog | Too large for a deterministic smoke test. |
| Full Primary Wave catalog | Primary Wave owns many rights that are not cleanly modeled as record labels in MusicBrainz. |
| Live data source | This is a Sandbox smoke test first. |
| Destructive cleanup | Only reset rows explicitly tied to the seed MBIDs. |
| UI build | FalkorDB Browser is enough for the first demo. |
