The [Person Enrichment Waterfall](/guides/enrichment/waterfall/person-waterfall) uses the canonical graph edge definitions in [Ontology → Edge Types](/guides/ontology/edges). Write-side Cypher lives on [Cyphers → Person Pipeline](/guides/cyphers/person-pipeline) (funding on [Cyphers → Funding](/guides/cyphers/funding)); canonical weights are on [Edge Weights](/guides/ontology/edge-weights) (**lower = stronger**; hub-node edges sit Weak or below; never sub-1 — the same shared scale as the [company](/guides/enrichment/waterfall/company-edges), [IMDB](/guides/enrichment/waterfall/imdb-edges), and [music](/guides/enrichment/waterfall/music-edges) edges).

This page is the **pipeline-scoped catalog**: which edges this pipeline emits, at what weight, from which writer — including per-edge **live status**. Edge patterns, properties, and examples should be edited only in the ontology source of truth. Writer internals live on [Person Waterfall Functions](/guides/enrichment/waterfall/person-functions); node MERGE keys on [Person Nodes](/guides/enrichment/waterfall/person-nodes). Provenance map: work, education, certification, honor, work-product, and volunteering edges are written in **phase 7** (#12828 v1.3 — for `current_company_only` people only education runs; the other resolvers are skipped); expertise is **phase 8**, the music `SAME_AS` bridge **phase 8b**, the direct angel portfolio edges **phase 9** (double-invoked by orchestrator #13040 v3.33), the location chain **phase 2**. Under the **controlled-waterfall policy (2026-07-04/05)** a person run full-enriches its selected current company \+ schools, queues past employers as depth-1 leaf companies, and links everything else lookup-/local-only.

## Work edges

Written by the shared `mvp/edges/create-work-edges` (#2800, **v2.12 (2026-07-08)**) — one Cypher MERGEs Person \+ Company stubs and writes the edge types in a pass via per-edge `FOREACH (CASE WHEN …)` guards, with stable `r.uuid = coalesce(r.uuid, randomUUID())`, `created_at` coalesce \+ `updated_at`, deterministic fallback `description`s, and `valid_from` / `valid_to` on every edge. On this pipeline it is invoked from `resolve-edges-work` (#12562 v1.13) and phase 3 (#12824 v3.6, for the selected current role). **Since v2.12 edge descriptions always run inline** — the `write_descriptions` input is retained but ignored (see [Edge descriptions](#edge-descriptions)).

- `(Person)-[:WORKS_AT]->(Company)` · weight = `seniority_rank` **\+20 when past** (no current role) — 14 buckets across 5–40, current C-suite strongest (5), past individual contributor weakest (40, the largest live bucket). Props `title`, `seniority_level`, `total_roles`, `role_history`. Full mapping: [Edge Weights → WORKS\_AT buckets](/guides/ontology/edge-weights#works_at-weight-buckets).
- Title-classified alternatives: `(Person)-[:FOUNDED]->(Company)` · weight **5** (templated founded-vs-co-founded description; since v1.9/v2.x also OR-ed in from Fundable structured `is_founder` and Company\_LLM founder evidence, with the **phantom-founder guard** dropping title-only claims that authoritative Fundable founder lists exclude) · `BOARD_MEMBER_OF` · **8** · `ADVISOR_TO` · **12** · `MEMBER_OF` `(Person)→(Company / Organization)` · **20**.
- **`INVESTED_IN` (work-derived, w15) is a dead branch since v2.5 (2026-07-05):** the classifier hardcodes `isInvestor: false`, so work-history titles never emit it — "Angel Investor at Leeds Angels" is a work-history relationship now. Investment edges come from the [deal writers](#funding-edges) (phase 9 direct edges w25; round edges via #12702). Existing w15 work-derived edges linger in the graph.

### Title routing & guards (§3.3 / §3.7 \+ the v2.x wave)

The title classifier inside #2800 routes before it writes: **advisory / board / membership titles are excluded from `WORKS_AT`** (general/managing partners stay employment-eligible via `isOperatingPartner()`); **unions → `MEMBER_OF`**, suppressing `WORKS_AT` (name allowlist — SAG-AFTRA, the writers/directors guilds, IATSE, AFM, Actors' Equity, …); **non-executive directors → `BOARD_MEMBER_OF` only**; "Founder in Residence" and "Advisor to Founder/CEO" variants never classify as founders (v2.3/v2.6); the **`Unknown Company` stub guard** skips unresolved stubs (`name == "Unknown Company"`, empty `company_domain`); and **generic placeholder companies** ("Angel Investor", "Board Member", "Stealth Startup" — v2.4/v2.7, when domainless) receive no edges at all. Audit trail \+ backfill runbook: [edge-audit §3.3 / §3.7](/guides/robert-mark/edge-audit).

### The `HAS_WORKED_AT` ghost

**There is no `HAS_WORKED_AT` edge type** — no writer for it exists anywhere in workspace 3, and #2800 has never emitted one (the old C-suite-mode doc named it; past roles actually live on `WORKS_AT` at the \+20 buckets). Legacy `WORKED_AT` edges from the pre-#2800 writer linger in the graph **read-only** — swept by the #13006/#13008 description tooling, otherwise inert.

## Education edges

Written by `mvp/edges/create-education-edges` (#1898, hardened 2026-07-04/07 — JSON-encoded dynamic strings, guaranteed `uuid`/`description`/`source: "education_experience"` on merged edges, same-school duplicate-`ATTENDED` cleanup, and idempotent re-runs over **all** resolved education rows so bad descriptions self-repair), called from `resolve-edges-education` (#12560 v1.13 — schools full-enrich inline; strong school-evidence gates).

- `(Person)-[:ATTENDED]->(Company school)` · weight **40** · props `degree`, `field_of_study`, `activities_societies`, `valid_from` / `valid_to` — `FOREACH`-gated so an empty string never overwrites a prior value; `valid_to` is year-guarded to drop the future end-dates LinkedIn sometimes emits; a default "X attended Y" description is always present.
- `(Person)-[:ATTENDED_TOGETHER]-(Person)` (undirected) · weight **35 / 70** — same `field_of_study` (`relationship_strength: 3`) / time-overlap only (`relationship_strength: 1`). _The old three-tier 35/50/70 major-vs-field split is gone — the live Cypher compares `field_of_study` only._
- `(Student:Person)-[:STUDIED_UNDER]->(Instructor:Person)` · weight **10** — **dormant**: it derives from a `TAUGHT_AT` edge on the same school, and no creator writes `TAUGHT_AT` yet.

## Certification edges

Written by `resolve-edges-certifications` (#12578, **v2.6 (2026-07-07)**). v2.5 scopes every query to the input `master_person_id`; cert descriptions generate at resolve time via `openai/gpt-oss-120b` with OpenRouter web search for issuer domains and are normalized before graph writes (v2.6); issuers are always `cascade_depth=1` leaves. All three edges carry `uuid` / `created_at` / `updated_at` / `description`.

- `(Person)-[:HAS_CERTIFICATION]->(Certification)` · weight **15** for FINRA `Series …` certs, **20** otherwise · props `valid_from` (issue date), `description`.
- `(Certification)-[:GRANTED_BY]->(Organization:Company)` · weight **10** — the writer adds the `:Organization` label to the issuer's Company node.
- `(Person)-[:CERTIFIED_BY]->(Organization:Company)` · weight **25** — the direct one-hop shortcut; `cert_type` is part of the MERGE key (one edge per cert type), `certification_count` bumped on re-merge.

## Honor edges

Written by `resolve-edges-honor` (#4574, **v4 (2026-06-23)** — issuer-resolution queries scoped to the input `master_person_id`; issuers are depth-1 leaves (v3), no company enqueue since the 2026-06-23 sweep). CREATE-only: each honor row mints a fresh `Honor` node — uniqueness is caller-side, not a MERGE key (see [Person Nodes](/guides/enrichment/waterfall/person-nodes)).

- `(Person)-[:RECEIVED_HONOR]->(Honor)` · weight **20**.
- `(Honor)-[:ISSUED_BY]->(Organization)` · weight **25** — fires only when the issuer node resolves (`OPTIONAL MATCH` \+ `FOREACH` guard).

## Work-product edges

Written by `resolve-edges-projects-publications` (#12579, **v8 (2026-07-07)**). Project nodes are `Entity:Work:Project` since v6, which also made the resolver process existing project rows (repair instead of self-skip) and stamp descriptions \+ display labels on nodes and `AUTHORED` edges; publication re-run protection still lives in the relational rows. Since v4 contributor resolution is **seed-person-only**; v5 makes publisher companies **leaf/local-only** (`queue=false` \+ `hold_downstream=true`).

- `(Person)-[:CONTRIBUTED_TO]->(Work:Project)` · weight **recency-graded 40 / 35 / 30 / 25 / 20 / 15** — 40 for current/ongoing projects, stepping down per year since `end_year`, 15 at 5\+ years · props `is_current`, timestamps, description.
- `(Person)-[:AUTHORED]->(Work:Publication)` · weight **recency-graded 25 / 22 / 18 / 15** by publication year.
- `(Work:Publication)-[:PUBLISHED_BY]->(Company)` · weight **20** — fires only when a publisher company node resolved.

## Volunteering edges

Written by `resolve-edges-volunteering` (#4575, **v4 (2026-06-04)** — all queries scoped to the input person; orgs resolve as depth-1 leaves — no company enqueue since the 2026-06-23 sweep). MERGEs the `Organization:Company` node by name, then CREATEs a per-row edge.

- `(Person)-[:VOLUNTEERED_AT]->(Organization:Company)` · weight **38** current / **50** past (`endYear`-keyed, `isCurrent` prop) · props `title`, start/end month \+ year.

## Location edges

Written by `primary-location-nodes-edges_v2` (#4690) from phase 2 via `add-person-primary-location` (#2766) — the company side uses the sibling `location-nodes-edges-company` (#4528). One of four Cypher branches fires depending on which of city/region/country resolved; all endpoints MERGE.

- `(Person)-[:LOCATED_IN]->(City / Region / State)` · weight **70**; direct `(Person)-[:LOCATED_IN]->(Country)` (no city/region) · weight **80** · stamps `valid_from` (`Y-m` of write time). _(§3.6 — was 10/50 pre-audit; at 10 everyone sharing a city hub was near-maximally linked.)_
- `(City)-[:IN_REGION]->(Region / State)` · weight **25**.
- `(Region / City)-[:IN_COUNTRY]->(Country)` · weight **50** — structural geography backbone, unchanged by the audit.

## Expertise edges

### HAS\_EXPERTISE

`(Person)-[:HAS_EXPERTISE]->(SubDomainExpertise / DomainExpertise)` — written by `resolve-person-expertise-v2` (#12926, **v2.8 (2026-07-08)**; locked spec promoted 2026-06-18; chat on `qwen/qwen3.7-plus`, reasoning disabled), fed by `llm-identify-person-expertise` (#12666, `qwen/qwen3.7-plus`, reasoning disabled, `max_tokens` 2500). #12666 **DELETEs all existing `HAS_EXPERTISE` edges for the person first** (and, since 2026-07-06, their `person_has_expertise` mirror rows) — every phase-8 run is a full rewrite, not incremental. #12926 runs a Gemini `expert_embeddings` KNN with a **3-way guard band** on vector distance: **\< 0.15** auto-match · **0.15–0.25** Qwen LLM adjudicate · **\> 0.25** create a new `SubDomainExpertise` node. Since **v2.8** every MERGE stamps the full metadata envelope — `uuid`, `created_at`, `updated_at`, `source: "llm_identify_person_expertise"` — alongside the match/evidence fields.

Two weight rules — the score is a vector **distance** (lower = closer):

- **Matched** to an existing node: `weight = min(round(10 + match_score × 160), 50)` — 10 at distance 0, capped at 50 near the create threshold.
- **Created** node: fixed `weight = 10` — a freshly minted node perfectly represents the identified expertise. _(Older docs and #12925's Xano input note still say `× 80` — a stale comment; the live resolver writes `× 160`. Do not propagate it.)_

`expertise_strength` is a **separate 0–100 property**, decoupled from the path weight — computed from depth \+ recency \+ duration signals (the v5 prompt emits per-signal `start_year` / `end_year`). On re-merge, `evidence` / `identified_as` / `signal_type` pipe-concatenate, the `depth` and `recency` ladders only escalate, and `match_score` / `weight` only improve (decrease). Every successful MERGE dual-writes `person_has_expertise` (table 711) via #12925 (v1.2, 2026-07-02 — `|concat` fix \+ `CONTAINS` dedupe guards); the deep-research webhook (#8303 v1.1, 2026-07-02) re-runs the whole chain on success, relying on this idempotency.

### HAS\_SUBDOMAIN

`(Entity:DomainExpertise)-[:HAS_SUBDOMAIN]->(Entity:SubDomainExpertise)` · weight **100** — a **deliberate hub-damping value, not closeness**. Written by `attach-subdomain-parent` (#13142) from #12926's create branch after each verified node CREATE, parent picked by the `resolve-parent-domain` #13141 LLM resolver (v1.4, 2026-07-02 — never auto-creates a domain, throws after 3 failed attempts); on a NONE verdict #13142 instead flags the node `parent_unresolved`. The same forward-fix serves the company pipeline's `SPECIALIZES_IN` create branch — detail on [Company Edges](/guides/enrichment/waterfall/company-edges#has_subdomain) \+ [Cyphers → Expertise](/guides/cyphers/expertise).

## Mentorship edges

`(Person YC partner)-[:MENTOR]->(Person founder)` · weight **14** · `source: 'inferred'`, `domain: 'accelerator'` — person-to-person, but written by the **company** pipeline's seed-only YC path (`process-yc-people` #12700), not by any person-run phase; listed because both endpoints are this pipeline's `Person` nodes. **As-built (flagged):** the MERGE key includes `company_uuid` **and** `valid_from` (derived from `master_company.founded ?? 1970`), so a changed founding year mints a duplicate edge — detail on [Company Edges → YC founding & mentorship](/guides/enrichment/waterfall/company-edges#yc-founding--mentorship-edges).

## Funding edges

<Note>
  **The 2026-07-02 D-series split.** The person side and company side now write different funding edges. **Person phase 9** (#12830) writes **direct angel portfolio edges only**: per `fundable_angel_investments` row it MERGEs `(Person:Angel)-[:INVESTED_IN w25]->(Company)` against the canonical `master_company_node_uuid` (ON CREATE stamps a self-healing stub; skipped only when the org has no company node uuid) with an LLM-or-deterministic description — no `Funding_Round` materialization, no co-investor fanout, and the gate **defaults ON for depth-0 angels** (`allow_investor_network_cascade|first_notnull:true && cascade_depth == 0`; pass `false` to suppress). The **round-scoped edge set below** is written by the company-side cascade (#12703 → #12856 v1.10 → #12702 v1.17), where the orchestrator invokes phase 9 twice (early `build_investment_thesis=false`, late `=true`) and MERGEs keep double passes idempotent. Deals processed in the 2026-06-23 → 07-02 depth-coercion outage carry `Funding_Round` \+ `RAISED` only (backfill declined, accepted as historical).
</Note>

On the company side, #12856 cascades each deal — portfolio-company get-add (local-only, `hold_downstream`), IPO signal, `Funding_Round` node (`:Entity`-labeled since #12701 v1.10), then #12702 for the investor side:

- `(Company)-[:RAISED]->(Funding_Round)` · weight **10** — written by #12701 after the node MERGE; edge `created_at` coalesces to the deal's announced date; deterministic `uuid`.
- `(Company:VC_Firm)-[:LEAD_INVESTED_IN]->(Funding_Round)` · weight **15** · `is_lead_investor: true` — lead detection is the Fundable `lead_investor` column, no inference; the `:VC_Firm` label is added on the same pass.
- `(Company:VC_Firm)-[:INVESTED_IN]->(Funding_Round)` · weight **20** — non-lead; or `FOLLOW_ON_INVESTED_IN` · weight **20** when an `OPTIONAL MATCH` finds an earlier round the same firm backed (strict `<` date compare, so re-runs don't self-count).
- `(Person:Angel)-[:LEAD_INVESTED_IN]->(Funding_Round)` · weight **15**, or `(Person:Angel)-[:INVESTED_IN]->(Funding_Round)` · weight **25** — angels get the `:Angel` label, no follow-on detection, and a deliberately weaker non-lead weight than institutional.
- `(Person)-[:INVESTMENT_PARTNER_IN]->(Funding_Round)` · weight **15** \+ `(Person)-[:INVESTMENT_PARTNER_AT]->(Company:VC_Firm)` · weight **20** — one Cypher pair per resolved deal partner (no `Angel` label since v1.10).

Co-investor people and companies are created as **queued depth-1 leaves** (people at tier 1; investor firms also at tier 1 since #12702 v1.14, 2026-07-03), not full-enriched inline. Every #12702 edge stamps `uuid` / `created_at` / `updated_at` \+ a description (LLM helper with deterministic fallback, JSON-encoded in Twig since v1.16). Deep per-edge treatment: [Company Edges → Investor edge set](/guides/enrichment/waterfall/company-edges#investor-edge-set-12702) — linked, not duplicated.

## Film & TV edges (phase 8 handoff)

The person pipeline writes none of these directly. ## Edge descriptions

**Work \+ relationship edges** use the shared trios: `create-work-edge-description-from-graph` #13003 → `write-work-edge-description-to-graph` #13008 for `WORKS_AT` (and legacy `WORKED_AT`), and #13009 → #13010 for `BOARD_MEMBER_OF` / `ADVISOR_TO` / `INVESTED_IN` / `MEMBER_OF` — both create functions on `deepseek/deepseek-v4-flash` (reasoning off; the 2026-06-17 v2 swap fixed the gpt-oss-120b dropped-space/U\+2011 artifacts), plus the shared relationship writer for `FOUNDED` since #2800 v2.9. **Since #2800 v2.12 (2026-07-08) these writers always run inline at edge creation** — the `write_descriptions` input is retained for old callers but ignored, and every edge also gets a deterministic fallback description in the Cypher itself, so new person-origin work edges are never born description-less. The async drainers — #13006 (Tasks #171/#167/#168, keyed `description IS NULL`) and #13011 (Tasks #166/#169/#170, keyed `description_generated_at IS NULL`), all six inactive — remain as catch-up tooling for the legacy window (edges written while the 2026-06-23 → 07-08 `write_descriptions: false` era was in effect). Contrast: these drainers key on per-edge description markers, the [IMDB trio](/guides/enrichment/waterfall/imdb-edges#edge-descriptions) keys on `(person_uuid, film_uuid)` endpoint pairs, and the [music trio](/guides/enrichment/waterfall/music-edges#edge-descriptions) keys on `e.uuid`.

**`HAS_EXPERTISE`** gets a 15–30-word LLM sentence via `write-expertise-edge-description-to-graph` #13020 (**v1.1 (2026-06-25)** safe-rows fix), called **inline from both branches** of #12926 with `description_generated_at` as the skip marker — phase 8's delete-then-rewrite wipes the markers, so re-enriched people get fresh descriptions in-band. Catch-up backfill: **Task #177** `backfill-expertise-edge-descriptions-v2` → #13046 (20 edges/batch, 120 s tick, ships `active = false`, deactivate at `no_pending_edges`). **Task #176 was DELETED** (buggy) — any doc still citing it is stale. The create side #13019 is on the same `deepseek/deepseek-v4-flash` pattern (v1.2, 2026-06-18, reasoning off), plus a v1.3 out-of-credits guard that throws loudly on OpenRouter 402s — via `mvp/llm/check-openrouter` #13136 — instead of silently writing null.

**Certifications and education** generate their description text at resolve time — #12578 v2.5 via `openai/gpt-oss-120b` \+ OpenRouter web search; #1898's `ATTENDED` description via `openai/gpt-oss-120b` (since 2026-05-30). **Everything else** — `FOUNDED`, honors, work products, volunteering, location, funding, `MENTOR` — carries **inline templated description strings** written in the edge Cypher itself, not LLM output.

<Warning>
  **Descriptions are one-shot.** The backfills generate only where the marker is absent and never regenerate when edge data later changes — describe only **after** data cleanups, or stale facts are baked into prose permanently (observed live on a union `WORKS_AT` later converted to `MEMBER_OF`). Coverage matrix: [edge-audit → description coverage](/guides/robert-mark/edge-audit); model migration: [Refactor Edge Descriptions](/guides/robert-mark/refactor-edge-descriptions).
</Warning>
