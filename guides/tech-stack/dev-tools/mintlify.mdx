## What it is

[Mintlify](https://mintlify.com) is a documentation platform that turns MDX files in a git repo into a polished docs site. It supports API reference generation, component-based layouts, search, and versioning — all configured via a single `docs.json` file.

## Why we use it

- **Git-synced** — docs live in the repo alongside code, deployed on push to `main`
- **MDX** — Markdown with React components (Accordions, Cards, Tabs, Steps, etc.)
- **API reference** — auto-generated from OpenAPI specs or hand-authored with `ParamField` components
- **Search** — built-in full-text search across all pages
- **Custom domains** — deployed at our own domain with zero infrastructure management

## How we use it at Orbiter

This documentation site is built with Mintlify. The source lives at [`M-Pederson/docs`](https://github.com/M-Pederson/docs) on GitHub. Pages cover the graph ontology, cypher queries, enrichment pipeline documentation (including LLM system prompts and model summaries), skills, and the Xano API reference.

Key conventions:
- **Guides tab** — architecture, ontology, cyphers, enrichment pipelines, skills
- **API Reference tab** — Xano-generated endpoint docs
- **`docs.json`** — all navigation, tabs, and site config
- **Local dev** — `mintlify dev` on port 3000
