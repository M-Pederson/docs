The complete edge catalog for the [Music Industry Waterfall](/guides/enrichment/waterfall/music-industry-waterfall). Weights follow the canonical [Edge Weights](/guides/ontology/edge-weights) scale (**lower = stronger**; hub-node edges sit Weak or below; never sub-1). See [Music Nodes](/guides/enrichment/waterfall/music-nodes) for the node side.

<Warning>
**Live audit 2026-07-01, forward path only.** The music edge-description allowlist contains 25 edge types. Forward docs treat **21** as actively forward-emitted today: `COLLABORATED_WITH`, `SUPPORTING_MUSICIAN_FOR`, and `TRIBUTE_TO` are still design/allowlist entries with no writer, and `FROM_AREA` is excluded until it is wired into ingest instead of a sweep. Fixed-forward 2026-07-01: `SAME_AS` now uses identity weight **1** in #13091/#13121, and forward edge writers stamp semantic-null `description` / `description_generated_at` on create.
</Warning>

<Note>
**`Artist` in a pattern below = `Person:Music_Artist` (a person) *or* `Music_Group` (a band).** Either can perform, release, headline, or be credited. `MEMBER_OF` is the exception ‚Äî it is always `(Person:Music_Artist)‚Üí(Music_Group)`. Recording relationship credits (`PLAYED_ON`, `PRODUCED`, `ENGINEERED`) are **type-aware** in #13100: person credits land on `Person:Music_Artist`, group credits land on `Music_Group`.
</Note>

<Note>
**Every live edge carries the standard envelope.** Structural writers stamp `uuid`, `weight`, `created_at`, `updated_at`, and semantic-null `description` / `description_generated_at`; the async edge describer later stamps both fields with a GOOD generated description.
</Note>

## Standard edge envelope

Every music edge should converge on this relationship envelope. Live structural writers stamp `uuid`, `weight`, `created_at`, `updated_at`, and semantic-null `description` / `description_generated_at`; the async describer fills the description when available (FalkorDB `timestamp()` = epoch-millis).

```cypher
MERGE (a)-[r:EDGE_TYPE]->(b)
SET r.uuid                    = coalesce(r.uuid, randomUUID()),
    r.weight                  = <band weight>,
    r.created_at              = coalesce(r.created_at, timestamp()),
    r.updated_at              = timestamp(),
    r.description             = <generated sentence | null>,
    r.description_generated_at = <timestamp | null>,
    // ‚Ä¶ edge-specific parameters ‚Ä¶
```

| Field | Type | Required | Notes |
|---|---|:--:|---|
| `uuid` | string | ‚úÖ | `randomUUID()`, stable across re-runs via `coalesce` |
| `weight` | int | ‚úÖ | from the [edge-weights scale](/guides/ontology/edge-weights) |
| `created_at` | timestamp | ‚úÖ | `coalesce(r.created_at, timestamp())` ‚Äî set once |
| `updated_at` | timestamp | ‚úÖ | `timestamp()` ‚Äî bumped every write |
| `description` | string | ‚úÖ | one-sentence NL summary; semantic null until the description pass runs |
| `description_generated_at` | timestamp | ‚úÖ | guard for the async describer (mirrors the IMDB `WORKED_ON` pattern); semantic null until generated |

## Structural music (hub-aware)

| Edge | Pattern | Weight | Edge-specific parameters | Sample `description` |
|---|---|:--:|---|---|
| `[:RECORDING_OF]` | `(Recording)‚Üí(Work)` | **30** | `live` (bool) ¬∑ `partial` (bool) ¬∑ `cover` (bool) | "This recording is a studio performance of the work 'Smells Like Teen Spirit'." |
| `[:RELEASED]` | `(Artist)‚Üí(Release_Group)` | **10** | `joinphrase` (str) | "Nirvana released the album 'Nevermind' in 1991." |
| `[:RELEASE_OF]` | `(Release)‚Üí(Release_Group)` | **30** | ‚Äî | "This 1991 CD is the canonical official release of the album 'Nevermind'." ‚Äî effectively 1:1, since only the [canonical release](/guides/enrichment/waterfall/music-industry-waterfall#resolved-design-decisions-2026-06-11) materializes per release-group |
| `[:APPEARS_ON]` | `(Recording)‚Üí(Release)` | **45** | `track_number` (int) ¬∑ `position` (int) ¬∑ `disc` (int) | "'Smells Like Teen Spirit' appears as track 1 on the 'Nevermind' release." |
| `[:RELEASED_ON]` | `(Release)‚Üí(Company:Music_Label)` | **50** | `catalog_number` (str) | "The 'Nevermind' release was issued on DGC Records (cat. DGCD-24425)." |
| `[:ON_RECORD_LABEL]` | `(Recording)‚Üí(Company:Music_Label)` | **55** | `via_release` (str ‚Äî canonical-release mbid the label was derived from) ¬∑ `catalog_number` (str ‚Äî carried from that release's `RELEASED_ON`) | "'Smells Like Teen Spirit' was released by DGC Records (cat. DGCD-24425)." |
| `[:SUBSIDIARY_OF]` | `(Company:Music_Label)‚Üí(Company:Music_Label)` | **15** | `begin` (str) ¬∑ `end` (str) | "Sub Pop Records is partly owned by Warner Music Group." |

<Note>
**`ON_RECORD_LABEL` is a *derived convenience edge*, not a fresh MusicBrainz lookup ‚Äî built 2026-06-17.** In MusicBrainz the label belongs to the **release** (a release has `label-info[] = {label, catalog_number}`), so the authoritative tie is the existing `(Release)-[:RELEASED_ON]->(Company:Music_Label)` edge (weight **50**). `ON_RECORD_LABEL` materializes the one-hop shortcut `(Recording)-[:APPEARS_ON]->(Release)-[:RELEASED_ON]->(Company:Music_Label)` directly onto the recording, sourced from its **canonical release** (only one `Release` materializes per `Release_Group`). It is weight **55** ‚Äî one notch *weaker* than the direct `RELEASED_ON` (50) because it is derived and indirect, and it stays in the **Weak** band as a [hub-node edge](/guides/ontology/edge-weights#weight-scale) (a label connects thousands of recordings). `via_release` records which release the label came from for provenance/audit. **Multiple-labels rule:** a recording can appear on releases from different labels ‚Äî the canonical release's label wins. The Recording writer ([`mb/run-base-recording-enrich` #13100](/guides/enrichment/waterfall/music-functions)) does **not** fetch label/release data (`inc = artist-credits+work-rels+artist-rels+place-rels+genres+isrcs+ratings`), so this edge is **derived in-graph** by #13131 when the release-group orchestrator (#13099) calls it with `release_mbid` for the just-created canonical release. `RELEASED_ON` remains the source of truth; `ON_RECORD_LABEL` is a denormalized read-time accelerator.
</Note>

<Note>

## Edge descriptions

The `description` + `description_generated_at` fields mirror the IMDB pipeline's `WORKED_ON` edge-description pattern ([#13016 / #13017 / #13018](/guides/enrichment/waterfall/imdb-waterfall-plan#tooling-operator-invoked-not-pipeline-wired)). The music equivalents were **built 2026-06-12** ‚Äî and unlike the IMDB/person pipelines, which ship one trio per edge type, **a single generic trio covers all music edge types**, keyed on `e.uuid`:

<Card icon="wrench">

```
mb/edge/create-music-edge-description ‚Äî 13095
mb/edge/write-music-edge-description ‚Äî 13096
mb/tool/backfill-music-edge-descriptions ‚Äî 13097  (legacy name; forward async describer)
```

- **create** (v1.4) ‚Äî given an edge's structured properties (with the opaque `via_release` mbid stripped) + both endpoint node names, **`deepseek/deepseek-v4-flash`** (temp 0.3, `reasoning:{enabled:false}` + `max_tokens:200` ‚Äî it's a reasoning model, see trap #13) returns a single factual 15‚Äì30-word sentence, then **cleans the output**: strips wrapping quotes + leading meta-preambles ("According to‚Ä¶", "Based on‚Ä¶"), normalizes U+2011/U+2010 ‚Üí `-`, and a GOOD-or-null reject guard maps any refusal/meta-commentary to empty. **Model swapped from `gpt-oss-120b` (2026-06-17)** to eliminate its ~15% dropped-space tokenizer artifact ("Stevensis", "Themusicgroup").
- **write** ‚Äî sets `e.description` + `e.description_generated_at` (ISO) on the edge via Cypher, matched on `e.uuid`. **No-ops on an empty description** (a guard-rejected edge stays unstamped ‚Üí retries on a future async description pass).
- **async describe** (v1.4) ‚Äî drains newly written edges where `description_generated_at IS NULL`, selected by edge type (the **25**-type allowlist now includes `ON_RECORD_LABEL`) + `labels()` membership, in `try_catch` batches of 1‚Äì100 (default 10); idempotent via the guard.

</Card>

Driven by **Task 181** `backfill-music-edge-descriptions` (legacy name; every 300 s, 20 edges/tick ‚Äî **ACTIVE as of 2026-06-17**; self-maintaining, cheap no-op when no pending) and the operator endpoint `GET tool/peek-edge-descriptions` (#8646) in the `musicbrainz` API group ‚Äî see [Music Waterfall Functions](/guides/enrichment/waterfall/music-functions#llm-edge-descriptions). Sandbox smoke test processed 11/11 edges with 0 pending; a real generated sample:

> "Kurt Cobain was an original member of the US grunge band Nirvana from 1987 until April 5 1994."

<Note>
Because `description` is generated *after* the structural write, new edges start as semantic `description = null` and `description_generated_at = null` until the async describer fills them. Fixed-forward 2026-07-01: every forward structural writer stamps those null fields on create.
</Note>

<Note>
**GOOD-or-null applies to edge descriptions too ‚Äî now enforced in code.** An edge `description` is a clean factual sentence **or** `null` ‚Äî never an LLM refusal/meta-commentary echo or placeholder. As of **#13095 v1.3+** the music edge describer enforces this: an output reject-guard maps refusal/meta-commentary to empty (writer #13096 skips empty ‚Üí the edge stays unstamped and retries), plus a preamble-strip + U+2011/U+2010 normalization. The model was **swapped `gpt-oss-120b` ‚Üí `deepseek-v4-flash` (2026-06-17)** to remove the residual ~15% dropped-space artifact. The cosmetic gpt-oss artifacts now remain **only on the people-graph edge describers** (#13006/#13011/#13018) ‚Äî see [edge-audit item 5](/guides/robert-mark/edge-audit#audit-findings). Node-side enforcement: [GOOD-or-null invariant](/guides/dev-updates/2026-06-16-music-node-descriptions#good-or-null-invariant).
</Note>
