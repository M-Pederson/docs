## Overview

Retrieve all suggestion request records, filtered by copilot mode and optional search query.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login or signup.
</ParamField>

## Query Parameters

<ParamField body="copilot_mode" type="string" required>
  The copilot mode to filter by. One of `outcome`, `loop`, or `serendipity`.
</ParamField>

<ParamField body="query" type="string">
  Optional search term to filter results.
</ParamField>

<ParamField body="page" type="integer">
  Page number for pagination.
</ParamField>

<ParamField body="limit" type="integer">
  Number of items per page.
</ParamField>

<ParamField body="offset" type="integer">
  Number of items to skip.
</ParamField>

## Response

<ResponseField name="itemsReceived" type="integer">
  Number of items returned in the current page.
</ResponseField>

<ResponseField name="curPage" type="integer">
  Current page number.
</ResponseField>

<ResponseField name="nextPage" type="integer">
  Next page number, or null if no more pages.
</ResponseField>

<ResponseField name="prevPage" type="integer">
  Previous page number, or null if on first page.
</ResponseField>

<ResponseField name="items" type="array">
  Array of suggestion request records.
</ResponseField>

<RequestExample>
```bash cURL
curl -X GET "https://xh2o-yths-38lt.n7c.xano.io/api:MkA4QsNh/suggestion-requests?copilot_mode=outcome&page=1&limit=20" \
  -H "Content-Type: application/json" \
  -H "X-Data-Source: YOUR_DATA_SOURCE" \
  -H "X-Branch: v1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "itemsReceived": 10,
  "curPage": 1,
  "nextPage": 2,
  "prevPage": null,
  "items": []
}
```
</ResponseExample>
