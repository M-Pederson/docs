All Cypher queries for creating Film & TV title nodes, IMDB credit edges (WORKED_ON), and Film & TV Award nodes with award edges (HAS_AWARD, WON_AWARD, NOMINATED_FOR).

<Warning>
**`#10514 add-imdb-credit-edges` was deleted 2026-04-23.** The IMDB pipeline was replaced by the waterfall architecture described in [IMDB Waterfall Plan](/guides/enrichment/waterfall/imdb-waterfall). `WORKED_ON` credit edges are now created inline by `#2334 add-node-edge-imdb-award` (documented below) and by `#12881 run-base-imdb-title-enrich` as part of the title enrichment graph write.

`#2320 add-film-tv-node` and `#2334 add-node-edge-imdb-award` are still in active use.
</Warning>

## mvp/node/add-film-tv-node — #2320 ⚪

> ⚪ Illustrative — this function assembles its Cypher in an `api.lambda` string builder; the fence below approximates the generated query and is deliberately not byte-synced. *(Cypher Sync Audit, 2026-07-06.)*

Idempotently MERGEs a Film_TV node for an IMDB title keyed on `imdb_url` (v2, 2026-05-17 — switched `CREATE` → `MERGE` so it no longer duplicates nodes already created by `#2334` in award context). `ON CREATE` stamps the full property set; `ON MATCH` re-stamps the mutable fields plus `updated_at`. The node always gets `:Entity` plus a dynamic type label based on the title type (Movie, TV_Series, TV_Movie, TV_Mini_Series, Short_Film, Documentary, TV_Special, Video_Game, Music_Video).

```text
mvp/node/add-film-tv-node — #2320
```

```cypher
// Version v2
MERGE (f:Film_TV { imdb_url: $imdbUrl })
ON CREATE SET
  f.uuid = randomUUID(),
  f.created_at = $now,
  f.display_label = "Film_TV",
  f.title = $title,
  f.type = $type,
  f.tagline = $tagline,
  f.description = $description,
  f.genres = $genres,
  f.year = $year,
  f.imdb_rating = $imdbRating,
  f.running_time = $runningTime,
  f.mpaa_rating = $mpaaRating,
  f.countries = $countries,
  f.languages = $languages,
  f.image_url = $imageUrl,
  f.source = $source,
  f.name_embedding = vecf32($embeddingArray)
ON MATCH SET
  f.updated_at = $now,
  f.title = $title,
  f.type = $type,
  f.tagline = $tagline,
  f.description = $description,
  f.genres = $genres,
  f.year = $year,
  f.imdb_rating = $imdbRating,
  f.running_time = $runningTime,
  f.mpaa_rating = $mpaaRating,
  f.countries = $countries,
  f.languages = $languages,
  f.image_url = $imageUrl,
  f.source = $source
SET f:Entity
// SET f:<labelType> — dynamic type label (Movie, TV_Series, TV_Movie,
// TV_Mini_Series, Short_Film, Documentary, TV_Special, Video_Game,
// Music_Video), appended only when the title type maps to one.
RETURN f.uuid AS uuid
```

## mvp/edge/add-node-edge-imdb-award — #2334

Creates Film_TV_Award nodes and all award-related edges. As of **v1.6** (2026-04-24) the function splits the work into two embedded Cyphers plus a fallback read-back:

- **Query A** (always runs) — MERGEs the Film_TV node (with dynamic type label) and HAS_AWARD edge **only when `hasFilm`**, then MERGEs the Film_TV_Award node (with Winner/Nominee label). For person-only honorary awards (`hasFilm` false, null `film_tv_id`), the Award MERGE key swaps `film_tv_imdb_url` → `person_uuid` to preserve per-person uniqueness. Returns `a.uuid`. Query A is run up to twice (idempotent MERGE retry on a null read-back).
- **Fallback** — if both Query A attempts return null, a MATCH-only read-back fetches the just-MERGEd Award node by its MERGE keys.
- **Query B** (only when `hasPerson` AND Query A returned a uuid) — MATCHes Person + Award, MERGEs the WORKED_ON edge (gated on `hasFilm`), and the WON_AWARD or NOMINATED_FOR person→award edge (always fires so person-only awards still connect).

The **§3.6 edge-weight audit** (2026-05-31) raised the hub-node edge weights out of sub-1 values: `HAS_AWARD` 0.5 → **40**, `WORKED_ON` 0.5 → **30**, `WON_AWARD` 0.3 → **30**, `NOMINATED_FOR` 0.7 → **35**. The pre-v1.5 `w.credit_role` property on WORKED_ON is gone.

Live encodes every string literal via `JSON.stringify` (double-quoted, Cypher-safe). The Cypher-param form below (`$imdbUrl`, `$now`, `$personUuid`, etc.) renders that cleanly.

```text
mvp/edge/add-node-edge-imdb-award — #2334
```

### Query A — Film_TV + Film_TV_Award + HAS_AWARD

```cypher
// v1.5 Query A — JSON-quoted literals, hasFilm-gated Film_TV
{% if $var.lit.hasFilm %}
MERGE (f:Film_TV { imdb_url: {{ $var.lit.imdbUrl }} })
ON CREATE SET
  f.uuid = randomUUID(),
  f.created_at = {{ $var.lit.now }},
  f.display_label = 'Film_TV',
  f.title = {{ $var.lit.title }},
  f.type = {{ $var.lit.type }},
  {% if $var.lit.hasTagline %}
  f.tagline = {{ $var.lit.tagline }},
  {% endif %}
  f.description = {{ $var.lit.description }},
  f.genres = {{ $var.lit.genres }},
  f.year = {{ $var.lit.year }},
  {% if $var.lit.hasImdbRating %}
  f.imdb_rating = {{ $var.lit.imdbRating }},
  {% endif %}
  {% if $var.lit.hasRunningTime %}
  f.running_time = {{ $var.lit.runningTime }},
  {% endif %}
  {% if $var.lit.hasMpaaRating %}
  f.mpaa_rating = {{ $var.lit.mpaaRating }},
  {% endif %}
  f.countries = {{ $var.lit.countries }},
  f.languages = {{ $var.lit.languages }},
  {% if $var.lit.hasImageUrl %}
  f.image_url = {{ $var.lit.imageUrl }},
  {% endif %}
  f.source = {{ $var.lit.source }}
ON MATCH SET
  f.updated_at = {{ $var.lit.now }},
  f.title = {{ $var.lit.title }},
  f.description = {{ $var.lit.description }},
  {% if $var.lit.hasImdbRating %}
  f.imdb_rating = {{ $var.lit.imdbRating }},
  {% endif %}
  {% if $var.lit.hasImageUrl %}
  f.image_url = {{ $var.lit.imageUrl }},
  {% endif %}
  f.source = {{ $var.lit.source }}

{% if $var.lit.filmType == 'movie' %}
SET f:Movie
{% elseif $var.lit.filmType == 'tvSeries' or $var.lit.filmType == 'tv_series' %}
SET f:TV_Series
{% elseif $var.lit.filmType == 'tvMovie' or $var.lit.filmType == 'tv_movie' %}
SET f:TV_Movie
{% elseif $var.lit.filmType == 'tvMiniSeries' or $var.lit.filmType == 'tv_mini_series' %}
SET f:TV_Mini_Series
{% elseif $var.lit.filmType == 'short' %}
SET f:Short_Film
{% elseif $var.lit.filmType == 'documentary' %}
SET f:Documentary
{% elseif $var.lit.filmType == 'tvSpecial' or $var.lit.filmType == 'tv_special' %}
SET f:TV_Special
{% endif %}

WITH f
{% endif %}

MERGE (a:Film_TV_Award {
  year: {{ $var.lit.awardYearInt }},
  award_type: {{ $var.lit.awardType }},
  award_description: {{ $var.lit.awardDescription }},
  {% if $var.lit.hasFilm %}
  film_tv_imdb_url: {{ $var.lit.projectImdbUrl }}
  {% else %}
  person_uuid: {{ $var.lit.personUuid }}
  {% endif %}
})
ON CREATE SET
  a.uuid = randomUUID(),
  a.created_at = {{ $var.lit.now }},
  a.display_label = 'Film_TV_Award',
  a.description = {{ $var.lit.awardText }},
  a.nominee_or_winner = {{ $var.lit.nomineeOrWinner }},
  {% if $var.lit.hasSharedWith %}
  a.shared_with = {{ $var.lit.sharedWith }},
  {% endif %}
  a.project_title = {{ $var.lit.projectTitle }},
  a.name_embedding = vecf32({{ $var.awardEmbeddingArray }})
ON MATCH SET
  a.updated_at = {{ $var.lit.now }},
  a.description = {{ $var.lit.awardText }},
  a.nominee_or_winner = {{ $var.lit.nomineeOrWinner }},
  {% if $var.lit.hasSharedWith %}
  a.shared_with = {{ $var.lit.sharedWith }},
  {% endif %}
  a.project_title = {{ $var.lit.projectTitle }},
  a.name_embedding = vecf32({{ $var.awardEmbeddingArray }})

{% if $var.lit.isWinner %}
SET a:Winner
{% else %}
SET a:Nominee
{% endif %}

{% if $var.lit.hasFilm %}
MERGE (f)-[h:HAS_AWARD]->(a)
ON CREATE SET
  h.created_at = {{ $var.lit.now }},
  h.valid_from = {{ $var.lit.validFrom }},
  h.weight = 40,
  h.source = {{ $var.lit.source }}
ON MATCH SET
  h.updated_at = {{ $var.lit.now }},
  h.weight = 40,
  h.source = {{ $var.lit.source }}
{% endif %}

RETURN a.uuid AS uuid
```

### Query B — Person edges (WORKED_ON + WON_AWARD / NOMINATED_FOR)

```cypher
// cypherB
// v1.5 Query B — hasFilm-gated WORKED_ON edge
MATCH (p:Person { uuid: {{ $var.lit.personUuid }} })
{% if $var.lit.hasFilm %}
MATCH (f:Film_TV { imdb_url: {{ $var.lit.imdbUrl }} })
{% endif %}
MATCH (a:Film_TV_Award { uuid: "{{ $var.extractedUuid }}" })

{% if $var.lit.hasFilm %}
MERGE (p)-[w:WORKED_ON]->(f)
ON CREATE SET
  w.created_at = {{ $var.lit.now }},
  w.valid_from = {{ $var.lit.validFrom }},
  w.weight = 30,
  {% if $var.workedOnEdgeDescription is not empty %}
  w.description = {{ $var.lit.workedOnDescription }},
  w.description_generated_at = {{ $var.lit.now }},
  {% endif %}
  w.source = {{ $var.lit.source }}
ON MATCH SET
  w.updated_at = {{ $var.lit.now }},
  w.weight = 30,
  {% if $var.workedOnEdgeDescription is not empty %}
  w.description = {{ $var.lit.workedOnDescription }},
  w.description_generated_at = {{ $var.lit.now }},
  {% endif %}
  w.source = {{ $var.lit.source }}
{% endif %}

{% if $var.lit.isWinner %}
MERGE (p)-[r:WON_AWARD]->(a)

  r.valid_to = {{ $var.lit.validTo }},

  r.description = {{ $var.lit.edgeDescription }},
  {% if $var.generatedAwardEdgeDescription is not empty %}
  r.description_generated_at = {{ $var.lit.now }},
  {% endif %}
  r.source = {{ $var.lit.source }}
ON MATCH SET
  r.updated_at = {{ $var.lit.now }},
  r.weight = 30,
  r.description = {{ $var.lit.edgeDescription }},
  {% if $var.generatedAwardEdgeDescription is not empty %}
  r.description_generated_at = {{ $var.lit.now }},
  {% endif %}
  r.source = {{ $var.lit.source }}
{% else %}
MERGE (p)-[r:NOMINATED_FOR]->(a)
ON CREATE SET
  r.created_at = {{ $var.lit.now }},
  r.valid_from = {{ $var.lit.validFrom }},
  r.valid_to = {{ $var.lit.validTo }},
  r.weight = 35,
  r.description = {{ $var.lit.edgeDescription }},
  {% if $var.generatedAwardEdgeDescription is not empty %}
  r.description_generated_at = {{ $var.lit.now }},
  {% endif %}
  r.source = {{ $var.lit.source }}
ON MATCH SET
  r.updated_at = {{ $var.lit.now }},
  r.weight = 35,
  r.description = {{ $var.lit.edgeDescription }},
  {% if $var.generatedAwardEdgeDescription is not empty %}
  r.description_generated_at = {{ $var.lit.now }},
  {% endif %}
  r.source = {{ $var.lit.source }}
{% endif %}

RETURN p.uuid AS p_uuid
```

### Fallback — MATCH-only Award read-back (v1.6)

Runs only when both Query A attempts return a null uuid (MERGE committed but the `RETURN` clause dropped the row). Reads the Award node back by the same MERGE keys Query A used.

```cypher
// v1.6 Fallback — MATCH-only read-back by MERGE keys
MATCH (a:Film_TV_Award {
  year: {{ $var.lit.awardYearInt }},
  award_type: {{ $var.lit.awardType }},
  award_description: {{ $var.lit.awardDescription }},
  {% if $var.lit.hasFilm %}
  film_tv_imdb_url: {{ $var.lit.projectImdbUrl }}
  {% else %}
  person_uuid: {{ $var.lit.personUuid }}
  {% endif %}
})
RETURN a.uuid AS uuid
LIMIT 1
```
