The IMDB (Film & TV) Waterfall uses the canonical graph node definitions in [Ontology → Node Types](/guides/ontology/nodes). Titles and awards get dedicated Film & TV node types; people reuse the shared `Person` node, extended with IMDB roles, biography, and avatar.

This page is the **pipeline-scoped catalog**: which node types this pipeline writes, keyed how, by which function. Node labels, properties, vector fields, and examples should be edited only in the ontology source of truth.

<Note>
  Ground truth: [Node Types](/guides/ontology/nodes). The waterfall implementation should link to those sections rather than redefining node schemas. Write-side Cypher for the Film & TV nodes lives on [Cyphers → Film & TV](/guides/cyphers/film-tv).
</Note>

## Titles

### Film\_TV

Canonical section: [Film\_TV](/guides/ontology/nodes#film_tv). The title node — `:Entity:Film_TV` plus a dynamic subtype label (`:Movie`, `:TV_Series`, `:TV_Movie`, `:TV_Mini_Series`, `:Short_Film`, `:Documentary`, `:TV_Special`, `:Video_Game`, `:Music_Video`). MERGE-keyed on `imdb_url` by [`add-film-tv-node` #2320](/guides/cyphers/film-tv). Carries a `name_embedding` semantic vector.

## Awards

### Film\_TV\_Award

Canonical section: [Film\_TV\_Award](/guides/ontology/nodes#film_tv_award). The award node — `:Entity:Film_TV_Award` plus a `:Winner` / `:Nominee` subtype label. MERGE-keyed on `(year, award_type, award_description, film_tv_imdb_url)`, or on `person_uuid` for person-only honoraries. Written by [`add-node-edge-imdb-award` #2334](/guides/cyphers/film-tv). Carries a `name_embedding` semantic vector.

## People

### Person

Canonical section: [Person](/guides/ontology/nodes#person). IMDB does **not** create a person type — it enriches the **shared** `Person` node with IMDB roles, short/long biography, and avatar, and stamps `visibility = true` in Phase 4a. The `imdb_person.master_person_id` \+ `node_uuid` link the Xano row to the graph node (`add-person-node` #2601 / `update-person-node` #2758 — both dual-write `name_embedding` (OpenAI) **and** `embeddings` (Gemini 3072) since 2026-06-15).

## Expertise

### SubDomainExpertise

Canonical section: [SubDomainExpertise](/guides/ontology/nodes#subdomainexpertise). Phase 4b runs `llm-identify-person-expertise` (#12666 v2) → `resolve-person-expertise-v2` (#12926), which MERGEs `SubDomainExpertise` nodes and connects them to the `Person` via `HAS_EXPERTISE` edges — see [IMDB Edges](/guides/enrichment/waterfall/imdb-edges).

## Timestamp coverage

Every node create/update reachable from the IMDB waterfall stamps `created_at` on create and `updated_at` on update. The per-node-type record is audited in [Node Types → Timestamp coverage audit](/guides/ontology/nodes#timestamp-coverage-audit).
