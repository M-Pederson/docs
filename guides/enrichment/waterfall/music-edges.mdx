The complete edge catalog for the [Music Industry Waterfall](/guides/enrichment/waterfall/music-industry-waterfall). Weights follow the canonical [Edge Weights](/guides/ontology/edge-weights) scale (**lower = stronger**; hub-node edges sit Weak or below; never sub-1). See [Music Nodes](/guides/enrichment/waterfall/music-nodes) for the node side.

<Note>
**`Artist` in a pattern below = `Person:Music_Artist` (a person) *or* `Music_Group` (a band).** Either can perform, release, or be credited. `MEMBER_OF` is the exception — it is always `(Person:Music_Artist)→(Music_Group)`. Person-only roles (`PLAYED_ON`, `PRODUCED`, `ENGINEERED`) use `Person:Music_Artist` explicitly.
</Note>

<Note>
**Every edge carries the standard envelope.** At a minimum, **all** edges below are created with `created_at`, `updated_at`, and `description` — plus `uuid` and `weight`. Edge-specific parameters are listed per edge. The natural-language `description` is populated by a post-write LLM pass (see [Edge descriptions](#edge-descriptions)); it is `null` between write and generation.
</Note>

## Standard edge envelope

Every music edge is written with this property set on the relationship variable `r`. `created_at` uses `coalesce` so re-enrichment never resets it; `updated_at` is stamped on every write (FalkorDB `timestamp()` = epoch-millis).

```cypher
MERGE (a)-[r:EDGE_TYPE]->(b)
SET r.uuid                    = coalesce(r.uuid, randomUUID()),
    r.weight                  = <band weight>,
    r.created_at              = coalesce(r.created_at, timestamp()),
    r.updated_at              = timestamp(),
    r.description             = <generated sentence | null>,
    r.description_generated_at = <timestamp | null>,
    // … edge-specific parameters …
```

| Field | Type | Required | Notes |
|---|---|:--:|---|
| `uuid` | string | ✅ | `randomUUID()`, stable across re-runs via `coalesce` |
| `weight` | int | ✅ | from the [edge-weights scale](/guides/ontology/edge-weights) |
| `created_at` | timestamp | ✅ | `coalesce(r.created_at, timestamp())` — set once |
| `updated_at` | timestamp | ✅ | `timestamp()` — bumped every write |
| `description` | string | ✅ | one-sentence NL summary; `null` until the description pass runs |
| `description_generated_at` | timestamp | ✅ | guard for the backfill (mirrors the IMDB `WORKED_ON` pattern); `null` until generated |

## Structural music (hub-aware)

| Edge | Pattern | Weight | Edge-specific parameters | Sample `description` |
|---|---|:--:|---|---|
| `[:RECORDING_OF]` | `(Recording)→(Work)` | **30** | `live` (bool) · `partial` (bool) · `cover` (bool) | "This recording is a studio performance of the work 'Smells Like Teen Spirit'." |
| `[:RELEASED]` | `(Artist)→(Release_Group)` | **10** | `joinphrase` (str) | "Nirvana released the album 'Nevermind' in 1991." |
| `[:RELEASE_OF]` | `(Release)→(Release_Group)` | **30** | — | "This 1991 CD is the canonical official release of the album 'Nevermind'." — effectively 1:1, since only the [canonical release](/guides/enrichment/waterfall/music-industry-waterfall#resolved-design-decisions-2026-06-11) materializes per release-group |
| `[:APPEARS_ON]` | `(Recording)→(Release)` | **45** | `track_number` (int) · `position` (int) · `disc` (int) | "'Smells Like Teen Spirit' appears as track 1 on the 'Nevermind' release." |
| `[:RELEASED_ON]` | `(Release)→(Music_Label)` | **50** | `catalog_number` (str) | "The 'Nevermind' release was issued on DGC Records (cat. DGCD-24425)." |
| `[:ON_RECORD_LABEL]` | `(Recording)→(Music_Label)` | **55** | `via_release` (str — canonical-release mbid the label was derived from) · `catalog_number` (str — carried from that release's `RELEASED_ON`) | "'Smells Like Teen Spirit' was released by DGC Records (cat. DGCD-24425)." |
| `[:SUBSIDIARY_OF]` | `(Music_Label)→(Music_Label)` | **15** | `begin` (str) · `end` (str) | "Sub Pop Records is partly owned by Warner Music Group." |

<Note>
**`ON_RECORD_LABEL` is a *derived convenience edge*, not a fresh MusicBrainz lookup — built 2026-06-17.** In MusicBrainz the label belongs to the **release** (a release has `label-info[] = {label, catalog_number}`), so the authoritative tie is the existing `(Release)-[:RELEASED_ON]->(Music_Label)` edge (weight **50**). `ON_RECORD_LABEL` materializes the one-hop shortcut `(Recording)-[:APPEARS_ON]->(Release)-[:RELEASED_ON]->(Music_Label)` directly onto the recording, sourced from its **canonical release** (only one `Release` materializes per `Release_Group`). It is weight **55** — one notch *weaker* than the direct `RELEASED_ON` (50) because it is derived and indirect, and it stays in the **Weak** band as a [hub-node edge](/guides/ontology/edge-weights#weight-scale) (a label connects thousands of recordings). `via_release` records which release the label came from for provenance/audit. **Multiple-labels rule:** a recording can appear on releases from different labels — the canonical release's label wins. The Recording writer ([`mb/run-base-recording-enrich` #13100](/guides/enrichment/waterfall/music-functions)) does **not** fetch label/release data (`inc = artist-credits+work-rels+artist-rels+place-rels+genres+isrcs+ratings`), so this edge is **derived in-graph** (not by expanding the MB `inc`) by [`mvp/music/backfill-on-record-label-edges` #13131](/guides/enrichment/waterfall/music-functions#on_record_label-edges) via the operator endpoint `POST tool/backfill-on-record-label-edges` (#8688). Idempotent `MERGE` per (recording, label) pair; pass `recording_mbid` to scope to one recording (forward/targeted), or omit for a full sweep. **Sandbox-verified 2026-06-17** (12 edges, weight 55, `via_release`/`catalog_number` carried, idempotent). `RELEASED_ON` remains the source of truth; `ON_RECORD_LABEL` is a denormalized read-time accelerator.
</Note>

## Identity & ownership bridges

| Edge | Pattern | Weight | Edge-specific parameters | Sample `description` |
|---|---|:--:|---|---|
| `[:SAME_AS]` | `(Music_Label)→(Company)` | **1** | `match_method` (str) · `confidence` (float 0–1) | "Sub Pop Records is the same entity as the Orbiter company profile for Sub Pop." |
| `[:OWNED_BY]` | `(Music_Label)→(Company)` | **15** | `match_method` (str: `mb_subsidiary` / `exa_llm`) · `confidence` (float 0–1) · `source` (str — citation URL when Exa-derived) | "Matador, a US indie rock label, is owned by the Beggars Group." |

<Warning>
`SAME_AS` now applies to **labels only** (`Music_Label`→`Company`). **Person artists are not bridged** — a `Person:Music_Artist` *is* an Orbiter `Person` node, so it reconciles to an existing person by **node dedup/MERGE** (name + ISNI + URLs), not by an edge. `SAME_AS` is written by a **separate post-ingest reconciliation pass** — never at ingest (decided 2026-06-11) — and its `description` records *how* the match was made (`match_method`, `confidence`) for auditability. See [resolved design decisions](/guides/enrichment/waterfall/music-industry-waterfall#resolved-design-decisions-2026-06-11).
</Warning>

<Note>
**`OWNED_BY` is BUILT + forward-path-live (2026-06-17).** It captures the *owning company* of a label, distinct from `SAME_AS` (the label **is** a company) and `SUBSIDIARY_OF` (label→label music-catalog lineage). Resolved in **two tiers** by `mb/resolve-and-link-label-owner` (#13133): MB `label-rels` parent (`mb_subsidiary`) → with **Tier-1→Tier-2 fallback** to Exa/LLM owner discovery (`exa_llm`, source-cited, confidence ≥ 0.7, via #13132) when the MB parent doesn't resolve to a real company — both converging on `master-company-new` (#12558) and the direct label→Company edge. Wired into `mb/run-base-label-enrich` (#13091) at depth-0. Full spec: [Build spec — label owner resolution](/guides/enrichment/waterfall/music-reconciliation#build-spec--label-owner-resolution-owned_by).
</Note>

## Edge descriptions

The `description` + `description_generated_at` fields mirror the IMDB pipeline's `WORKED_ON` edge-description pattern ([#13016 / #13017 / #13018](/guides/enrichment/waterfall/imdb-waterfall-plan#tooling-operator-invoked-not-pipeline-wired)). The music equivalents were **built 2026-06-12** — and unlike the IMDB/person pipelines, which ship one trio per edge type, **a single generic trio covers all music edge types**, keyed on `e.uuid`:

<Card icon="wrench">

```
mb/edge/create-music-edge-description — 13095
mb/edge/write-music-edge-description — 13096
mb/tool/backfill-music-edge-descriptions — 13097
```

- **create** (v1.4) — given an edge's structured properties (with the opaque `via_release` mbid stripped) + both endpoint node names, **`deepseek/deepseek-v4-flash`** (temp 0.3, `reasoning:{enabled:false}` + `max_tokens:200` — it's a reasoning model, see trap #13) returns a single factual 15–30-word sentence, then **cleans the output**: strips wrapping quotes + leading meta-preambles ("According to…", "Based on…"), normalizes U+2011/U+2010 → `-`, and a GOOD-or-null reject guard maps any refusal/meta-commentary to empty. **Model swapped from `gpt-oss-120b` (2026-06-17)** to eliminate its ~15% dropped-space tokenizer artifact ("Stevensis", "Themusicgroup").
- **write** — sets `e.description` + `e.description_generated_at` (ISO) on the edge via Cypher, matched on `e.uuid`. **No-ops on an empty description** (a guard-rejected edge stays unstamped → retries next backfill).
- **backfill** (v1.4) — drains edges where `description_generated_at IS NULL`, selected by edge type (the **25**-type allowlist now includes `ON_RECORD_LABEL`) + `labels()` membership, in `try_catch` batches of 1–100 (default 10); idempotent via the guard. **`mode:"reset"`** nulls already-stamped music-edge descriptions (scoped to the `X-Data-Source` graph) so a normal drain re-describes them with the current model — used for the gpt-oss → deepseek re-describe.

</Card>

Driven by **Task 181** `backfill-music-edge-descriptions` (every 300 s, 20 edges/tick — shipped **INACTIVE**) and the operator endpoints `POST tool/backfill-edge-descriptions` (#8645) / `GET tool/peek-edge-descriptions` (#8646) in the `musicbrainz` API group — see [Music Waterfall Functions](/guides/enrichment/waterfall/music-functions#llm-edge-descriptions). Sandbox smoke test processed 11/11 edges with 0 pending; a real generated sample:

> "Kurt Cobain was an original member of the US grunge band Nirvana from 1987 until April 5 1994."

<Note>
Because `description` is generated *after* the structural write, every edge is created with `description = null` and `description_generated_at = null`, then filled by the backfill. The two fields are part of the [standard envelope](#standard-edge-envelope) so they exist on day one — the generator only populates them.
</Note>

<Note>
**GOOD-or-null applies to edge descriptions too — now enforced in code.** An edge `description` is a clean factual sentence **or** `null` — never an LLM refusal/meta-commentary echo or placeholder. As of **#13095 v1.3+** the music edge describer enforces this: an output reject-guard maps refusal/meta-commentary to empty (writer #13096 skips empty → the edge stays unstamped and retries), plus a preamble-strip + U+2011/U+2010 normalization. The model was **swapped `gpt-oss-120b` → `deepseek-v4-flash` (2026-06-17)** to remove the residual ~15% dropped-space artifact. The cosmetic gpt-oss artifacts now remain **only on the people-graph edge describers** (#13006/#13011/#13018) — see [edge-audit item 5](/guides/robert-mark/edge-audit#audit-findings). Node-side enforcement: [GOOD-or-null invariant](/guides/dev-updates/2026-06-16-music-node-descriptions#good-or-null-invariant).
</Note>
