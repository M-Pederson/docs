**Notes for downstream agents**

- `outcome_summary` is the human-readable one-liner; `role_spec` is the structured truth.
  Score on `role_spec`, render on `outcome_summary`.
- `must_haves` are filters — a candidate missing any one is excluded. `nice_to_haves` are
  ranking signals only.
- `signal.target_companies` is the strongest single feature for first-pass recall.
- `format_of_results` is one of `suggestion | csv | pdf`. `suggestion` returns enriched
  cards into the copilot UI; `csv`/`pdf` are operator-facing exports for recruiter handoff.
- `exclusion_list.master_person_ids` is populated when the user has already actioned
  candidates ("don't show me anyone I've already passed on").

## Description

This outcome type surfaces individual candidates for open roles. The User is hiring (or hiring on behalf of someone) and wants suggestions of specific people they should consider — typically for a defined role with experience requirements, location preferences, and cultural fit signals.

**Best for:** filling specific open headcount where the User can articulate the role.

**Adjacent outcomes:** [Find a co-founder](/guides/open-work/suggestion-core-concepts/find-cofounder) for equity/founder-level partnerships, [Find a mentor / advisor](/guides/open-work/suggestion-core-concepts/find-mentor-advisor) for board/advisor seats.
