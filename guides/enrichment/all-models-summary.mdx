# All Models Summary

Every OpenRouter model used across all enrichment pipelines, consolidated in one place. Each model links to the pipeline-specific page where its functions are documented in detail.

  **Several Film & TV entries below are deleted as of 2026-04-23** and will be removed from this table on the next doc pass. The IMDB pipeline was replaced by the waterfall architecture described in [IMDB Waterfall Plan](/guides/enrichment/waterfall/imdb-waterfall).

  Still in use: `#10509 add-imdb-to-master-person`, `#10516 tagline-generator`, `#12677 verify-imdb-url`.

  New waterfall functions not yet listed here: `#12873 get-add/imdb-person`, `#12874 get-add/imdb-title`, `#12880 run-base-imdb-person-enrich`, `#12881 run-base-imdb-title-enrich`, `#12879 add-award-records-from-title` (v1.6), `#12887 add-award-records-from-imdb-person` (v1.1h). All use `google/gemini-3.1-flash-lite-preview`. The two awards scrapers (`#12879` \+ `#12887`) now run at `max_tokens: 65536` and `timeout: 300s` (raised from 16384 / 90s on 2026-04-23 to handle prolific entities like Scorsese with ~580 award rows). `#12887`'s prompt schema was further slimmed from 10 to 7 fields in v1.1h so the full 580-row payload fits under the 65k ceiling — 452 rows persisted end-to-end on the Scorsese retest. Per-function live models: [IMDB Waterfall Functions → LLM calls at a glance](/guides/enrichment/waterfall/imdb-functions#llm-calls-at-a-glance). All other new waterfall functions retain the predecessor temp/token/timeout profile.
</Warning>
