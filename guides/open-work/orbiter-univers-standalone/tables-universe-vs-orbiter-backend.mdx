<Tip>
  **Target read model that shapes this page:** Orbiter Backend cards and side panels should read one
  Backend-owned SQL display record whenever possible — `person_display` for people and `company_display`
  for companies. Orbiter Universe owns the enrichment/source records and graph writes, then sends a final
  projection package back when an entity is created or updated. That package carries the display payload
  plus the graph `node_uuid` and the App identity (`master_person_id`, `master_company_id`, or the relevant
  display-table identity), so the product reads stable Backend rows while Universe remains the source of
  enrichment truth.
</Tip>

## Tables ONLY in Orbiter Backend

Backend display tables are product read models, not Universe source tables. Universe manages the source records,
updates the graph, and sends projection packages when entities are created or updated. Orbiter Backend owns these
tables, upserts them from projection packages, links rows back to the Universe graph by `node_uuid`, and reads them
for cards, search rows, side-panel headers, and compact display sections. These are the display tables from the
[Projection Pattern](/guides/open-work/orbiter-univers-standalone/projection-pattern#combined-display-tables).

| Physical App display table | Label(s) | Card type | Use |
| --- | --- | --- | --- |
| `person_display` | `Person`, `Person:Angel` | Person | Cards, search rows, side-panel headers, and compact side-panel summary data. |
| `company_display` | `Company`, `School`, `Company:VC_Firm`, `Organization:Company`, `Company:Music_Label` | Company | Cards, search rows, side-panel headers, and compact side-panel summary data. |
| `work_product_display` | `Project`, `Work:Publication`, `Work`, `Work:Song`, `Work:Symphony` | Work | Card-only by default for projects, publications, and music/composition works. |
| `funding_round_display` | `Funding_Round` | Funding round | Card-only by default for financing events. |
| `live_event_display` | `Event:Live_*`, `Event:Music_*` | Event | Card-only by default for business/live events and music events. |
| `film_tv_display` | `Film_TV:Movie`, `Film_TV:TV_Series`, `Film_TV:TV_Movie`, `Film_TV:TV_Mini_Series`, `Film_TV:Short_Film`, `Film_TV:Documentary`, `Film_TV:TV_Special`, `Film_TV:Video_Game`, `Film_TV:Music_Video` | Film/TV title | Card-only by default for movies, series, documentaries, video games, music videos, and specials. |
| `film_tv_award_display` | `Film_TV_Award:Winner`, `Film_TV_Award:Nominee` | Film/TV award | Card-only award/badge display contract. |
| `music_artist_display` | `Person:Music_Artist` | Music artist | Card-only by default for music artist cards. |
| `music_group_display` | `Music_Group` | Music group | Card-only by default for bands, orchestras, choirs, and other group artists. |
| `recording_display` | `Recording` | Recording | Card-only by default for track/recording cards. |

<Warning>
  **Target split.** *Tables ONLY on Orbiter Universe* have no App copy. Backend display tables and
  product dynamic tables live only in Orbiter Backend. Universe data crosses the boundary as projection
  packages that Backend applies into its own read models — see the
  [projection pattern](/guides/open-work/orbiter-univers-standalone/projection-pattern).
</Warning>
