---
title: 'Create Team Question'

description: 'Create a new team question'
---

## Overview

Create a new team question. Optionally pass `team_question_id` to thread it under a parent question. Supports deep research mode and multi-team targeting.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login or signup.
</ParamField>

## Request Body

<ParamField body="team_id" type="integer" required>
  The ID of the team to create the question for.
</ParamField>

<ParamField body="question" type="string" required>
  The question text.
</ParamField>

<ParamField body="team_question_id" type="integer" required>
  Parent question ID for threading. Pass `0` or `null` for top-level questions.
</ParamField>

<ParamField body="selected_nodes" type="string[]">
  Array of node UUIDs to associate with the question.
</ParamField>

<ParamField body="deep_research" type="boolean">
  Whether to enable deep research mode for this question.
</ParamField>

<ParamField body="team_ids" type="integer[]">
  Array of team IDs to target with this question.
</ParamField>

## Response

<ResponseField name="id" type="integer">
  The ID of the created team question.
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST "https://xh2o-yths-38lt.n7c.xano.io/api:idMy4x2v/team-questions" \
  -H "Content-Type: application/json" \
  -H "X-Data-Source: YOUR_DATA_SOURCE" \
  -H "X-Branch: v1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "team_id": 1,
    "question": "Who knows investors in the healthcare space?",
    "team_question_id": 0,
    "deep_research": false
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": 1
}
```
</ResponseExample>
