The Xano storage layer for the [IMDB Waterfall](/guides/enrichment/waterfall/imdb-waterfall). Queue tables are keyed on the **normalized `imdb_url`**, not on `imdb_person_id` / `film_tv_id`, because most discoveries happen before the underlying row exists.

## Conventions & indexes

The production conventions on these tables — the reference the [music tables](/guides/enrichment/waterfall/music-tables#conventions-mirrored-from-live-imdb-tables) were mirrored from (with music's deliberate deviations documented there):

- **`id`** int, primary key. **`created_at`** timestamp, not null, default `now`.
- **`updated_at`** timestamp, nullable, app-managed — source tables only; omitted on queue tables.
- **Raw payloads** are Xano `json` type, in a field literally named `json`.
- **Flags** are bool, not null, default `false`, with verbed names: `enrich_success`, `processing`.
- **FKs** are int, not null, **default `0` = unset**, with a Xano `tableref` — named for the target table, not generic `entity_id`.
- **Text fields** carry `trim` validators.
- **Queue keys:** `unique(imdb_url)` — uniqueness is enforced at the queue layer only (URLs need normalization first; the music tables also add `unique(mbid)` on source tables because MBIDs are stable).
- **Queue worker pickup index:** composite `btree(processing, priority_tier, cascade_depth)`, plus a `btree` on the FK (`imdb_person_id` / `film_tv_id`).
- **Source-table index:** `btree(created_at desc)` on `imdb_person` / `film_tv` (the convention the music source tables mirror); `film_tv_award` and `imdb_credit_join` carry the same `btree(created_at)`.

## Queue tables (2 — shared shape)

`queue_imdb_person` · `queue_imdb_title`

One row per discovered-but-not-yet-enriched IMDB URL. Written only by the `upsert-imdb-{person,title}` functions (keyed on normalized `imdb_url`: duplicate discovery → `count++` \+ tier upgrade \+ FK backfill); drained only by the [cron processors](/guides/enrichment/waterfall/imdb-crons).

### `queue_imdb_person`

| Field | Type | Notes |
| --- | --- | --- |
| `id` | int | PK |
| `created_at` | timestamp |  |
| `imdb_url` | text (unique, trim) | Normalized canonical URL |
| `name` | text, not null, `""` | Provisional name from discovery context |
| `imdb_person_id` | int, not null, `0` = unset, tableref | Populated once the stub row exists |
| `processing` | bool, not null, `false` | Lock flag — cron sets `true` on pickup |
| `count` | int, not null, `1` | Incremented on duplicate discovery |
| `cascade_depth` | int, not null, `0` | Hops from seed (1 or 2) |
| `priority_tier` | int 1–4, not null, `4` | Tier rule — see [Priority tiers](/guides/enrichment/waterfall/imdb-waterfall#priority-tiers) |
| `source_function` | text, not null, `""` | e.g. `cascade-crew-from-title`, `cascade-awards-from-person` |
| `source_entity_id` | int, not null, `0` | ID of the spawning entity |
| `source_entity_type` | text, not null, `""` | `imdb_person` or `film_tv` |
| `seed_imdb_person_id` | int, not null, `0` = unset, tableref | Root of this cascade (lineage) |
| `seed_film_tv_id` | int, not null, `0` = unset, tableref | Root of this cascade (lineage) |

### `queue_imdb_title`

Identical shape with `imdb_url`, `title` (instead of `name`), and `film_tv_id` (instead of `imdb_person_id`).

## Source table schema additions

The enrichment metadata layered onto the two source-of-truth tables by the waterfall.

### On `imdb_person` and `film_tv`

| Field | Type | Purpose |
| --- | --- | --- |
| `enriched_at` | timestamp nullable | Last full scrape. Used to skip re-scrape within 30 days |
| `cascade_depth` | int nullable | Depth at which this entity was first enriched |
| `enrich_success` | bool default false | Flipped true on successful orchestrator completion |

### On `imdb_person` only (added by #12887 Fix #5 hardening)

| Field | Type | Purpose |
| --- | --- | --- |
| `awards_json` | json nullable | Parsed LLM award array — post-hoc debug \+ idempotent re-process |
| `award_markdown` | text nullable | Raw scrape of `{imdb_url}/awards/` for debug |

### On `film_tv` only (poster subsystem)

| Field | Type | Purpose |
| --- | --- | --- |
| `image_url` | text nullable | The title poster. `run-base-imdb-title-enrich` #12881 canonicalizes the LLM-extracted `poster_url` (`@._V1_` size transform recomputed), uploads it via `save-imbd-poster` #13044, and persists the re-hosted GCS WebP URL (`imdb-posters/<imdb_id>`) — falling back to the canonicalized IMDB URL if the upload fails. There is no separate `poster_url` column; that name exists only inside the extraction JSON. |

## `imdb_credit_join` (#397)

Relational mirror of the `(Person)-[:WORKED_ON]->(Film_TV)` graph edges, written by the credit-edge writers so relational QA can verify person/title joins without querying FalkorDB.

| Field | Type | Notes |
| --- | --- | --- |
| `id` | int | PK |
| `created_at` | timestamp |  |
| `imdb_person_id` | int nullable | Person side (row FK) |
| `person_node_uuid` | text nullable | Person graph UUID |
| `imdb_person_url` | text nullable | Normalized person URL |
| `roles` | text | Credit role text |
| `position` | text nullable | Bill/credit position when known |
| `film_tv_id` | int nullable | Title side (row FK) |
| `film_tv_node_uuid` | text nullable | Title graph UUID |
| `imdb_title_url` | text nullable | Normalized title URL |
| `edge_created` | bool | `false` = pending — the film exists only as a queued stub; the same idempotent writer flips it `true` once the graph edge lands |
| `source` | text nullable | Writer provenance |

## URL normalization

All queue rows and `imdb_person` / `film_tv` rows are keyed on the **normalized** `imdb_url`. Normalization lives in the `get-add` gates: strip query params and the trailing slash.

```text
trim | split:"/?" | first | rtrim:"/"
```
