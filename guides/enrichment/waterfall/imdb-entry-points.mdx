New work enters the IMDB (Film & TV) waterfall through the two **get-add gates** — one per entity type. Each gate normalizes the `imdb_url`, dedups, ensures a stub row, then branches: `cascade_depth == 0` \+ `queue == false` invokes the orchestrator synchronously and returns `action: "enriched_direct"`; anything else upserts a queue row for the [queue crons](/guides/enrichment/waterfall/imdb-crons) to drain. Existing `master_person` rows can also enter through the central link writer or the person pipeline's Phase 8 attach/verify path. Architecture spec: [IMDB Waterfall overview](/guides/enrichment/waterfall/imdb-waterfall) · function roster: [IMDB Waterfall Functions](/guides/enrichment/waterfall/imdb-functions) · queue schemas: [IMDB Waterfall Tables](/guides/enrichment/waterfall/imdb-tables).

## Person-pipeline entry paths

These paths start IMDB enrichment for an existing `master_person` rather than creating a standalone IMDB-seeded person first.

  Central link writer. **v1.5 (2026-07-06):** when a person link is an IMDB name URL, dispatches `mvp/imdb/add-imdb-to-master-person` synchronously with the normalized URL and existing `master_person_id` so sandbox/live data-source context is preserved. The dispatch runs on new links, duplicate person links, and existing global-link returns, so partial earlier runs can be nudged by re-submitting the link. Title URLs do not use this path. **2026-07-07:** the downstream attach path now always creates/links an `imdb_person` stub before scrape; if optional Firecrawl returns no markdown, it upserts `queue_imdb_person` for retry instead of ending at a log-only skip.
