This is the executable build plan for **Scope A** of the copilot orchestration migration.

<Note>
  This page is a **plan only** — execute it phase by phase, verifying each acceptance check before moving on. It covers `cards.ts`, the route, typed data tools over the Robert API, `emitCard`, `prompts.ts`, `toCrayonSSE`, auth forwarding, and persistence: replace the blocking Xano `/chat` call with a streaming Vercel AI SDK (`ai@^6`) agent loop in a TanStack Start route, while Xano stays as data, auth, and tools. <Warning>

The first meeting message returns a synthetic `meeting_goal_card`; selecting a goal sends a `[MEETING_GOAL] ...` prompt that calls `/meeting-prep` directly and returns a `meeting_prep_card`. Likewise the **quick-leverage** dispatch (`"in every possible way"`, lines ~480-540) is a deterministic dispatch, not an LLM turn. The phrase "all 4 modes served by the route" does **not** mean a like-for-like agent-loop migration for these paths — see Phase 8 Step 3 for what "served" means per mode.
</Warning>

### Tools to build

Each typed tool wraps an existing Robert API endpoint (group `Bd_dCiOz`, id 1261). The `execute()` forwards the user's bearer token. These are the data tools for the agent loop — start with the first three behind the flag (Phase 8 rollout step 1).

| Tool (typed, Zod `inputSchema`) | Wraps Robert API endpoint | Method | Notes |
| --- | --- | --- | --- |
| `searchPersons(q, scope)` | `/search` | GET | Network/universe person search. `scope` selects network vs. universe. |
| `getPersonContext(master_person_id)` | `/person-context/{id}` | GET | Enriched person context (currently fetched client-side, passed as `person_context` string). |
| `getConnectionPath(target_master_person_id)` | `/connection-path` | GET | 2nd/3rd-degree path to a target. |
| `getNetwork()` | `/network` | GET | Network summary (currently fetched client-side, passed as `network_data` string). See Phase 3 / Risks for parity. |
| `getMeetingPrep(...)` | `/meeting-prep` | POST | Meeting prep generation (mode `meeting`). May stay behind the client interceptor — see Phase 8 Step 3. |
| `memory(op, ...)` | `/memory`, `/memory/{id}` | GET/POST/PATCH/DELETE | Memory read/write. {/* TODO: verify whether one tool with an `op` discriminator or separate tools is preferred. */} |
| `triggerGeneration(kind, ...)` | `/generate-leverage-loop`, `/generate-outcome`, `/generate-serendipity` | POST | Async generation; pair with `/process-status` polling. |
| `emitCard(card)` | none (local) | n/a | Validates a `CardEnvelope` against `cards.ts` before SSE. The only non-network tool. |

<Tip>
  `emitCard` is not a Robert API wrapper — it is the validation gate. The model calls it (or the route calls it on every model-emitted card) to run the `CardEnvelope` discriminated union through Zod. Reject/retry malformed cards **before** they reach `toCrayonSSE`.
</Tip>

        **File to create:** `/Users/markl.pederson/Documents/Projects/orbiter-frontend/src/features/copilot/server/tools.ts`

        Each tool has a Zod `inputSchema` and an `execute()` that calls the Robert API directly with the forwarded bearer token — **not** via `copilotFetch()` (client-only). The base URL comes from `env.XANO_API_URL` + the hardcoded group id `:Bd_dCiOz` (matches `xano-copilot.ts:13`). **Do not** use `process.env.VITE_API_URL` — `VITE_*` vars are client-only (`import.meta.env`) and resolve to `undefined` server-side, which would make the base `:Bd_dCiOz` (no host).        
        ```ts
        // src/features/copilot/server/tools.ts
        import { tool } from "ai";
        import { z } from "zod";
        import { env } from "@/env";
        import { CardEnvelope } from "../contracts/cards";

        const ROBERT_BASE = `${env.XANO_API_URL}:Bd_dCiOz`; // group id matches xano-copilot.ts:13

        export function buildTools(authHeader: string, dataSource: string, branch: string) {
          const headers = {
            "Content-Type": "application/json",
            Authorization: authHeader,        // forwarded verbatim from the request
            "X-Data-Source": dataSource,      // forwarded from the request
            "X-Branch": branch,               // forwarded from the request
          };

          return {
            searchPersons: tool({
              description: "Search the user's network or the wider universe for people.",
              inputSchema: z.object({ q: z.string(), scope: z.enum(["network", "universe"]).optional() }),
              execute: async ({ q, scope }) => {
                const url = new URL(`${ROBERT_BASE}/search`);
                url.searchParams.set("q", q);
                if (scope) url.searchParams.set("scope", scope);
                const res = await fetch(url, { headers });
                return res.json();
              },
            }),
            getPersonContext: tool({
              description: "Fetch enriched context for a person by master_person_id.",
              inputSchema: z.object({ master_person_id: z.number() }),
              execute: async ({ master_person_id }) => {
                const res = await fetch(`${ROBERT_BASE}/person-context/${master_person_id}`, { headers });
                return res.json();
              },
            }),
            getConnectionPath: tool({
              description: "Find the 2nd/3rd-degree connection path to a target person.",
              inputSchema: z.object({ target_master_person_id: z.number() }),
              execute: async ({ target_master_person_id }) => {
                const url = new URL(`${ROBERT_BASE}/connection-path`);
                url.searchParams.set("target_master_person_id", String(target_master_person_id));
                const res = await fetch(url, { headers });
                return res.json();
              },
            }),
            emitCard: tool({
              description: "Emit a UI card. Validates against the card contract before rendering.",
              // Wrap the union in an object so the tool input is a JSON object — all
              // provider tool-schema serializers expect an object at the top level.
              inputSchema: z.object({ card: CardEnvelope }),
              execute: async ({ card }) => CardEnvelope.parse(card), // throws on malformed
            }),
            // getNetwork, getMeetingPrep, memory, triggerGeneration added in later rollout steps
          };
        }
        ```
