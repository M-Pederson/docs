New work enters the IMDB (Film & TV) waterfall through the two **get-add gates** — one per entity type. Each gate normalizes the `imdb_url`, dedups, ensures a stub row, then branches: `cascade_depth == 0` \+ `queue == false` invokes the orchestrator synchronously and returns `action: "enriched_direct"`; anything else upserts a queue row for the [queue crons](/guides/enrichment/waterfall/imdb-crons) to drain. Two person-pipeline functions can also start IMDB work for an existing `master_person`. Architecture spec: [IMDB Waterfall overview](/guides/enrichment/waterfall/imdb-waterfall) · function roster: [IMDB Waterfall Functions](/guides/enrichment/waterfall/imdb-functions) · queue schemas: [IMDB Waterfall Tables](/guides/enrichment/waterfall/imdb-tables).

## Get-add gates

<Card icon="play">
  ```text
  mvp/imdb/get-add/imdb-person — #12873
  ```

  Gate for any IMDB person URL. **v1.3 (2026-04-22):** normalizes URL, dedups on a 30-day window, ensures stub, then either sync-enriches (depth 0 \+ `queue: false` → invokes `run-base-imdb-person-enrich` directly) or upserts the queue row. Seed-id propagation: when no caller seed is passed, the just-created stub IS the seed. **Primary entry point for new work.**
</Card>

<Card icon="play">
  ```text
  mvp/imdb/get-add/imdb-title — #12874
  ```

  Gate for any IMDB title URL. **v1.3 (2026-04-22):** same shape as the person gate — stub, dedup, sync-call branch, queue upsert, seed-id propagation. **v1.4 (2026-05-27):** pre-API local dedup — drops the 30-day staleness short-circuit in favor of an explicit existing-row check before any scrape. **v1.5 (2026-05-27):** adds a `refresh` bool input (default `false`); when `true`, forces a re-scrape even if the `film_tv` row is fresh. _(The person gate #12873 remains at v1.3 — it did not receive the dedup/refresh upgrade.)_
</Card>

## Person-pipeline entry paths

Both are called from the person pipeline's `process-person-phase-8` (**#12829**) rather than with a raw IMDB URL — they start IMDB enrichment for an existing `master_person`.

<Card icon="play">
  ```text
  mvp/imdb/add-imdb-to-master-person — #10509
  ```

  Attaches IMDB data to an **existing** `master_person_id`. Preserves inline scrape \+ LLM \+ `db.add imdb_person` because the caller owns the `master_person` link. **v1.5 (2026-05-17):** model is `google/gemini-3.1-flash-lite` (the Pro variant from v1.3 was returning empty bodies). **v1.4 (2026-05-17):** added 30-day `enriched_at` staleness check — re-scrapes existing `imdb_person` rows when `enriched_at` is null or \> 30 days old, db.edit-overwriting `json` / `markdown` / `image_url` / `name`. db.add path now sets `enriched_at: now` so future runs can dedup correctly. **v1.2 (2026-04-23):** Section 6 — non-fatal sync call to `#12887` after credits closes the Phase-8 honorary-award asymmetry. Credits loop delegates to `get-add/imdb-title` (queued, T2, depth\+1). Called from `process-person-phase-8` **#12829**.
</Card>

<Card icon="filter">
  ```text
  mvp/imdb/verify-imdb-url — #12677
  ```

  Reverse lookup — given a `master_person_id`, search Tavily \+ disambiguate with LLM, return verified IMDB URL. Called from `process-person-phase-8` **#12829**.
</Card>
