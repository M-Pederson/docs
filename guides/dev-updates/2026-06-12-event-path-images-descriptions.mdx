## Adversarial verification round

Four parallel reviewer agents re-fetched every stored artifact and tried to refute the build (the same pattern that caught 5 bugs in the first session). **8 findings accepted and patched, 3 refuted with evidence:**

Fixed:
- **13102**: `$nowMs = now` → `now|to_int` — the timestamp-object arithmetic trap. The backoff math (`$nowMs + retries × 300000`) would have thrown "Not numeric" on *any* enrich failure, leaving the queue row permanently locked at `processing = true`. This was latent in v1 across all five entity paths (the sandbox drain only exercised the success path) — one fix covers all six.
- **13105**: empty-`node_uuid` guard — if the node MERGE Cypher errors (send-cypher returns error *strings*, not throws), the function now stamps `enrich_success: false` and returns `node_failed` instead of falling through to a success stamp (the "zombie row" hole, which also exists pattern-wide in the v1 orchestrators — flagged for QA).
- **13105**: sub-label sanitizer now title-cases (`Award ceremony` → `Award_Ceremony`, matching the node spec — was producing `Award_ceremony`) and guards leading digits (a digit-leading label is a Cypher syntax error).
- **13105 + 13100**: Place `ON MATCH` now updates `latitude`/`longitude` via `coalesce(r.x, pl.x)` — a venue first seen without coordinates was permanently stuck null; coalesce also prevents a later null from wiping good coords.
- **13108**: each Cover Art Archive call wrapped in `try_catch` — uncertainty about whether `api.request` throws on CAA's routine 404s could have killed the 5-release fallback loop on the first miss.
- **13109**: serper call wrapped per the 12846 pattern; transient 429/5xx now return `serper_error` (row retries next run) instead of counting as a backfill error.
- **13112**: empty-record guard — an artifact with no MB JSON now returns null instead of asking the LLM to describe `{}`.
- **13113**: the unverified native `throw {}` construct replaced with a production-proven throwing `api.lambda` (the #4676 pattern) for `CYPHER_ERROR` surfacing.

Refuted (verifier wrong):
- *"deepseek-v4-flash doesn't exist"* — verifier used stale training data; the model is on the **live** OpenRouter models API (checked this session).
- *"events should queue at cascade_depth 0"* — misread of the depth model; events from a depth-0 seed are depth-1 children by design, otherwise the cascade re-opens unbounded.
- *"8649 missing `history = all`"* — present in the stored script.

Known pattern-wide gaps surfaced but deliberately not patched this session (QA items): `mb_place.node_uuid` is never written back by either 13100 or 13105 (the Place node's UUID isn't extracted from the HELD_AT/RECORDED_AT Cypher); the v1 orchestrators (13091/13094/13099/13100/13101) share the zombie-success hole fixed in 13105.

- Enum edits: Metadata API `PUT /workspace/3/table/{id}/schema` (full read-modify-write of the schema array) is the working route.
- The serializer behaved this session — every create/update was re-fetched and diffed; zero rewrites. `function.run` targets all stored non-empty (callees were always created before callers).
- `13104`/`13107` had a pre-existing bug pattern from their IMDB clone source: the precondition checked the raw input instead of the regex-extracted id — fixed in both.
