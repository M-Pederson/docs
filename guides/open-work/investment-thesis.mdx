## Root Causes Found

### 1. Company investor classification was being lost

FirstMark had `fundable_org_id = 63adbcbe-da14-4d0e-8006-6fd232a42de2`, and the matching `fundable_organizations` row had `is_investor: true`, but the final `master_company` row was ending as `is_vc: false`.

Fixes:

- #12992 v4.9 repairs `is_vc` before Signal NFX/thesis gates.
- #12992 v4.10 reasserts `is_vc` after `update-company-node`.

### 2. Signal NFX wrote the profile, but thesis stayed empty

Signal NFX company scraping worked and #478 existed, but `investment_thesis` (#709) contained an empty row. The company orchestrator saw that row and skipped future thesis builds because it only checked row existence/staleness.

Fix:

- #12992 v4.11 checks `investment_thesis.derived_thesis_summary`; if empty, the row is incomplete and #12978 is called again.

### 3. #12978 had two response-handling failures

The first failure was XanoScript-specific: using `$response` as a local variable name caused Xano to serialize the filter expression incorrectly. That produced the crash:

```text
Unable to locate response:
```

The second failure was payload shape: Cloud Run returned HTTP 200, but the thesis object was nested under:

```json
{
  "response": {
    "result": {
      "data": { "... thesis fields ..." },
      "error": null,
      "meta": {}
    },
    "status": 200
  }
}
```

Fix:

- #12978 v1.4 reads from `response.result.data.*`.
- #12978 now writes `derived_thesis_summary` to #709 and mirrors it to #478 for company investors or #475 for person investors.
