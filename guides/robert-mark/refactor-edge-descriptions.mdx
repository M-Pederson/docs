---
title: "Refactor Edge Descriptions"
description: "Migrate all LLM-generated edge descriptions to OpenRouter with the standard model config."
---

## Goal

Every function that calls an LLM to generate a graph edge `description` property should be updated to use OpenRouter instead of any direct provider call.

## Standard OpenRouter Config

All edge description calls must use this model and provider config:

```json
{
  "model": "openai/gpt-oss-120b",
  "provider": {
    "sort": "throughput",
    "allow_fallbacks": true,
    "data_collection": "deny"
  }
}
```

## Scope

Audit every Xano function that writes a `description` property to a graph edge via LLM and update the model call to use the config above. Known starting points:

- `enrich/write-expertise-edge-description-to-graph` — writes `description` + `description_generated_at` on `HAS_EXPERTISE` edges
- Any other function that generates edge descriptions inline (check for LLM calls adjacent to Cypher `SET r.description`)

## Why

- `gpt-oss-120b` via OpenRouter with `sort: throughput` maximises parallelism during backfills
- `data_collection: deny` keeps edge content out of provider training pipelines
- `allow_fallbacks: true` prevents silent failures when the primary model is rate-limited
