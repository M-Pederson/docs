## What it is

[OpenRouter](https://openrouter.ai) is a unified API gateway that provides access to hundreds of LLM models from OpenAI, Anthropic, Google, Meta, Mistral, and others through a single OpenAI-compatible endpoint. It handles model routing, fallbacks, rate limiting, and billing across providers.

## Why we use it

- **Single API, all models** — one `api.request` call pattern regardless of which model we're using
- **Model flexibility** — swap between providers (Claude, Gemini, DeepSeek) by changing a model string, no code changes
- **Cost tracking** — per-request cost reporting across all models
- **Rate limit handling** — built-in queuing and retries across providers
- **No vendor lock-in** — same request format works for every model

## How we use it at Orbiter

Every LLM call in the enrichment pipelines goes through OpenRouter. Functions in Xano workspace 3 call `https://openrouter.ai/api/v1/chat/completions` with an `Authorization: Bearer $env.openRouter` header. Models used include `anthropic/claude-sonnet-4`, `google/gemini-3-flash-preview`, and others — see the [Model Summary](/guides/enrichment/general/model-summary) pages for the full list.

The `check-models` skill audits which models are live in Xano against OpenRouter pricing and updates the Mintlify docs automatically.
