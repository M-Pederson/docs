Material contract changes since the deleted 2026-07-02 edition, all verified in live XanoScript:

1. **Queue explicit-zero semantics reversed** — `upsert-enrich-company` #12855 v1.2 reads `priority_tier` via `first_notnull:4`: an explicit tier `0`/`1` is now stored, not coerced to 4. Re-upserts apply a count-to-tier escalation (drop 1 tier per 3 re-references, floor 0).
2. **Entry gate returns a structured shell** — every #12558 branch now returns `company_created: true|false` plus a typed miss shell (2026-07-09), and two new pre-create guards (school-input guard, non-company guard) block bad creations.
3. **Deal-fanout gate is depth-only** — the seed-marker chain (`is_vc` / `seeded_by_user` / source string) is gone from #12813; `effectiveCascadeDepth == 0` alone allows fanout, with per-deal protection moved into #12856 v1.9 company-origin containment.
4. **Company_LLM funding fallback subsystem** (#12813 v1.8–v1.19) — LLM-captured funding materializes one provisional `Funding_Round` node + `RAISED` edge + `company_funding_round` row when Fundable has no deals.
5. **Signal NFX URL discovery moved from Brave to Serper** (#2640 rewritten 2026-07-08; `$env.SERPER_DEV`).
6. **Shared LLM edge descriptions** — `generate-edge-description` #13181 + `normalize-edge-description` #13180 now write descriptions on investor, RAISED, HAS_SUBDOMAIN, and work edges, with parser-diagnostic hard-stops.
7. **`node_registry` (table #758, created 2026-07-09)** — graph writers register every created node UUID (column spelled `entitiy_type` — live typo).
8. **School path rebuilt** — the `school` table is retired (#4689 writes disabled 2026-07-04); schools are `master_company` rows with `is_school = true`, full-enrich but skip Fundable/Signal/thesis and force `is_vc = false`.
9. **Reverse music bridge** — orchestrator step 12.5 (`$env.reverse_music_bridge == "on"`, depth 0) detects music-label companies and enters them into the MusicBrainz pipeline (#13175).
10. **Founders as company facts** — #12974 v1.20 emits structured `founders[]`; #12992 v4.21/v4.24 synthesizes founder people into the Exa C-suite loop; #2800 v2.x adds phantom-founder exclusion against Fundable structured `is_founder`.

This spec does not claim parity is proven. Golden fixtures have not been captured or run; the [parity QA smoke tests](/guides/enrichment/waterfall/company-parity-qa-smoke-tests) page is a human checklist, not a harness.

## 2. How to Use This Spec

Use this page as the authoritative build contract for the Go port. The existing Company waterfall pages remain the canonical narrative reference and detailed source catalogs; they are not the implementation checklist — and several are stale against live Xano (section 21).

Implement the shared Go foundations first, then the Company pipeline on top of them. Any proposed change to orchestration order, dedup keys, graph shape, queue semantics, provider request contracts, or historical bug/fix decisions needs Mark approval before implementation.

When in doubt, preserve current live behavior unless this page explicitly marks a behavior as `Fix in Go` or `Xano artifact — do not reimplement`.

## 7. End-to-End Worked Example

### Example: `resend.com` seed company

Input:

```json
{
  "domain": "resend.com",
  "profile_url": "https://www.linkedin.com/company/resend",
  "company_name": "Resend",
  "source": "User Input",
  "cascade_depth": 0,
  "priority_tier": 1,
  "queue": false,
  "hold_downstream": false,
  "return_fast": false
}
```

Expected control flow:

1. Entry gate passes the placeholder blocklist, normalizes the domain, resolves redirects, checks `profile_url` in `master_link`.
2. Kill switch (table row) off; school-input guard not triggered (no school evidence); governor allows create (no depth-leaf source_function).
3. LLM preflight #12974 runs (not `return_fast`): tier-1 `google/gemini-3.1-flash-lite` with `openrouter:web_search`, escalating to `anthropic/claude-sonnet-4.6` on parse/confidence failure. Envelope includes `founders[]` and is stored whole (never only `.data`).
4. Non-company guard passes (real company). Post-LLM dedup checks discovered profiles/domains, plus name/legal-name dedup for fanout callers.
5. Fundable lookup #12698 reads the local mirror; a miss stores the literal JSON string `"no_data"`.
6. Fresh create writes `master_company` (with `source: "User Input"`), `company_enrich_data`, relational fanout via #12989, initial Company graph node via #2602 (CREATE + stamp-back, dual embeddings, summary/description/funding props), and `node_registry` registration (`entitiy_type: "master_company"` — via #2602 and again idempotently via the gate).

## 13. Queue and Worker Contract

### `universe.company_queue` worker

Insert/upsert key: `entity_id` (one logical row per company).

Upsert semantics (must mimic #12855 v1.2 exactly):

```text
incoming_tier  = priority_tier is null ? 4 : priority_tier      // first_notnull — explicit 0/1 survives
incoming_depth = cascade_depth empty  ? 0 : cascade_depth       // first_notempty — 0 is also the fallback
on conflict:
  count       = count + 1
  merged_tier = min(existing_tier, incoming_tier)
  delta_drop  = floor(count/3) - floor((count-1)/3)              // 1 on every 3rd reference, else 0
  priority_tier = max(0, merged_tier - delta_drop)
  cascade_depth = min(existing_depth, incoming_depth)
  source metadata: update only on strict improvement; empty incoming never wipes
```

Priority ordering and drain gate: `priority_tier asc, cascade_depth asc, id asc`, drain restricted to `priority_tier <= 1` (Task #127 contract, 2026-07-03 — investors enter at tier 1, everything else escalates toward it via the count rule).

Locking SQL:

```sql
WITH picked AS (
  SELECT entity_id
  FROM universe.company_queue
  WHERE processing = false AND priority_tier <= 1
  ORDER BY priority_tier ASC, cascade_depth ASC, entity_id ASC
  FOR UPDATE SKIP LOCKED
  LIMIT 1
)
UPDATE universe.company_queue q
SET processing = true, updated_at = now()
FROM picked
WHERE q.entity_id = picked.entity_id
RETURNING q.*;
```

Processing boundary: the lock/update is its own short transaction; orchestration runs outside it; success deletes the row; failure releases it and increments `retry_count`; after configured max retries, move to DLQ (Go addition — Xano has neither and can poison-block).

Cron state is part of the frozen contract: Task #127 ships `active: false` at 600s. The Go worker ships behind an equivalent off-by-default deployment flag until go-live.

Explicit-zero behavior (changed vs the 2026-07-02 edition): `priority_tier: 0` is stored as 0 and is immediately drain-eligible. `cascade_depth: 0` remains 0.

Acceptance checks: duplicate hit increments `count`; third re-reference drops tier by exactly 1 (floor 0, no compounding on later merges); better tier/depth improves the row; source metadata updates only on improvement; a tier-4 row never drains while the tier-gate is 1 (it must escalate first); workers cannot double-process a row.

Writer: #2602 create, #4659 sync. Labels: `:Entity:Company` plus create-time subtype `:School`, `:VC_Firm`, or `:Organization` (School wins over VC).

## 16. Idempotency and Dedup Contract

Company resolve order (live procedural behavior → Universe identifier model):

1. Incoming/profile URL via `master_link` → strong `linkedin` (or platform) identifier hit.
2. Resolved canonical domain → domain identifier (strength pending — section 22).
3. Original pre-redirect domain → same.
4. LLM alt domains vs existing canonical domains.
5. LLM-discovered and Fundable-known profile URLs → strong identifiers.
6. Fundable resolved domain/original domain/known URLs → `fundable` strong id + soft keys.
7. Name/legal-name match (2026-07-09, fanout callers) → soft `name` key: candidate list, resolved by seed-company context — never an automatic merge in Universe.

Universe mechanics: every lookup enters through `entity_identifier`; a strong-key insert collision IS the duplicate-detection event and triggers the merge flow (`merged_into`, identifier repoint with soft-key dedupe, fact repoint both directions, typed-table re-distill, graph node merge). This replaces Xano's procedural existing-row branches; observable behavior (return the canonical row) is unchanged.

Queue idempotency: one logical row per company; upsert per section 13.

Graph idempotency:

- Company create is row/entity-gated (Xano CREATE has a duplicate window — Go closes it with MERGE-on-uuid while preserving the row gate).
- Funding_Round and most edges are MERGE-keyed; RAISED/edge uuids are deterministic (`<companyUuid>-<dealId|roundUuid>`).
- `SPECIALIZES_IN` is a destructive rebuild — fixtures assert the exact post-run edge set, not accumulation.
- HAS_SUBDOMAIN repair is idempotent for matched subdomains (FK-first, no re-resolve).
- Location drain marks addresses processed (`path_triples_added`) even when an endpoint MATCH no-ops — Go should make endpoint absence observable/retryable unless Mark wants exact no-op parity.
- Registry/`graph_node_uuid` writes are write-once (skip on exists).

Provider idempotency gates:

- `company_llm.data` empty gates the LLM stage; the `_parse_fallback` shell counts as non-empty (no auto-retry — flag for fixtures).
- `exa_c_suite` empty gates Exa; a zero-match envelope is non-empty and terminal.
- `yc_data == "no_data"` retries next run; `fundable == "no_data"` is terminal.
- `Signal NFX` 30-day history debounce, failures included.
- Thesis rebuilds on no row, empty derived summary, or 30-day staleness.
- Funding fallback: `company_financial.rounds_added` is terminal across write/duplicate/total-only outcomes; round dedup by normalized type + ±3-day window.
- Deal cascade: `fundable_deals.processed_at`; containment SKIPs also stamp it.
- Music bridge: `mb_label` presence and `music_reverse_probe` status rows (`no_match` caching noted "not yet wired" — treat probe rows as the gate).
- Base-run 60s debounce on history.

## 17. Error Handling and Observability

Structured logs replace `log_crash`/`log_enrichment_company`: company entity id, function/step, severity, QA pass/fail, error, provider status, queue id, run id, fixture correlation id. Preserve the live note taxonomy — `SKIP:` (intentional gate, `qa_passed: true`), `WARN:` (non-fatal), `FALLBACK:` (degraded fill), `SAVED:` (fallback subsystem success), `CRASH:` — it is load-bearing for QA queries. The Xano `log_crash.phase` strict enum is an artifact (it broke the logger — 2026-07-02); Go uses open-vocabulary fields.

Continuations: orchestrator steps continue after isolated failures and mark final success false. Provider sentinel payloads (`"no_data"`, zero-match envelopes, `_parse_fallback` shells) must stay distinguishable from true misses AND from failures. Graph writer failures leave retryable state (no `graph_node_uuid`, no `processed_at`, no `rounds_added`).

Open QA follow-ups from the 2026-07-09 sandbox (waterfall-page warning; both `Fix in Go`, Mark-flagged):

1. Clean optional-source misses land as `enrich_success: false` history rows and read as `PASS_WITH_WARNINGS` — Go should emit terminal SKIP/no-op run rows for optional misses so healthy runs report clean.
2. When a later `Company_LLM` payload resolves to an existing company, additive fields (aliases, alternate domains, profile links, email/phone candidates) stay in payload rows without landing — Go should add conservative additive landing on existing-entity match paths without overwriting settled higher-confidence fields.

Operational improvements allowed (documented as Go upgrades, not parity): queue retry counters + DLQ, stale-processing reapers, strong-key unique constraints with merge handling, provider replay mode, per-step tracing.
