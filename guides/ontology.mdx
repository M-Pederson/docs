The Orbiter knowledge graph is built on [FalkorDB](https://www.falkordb.com/) using the openCypher query language. This section documents every node type and edge relationship in the graph, with real data examples, property definitions, and weight conventions.

## Conventions

- **All nodes carry the `Entity` base label** plus one or more type-specific labels (e.g. `Entity:Person`, `Entity:Company:VC_Firm`)
- **`uuid`** is the primary identifier on every node and edge (UUID v4, generated via `randomUUID()`)
- **`name_embedding`** is a `vecf32` vector embedding on most nodes, used for semantic search
- **Weight convention:** lower number = higher importance (e.g. `FOUNDED` = 5, `WORKS_AT` = 10-40, `VOLUNTEERED_AT` = 38-50)
- **Temporal convention:** currency is derived from `valid_to IS NULL` (no end date = currently active)
- **`display_label`** is a human-readable string stored on nodes for UI rendering
