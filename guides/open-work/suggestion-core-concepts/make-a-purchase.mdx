**Notes for downstream agents**

- `outcome_summary` is the human-readable one-liner; `purchase_spec` is the structured truth.
  Score on `purchase_spec`, render on `outcome_summary`.
- `must_haves` are hard filters — a vendor or provider missing any one is excluded before ranking.
- `deal_breakers` are applied as explicit negative filters; a single match disqualifies the candidate.
- `signal_targets.credential_signals` are the strongest single feature for first-pass recall in
  professional-services purchases — weight above geographic or network signals.
- `switching_context.prior_failures` is populated when the user is switching vendors; use it
  to apply additional negative filters matching the failure pattern.
- `format_of_results: "suggestion"` returns enriched person or vendor cards into the copilot
  UI. Use `csv` when the user wants a shortlist to circulate to their board or leadership team.
- For `sub_category: "real_estate"` purchases, matching targets are licensed professionals
  (brokers, agents) rather than the property itself — the dispatch remains a people-match.

## Description

This outcome type surfaces vendors, sellers, or service providers for purchases the User wants to make for use — not financial return. Could be services (coaches, lawyers, agencies), products (software, high-end goods), or assets (homes, art, equipment). The User wants vetted recommendations, not generic search results.

**Best for:** high-trust purchases where word-of-mouth and reputation matter more than reviews.

**Adjacent outcomes:** [Get advice / consultation](/guides/open-work/suggestion-core-concepts/get-advice) when the User isn't sure what to buy yet, [Find a mentor / advisor](/guides/open-work/suggestion-core-concepts/find-mentor-advisor) when the relationship is ongoing instead of transactional.
