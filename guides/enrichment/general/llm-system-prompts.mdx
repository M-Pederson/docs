## 1. Phone Number Formatter

`mvp/phones/llm-phone-format` — **#2540**

Parses any phone number input (raw digits, formatted, international prefix variations) and returns three normalized fields: `country_calling_code`, `phone` (E.164 format), and `phone_local_display` (country-specific display format). Accepts an optional `country` hint that is used only when the input does not start with `+`. Called from both the person and company enrichment pipelines when normalizing contact phone data. On primary model failure, retries once using the same model after logging the error.

### Model

`google/gemini-3.1-flash-lite-preview` (fallback: `moonshotai/kimi-k2-0905`)

Resolves via `$env.LLM_MODEL_JSON_PARSE`.
