description: 'Accept or reject a collection invitation'
---

## Overview

Allows a user to accept or reject an invitation to join a collection.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login or signup.
</ParamField>

## Path Parameters

<ParamField path="collection_join_id" type="integer" required>
  The ID of the collection join/invitation record.
</ParamField>

## Request Body

<ParamField body="action" type="enum" required>
  The action to take. Must be either `accept` or `reject`.
</ParamField>

## Response

Returns the updated invitation record.

<RequestExample>
```bash cURL
curl -X PUT https://xh2o-yths-38lt.n7c.xano.io/api:U_4ewelb/collection-members/1/invitations \
  -H "Content-Type: application/json" \
  -H "X-Data-Source: YOUR_DATA_SOURCE" \
  -H "X-Branch: v1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "action": "accept"
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": 1,
  "collection_id": 1,
  "status": "accepted"
}
```
</ResponseExample>
