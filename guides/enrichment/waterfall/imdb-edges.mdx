The IMDB (Film & TV) Waterfall uses the canonical graph edge definitions in [Ontology → Edge Types](/guides/ontology/edges). Write-side Cypher for the credit \+ award edges lives on [Cyphers → Film & TV](/guides/cyphers/film-tv); canonical weights are on [Edge Weights → Film / TV & Awards](/guides/ontology/edge-weights#film--tv--awards) (**lower = stronger**; hub-node edges sit Weak or below; never sub-1 — the same shared scale as the [music edges](/guides/enrichment/waterfall/music-edges)).

This page is the **pipeline-scoped catalog**: which edges this pipeline emits, at what weight, from which writer. Edge patterns, properties, and examples should be edited only in the ontology source of truth.

<Note>

## Credit edges

### WORKED\_ON

`(Person)-[:WORKED_ON]->(Film_TV)` · weight **30**. Written for every credit whose film is already in the graph by `add-credit-edges-from-person` (#13013) and `add-credit-edges-from-title` (#13015), plus `#2334` in award context. Since the 2026-07-08 v1.1–v1.5 series, both credit writers stamp `uuid`, full timestamps, **and a description at write time** (via the shared `generate-edge-description` #13181 — LLM with deterministic fallback), and mirror every resolved edge into `imdb_credit_join` (#397). When a person credit resolves only to a queued `film_tv` stub, #13013 writes a pending `imdb_credit_join` row with `edge_created: false`; a later resolved pass writes the graph edge and flips the join to true. See [Cyphers → Film & TV](/guides/cyphers/film-tv).

## Award edges

Award-edge writes run through `add-node-edge-imdb-award` (#2334) — invoked in bounded batches by `tool/backfill-award-edges-imdb` (#13014) since the award scrapers defer graph repair (v1.1p / v1.17, logged as `award_graph_deferred`). #2334 uses deterministic edge descriptions in this hot path (v1.11 removed per-award LLM generation).

### HAS\_AWARD

`(Film_TV)-[:HAS_AWARD]->(Film_TV_Award)` · weight **40**. Written by #2334 (Query A), gated on `hasFilm` (honorary person-only awards have no `Film_TV` anchor). Since v1.7 also fires for title-level award rows with no linked person.

### WON\_AWARD

`(Person)-[:WON_AWARD]->(Film_TV_Award:Winner)` · weight **30**. Written by #2334 (Query B), always fires so person-only awards still connect.

### NOMINATED\_FOR

`(Person)-[:NOMINATED_FOR]->(Film_TV_Award:Nominee)` · weight **35**. Written by #2334 (Query B).

## Expertise edges

### HAS\_EXPERTISE

`(Person)-[:HAS_EXPERTISE]->(SubDomainExpertise)` · weight **10–50** — the locked-spec formula `min(round(10 + match_score × 160), 50)` for matched nodes, fixed **10** for newly-created `SubDomainExpertise` nodes (see [Edge Weights → Expertise](/guides/ontology/edge-weights)). Written by `resolve-person-expertise-v2` (#12926 v2.8 — edge metadata stamped at MERGE time), invoked from Phase 4b via `llm-identify-person-expertise` (#12666). Refreshed idempotently — existing edges are deleted and rewritten per run, with the `person_has_expertise` mirror cleared alongside.

## Edge descriptions

`WORKED_ON` descriptions are stamped **at write time** by #13013 / #13015 via the shared `generate-edge-description` #13181 helper. The #13016 → #13017 → #13018 trio remains the backfill path for edges still missing `description_generated_at`, keyed on `(person_uuid, film_uuid)` (the [music trio](/guides/enrichment/waterfall/music-edges#edge-descriptions) keys on `e.uuid` instead). Model: `deepseek/deepseek-v4-flash` per [Refactor Edge Descriptions](/guides/robert-mark/refactor-edge-descriptions).
