## Overview

Handle a team invitation response by accepting or rejecting it.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login or signup.
</ParamField>

## Path Parameters

<ParamField path="team_member_id" type="integer" required>
  The ID of the team member invitation record.
</ParamField>

## Request Body

<ParamField body="status" type="string" required>
  The invitation response. Must be either `accepted` or `rejected`.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the invitation response was processed successfully.
</ResponseField>

<RequestExample>
```bash cURL
curl -X PUT "https://xh2o-yths-38lt.n7c.xano.io/api:idMy4x2v/team-members/1/invitations" \
  -H "Content-Type: application/json" \
  -H "X-Data-Source: YOUR_DATA_SOURCE" \
  -H "X-Branch: v1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "status": "accepted"
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "success": true
}
```
</ResponseExample>
