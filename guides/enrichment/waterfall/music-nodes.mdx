The Music Industry Waterfall uses the canonical graph node definitions in [Ontology → MusicBrainz Nodes](/guides/ontology/nodes#musicbrainz-nodes). This page is the **pipeline-scoped catalog**: which node types this pipeline writes, keyed how, by which function. Node labels, properties, vector fields, media fields, examples, and reconciliation rules should be edited only in the ontology source of truth.

<Note>
  Ground truth: [MusicBrainz Nodes](/guides/ontology/nodes#musicbrainz-nodes). The waterfall implementation links to those sections rather than redefining node schemas.
</Note>

## Standard node envelope

Canonical section: [Standard Music Node Envelope](/guides/ontology/nodes#music-standard-node-envelope). Every node: base `Entity` label, random `uuid` **plus `mbid` as the natural MERGE key**, `display_label`, `visibility`, `created_at` / `updated_at` via Cypher `timestamp()`, and dual vectors. All six orchestrators also register the node's UUID in the cross-pipeline `node_registry` table (#758) via `register-node-registry-by-mbid` #13184.

### Embeddings per node type

Canonical section: [Music Node Embeddings](/guides/ontology/nodes#music-node-embeddings). All types dual-write `name_embedding` (OpenAI `text-embedding-3-small`, 1536, via #4676) and `embeddings` (Google `gemini-embedding-001`, 1536 — was 3072 until 2026-07-06, via #13114). `Recording`, `Music_Group`, and `Music_Artist` vectorize from their **generated description** once the finalizer (#13117) runs; all other types vectorize `name + disambiguation`.

## People & groups

### Person:Music\_Artist

Canonical section: [Person:Music\_Artist](/guides/ontology/nodes#music-person-artist). A person-type MusicBrainz artist on a shared Orbiter `Person` node. Written by `add-music-artist-node` (#13092) from the artist orchestrator (#13094). Depth-0 seeds enter [person-resolution](/guides/enrichment/waterfall/music-functions#person-resolution) (`SAME_AS` → `master_person`).

### Music\_Group

Canonical section: [Music\_Group](/guides/ontology/nodes#music-group). Group / Orchestra / Choir artists — a band is never a `Person`. Same writer (#13092), split on MusicBrainz `type`.

## Recordings & works

### Recording

Canonical section: [Recording](/guides/ontology/nodes#music-recording). Written inline by the recording orchestrator (#13100).

### Work

Canonical section: [Work](/guides/ontology/nodes#music-work). Written by the work orchestrator (#13101) with a sanitized work-type sub-label (`:Song` etc.).

## Releases & labels

### Release\_Group

Canonical section: [Release\_Group](/guides/ontology/nodes#music-release-group). Written by the release-group orchestrator (#13099).

### Release

Canonical section: [Release](/guides/ontology/nodes#music-release). **Canonical edition only** — #13099 materializes exactly one `Release` per release-group per the [canonical-release rule](/guides/enrichment/waterfall/music-waterfall#resolved-design-decisions-2026-06-11).

### Music\_Label

Canonical section: [Music\_Label](/guides/ontology/nodes#music-label). Company-shaped (`Entity:Company:Music_Label`). Written by `add-music-label-node` (#13090) from the label orchestrator (#13091); depth-0 labels also get the `SAME_AS` → `Company` bridge and [owner resolution](/guides/enrichment/waterfall/music-functions#label--company-bridge--owner-resolution).

## Events & places

### Event

Canonical section: [Music Event](/guides/ontology/nodes#music-event). Shared `Entity:Event` label plus a sanitized `Music_*` sub-label (`:Music_Festival` / `:Music_Concert` / …). Written by the event orchestrator (#13105).

### Place

Canonical section: [Place](/guides/ontology/nodes#music-place). **Materialized inline** (no queue, no orchestrator) from embedded `place-rels` during recording (#13100) and event (#13105) enrich.

### Genres

Canonical section: [Music Genre Policy](/guides/ontology/nodes#music-genre-policy). Properties on the source music node — no `Genre` nodes, no `HAS_GENRE` edges.

## Media (`image_url`)

Canonical section: [Music Media (`image_url`)](/guides/ontology/nodes#music-media-image-url). Cover art (recordings), group images, and person avatars — fetch/save helpers on [Music Waterfall Functions](/guides/enrichment/waterfall/music-functions#cover-art--group-images).
