## mvp/expertise/llm-identify-person-expertise (function #12666)

### Delete Existing Expertise Edges

Clears all HAS_EXPERTISE edges for a person before re-identifying expertise (idempotent refresh).

```cypher
// Version v1
MATCH (p:Entity:Person {uuid: "{{$var.person.node_uuid}}"})-[r:HAS_EXPERTISE]->()
DELETE r
RETURN count(r) AS deleted_count
```

## mvp/expertise/resolve-person-expertise (function #12668)

Takes LLM-identified expertise, vector-searches against existing nodes, matches or creates new nodes, and connects the person via HAS_EXPERTISE edges.

### Vector Search for Matching Expertise

```cypher
// Version v1
CALL db.idx.vector.queryNodes(
  'Entity', 'expertise_embedding', 3, vecf32({{$var.embedding}})
)
YIELD node, score
WITH node, score ORDER BY score ASC LIMIT 1
RETURN node.uuid AS uuid, node.name AS name,
       node.display_label AS label, score
```

### MERGE HAS_EXPERTISE Edge (Matched)

When an existing expertise node matches above the similarity threshold:

```cypher
// Version v3
MATCH (p:Entity:Person {uuid: "{{$input.person_node_uuid}}"})
MATCH (e:Entity {uuid: "{{$var.bestUuid}}"})
MERGE (p)-[r:HAS_EXPERTISE]->(e)
ON CREATE SET
  r.skill = "{{$var.cleanArea}}",
  r.weight = {{$var.edgeWeight}},
  r.confidence = {{$var.item.confidence}},
  r.evidence = "{{$var.cleanEvidence}}",
  r.identified_as = "{{$var.cleanArea}}",
  r.signal_type = "{{$var.item.signal_type}}",
  r.depth = "{{$var.item.depth}}",
  r.recency = "{{$var.item.recency}}",
  r.match_score = {{$var.bestScore}},
  r.match_type = "matched"
ON MATCH SET
  r.skill = r.skill + " | " + "{{$var.cleanArea}}",
  r.weight = CASE WHEN {{$var.edgeWeight}} < r.weight THEN {{$var.edgeWeight}} ELSE r.weight END,
  r.evidence = r.evidence + " | " + "{{$var.cleanEvidence}}",
  r.identified_as = r.identified_as + " | " + "{{$var.cleanArea}}",
  r.signal_type = r.signal_type + " | " + "{{$var.item.signal_type}}",
  r.confidence = CASE WHEN {{$var.item.confidence}} > r.confidence THEN {{$var.item.confidence}} ELSE r.confidence END,
  r.match_score = CASE WHEN {{$var.bestScore}} < r.match_score THEN {{$var.bestScore}} ELSE r.match_score END
RETURN p.uuid AS person_uuid, e.uuid AS expertise_uuid,
       e.name AS expertise_name
```

Weight is computed before the cypher call: `edgeWeight = min(round(10 + (bestScore * 80)), 50)`. ON MATCH keeps the lower (stronger) weight.

### CREATE New SubDomainExpertise Node

When no existing node matches, creates a new SubDomainExpertise node with LLM-generated description:

```cypher
// Version v1
CREATE (s:Entity:SubDomainExpertise {
  uuid: randomUUID(),
  name: "{{$var.cleanDomain}}",
  description: "{{$var.cleanGenericDesc}}",
  display_label: "SubDomainExpertise",
  source: "llm_identified",
  name_embedding: vecf32({{$var.nodeEmbedding}}),
  expertise_embedding: vecf32({{$var.nodeEmbedding}})
})
RETURN s
```

## mvp/node/sync-domain-expertise-nodes (function #12663)

Syncs all `domain_expertise` and `sub_domain_expertise` records to FalkorDB. Only creates nodes for records missing a `node_uuid`. Safe to run repeatedly.

### CREATE DomainExpertise Node

```cypher
// Version v1
CREATE (d:Entity:DomainExpertise {
  uuid: randomUUID(),
  name: "{{$var.cleanName}}",
  description: "{{$var.cleanDesc}}",
  category: "{{$var.category.category}}",
  display_label: "DomainExpertise",
  name_embedding: vecf32({{$var.embedding}}),
  expertise_embedding: vecf32({{$var.embedding}})
})
RETURN d
```

### CREATE SubDomainExpertise Node (Sync)

```cypher
// Version v1
CREATE (s:Entity:SubDomainExpertise {
  uuid: randomUUID(),
  name: "{{$var.cleanSubName}}",
  description: "{{$var.cleanSubDesc}}",
  parent_domain: "{{$var.cleanParentName}}",
  display_label: "SubDomainExpertise",
  name_embedding: vecf32({{$var.subEmbedding}}),
  expertise_embedding: vecf32({{$var.subEmbedding}})
})
RETURN s
```

### CREATE HAS_SUBDOMAIN Edge

Links a SubDomainExpertise node to its parent DomainExpertise node:

```cypher
// Version v1
MATCH (d:Entity:DomainExpertise {uuid: "{{$var.parentDomain.node_uuid}}"})
MATCH (s:Entity:SubDomainExpertise {uuid: "{{$var.subNodeUuid}}"})
CREATE (d)-[:HAS_SUBDOMAIN]->(s)
RETURN d.uuid AS domain_uuid, s.uuid AS sub_uuid
```

## mvp/expertise/resolve-company-specialties (function #12746)

Takes a company's specialties from the `speciality_join` table, embeds each specialty string, vector searches against existing SubDomainExpertise nodes, matches or creates new nodes, and connects the company via SPECIALIZES_IN edges. Called as Section 5.5 of `run-base-company-process` (#12699).

### Delete Existing SPECIALIZES_IN Edges

Clears all SPECIALIZES_IN edges for a company before re-resolving (idempotent refresh).

```cypher
// Version v1
MATCH (c:Entity:Company {uuid: "{{$var.company.node_uuid}}"})-[r:SPECIALIZES_IN]->()
DELETE r
RETURN count(r) AS deleted_count
```

### Vector Search for Matching Expertise

