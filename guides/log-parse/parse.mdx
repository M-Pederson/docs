---
title: "Parse Functions"
description: "Email activity parsing — parse-from, parse-to, parse-cc, and the shared log-participant-activity helper"
---

Called by `message-created-parse` (#1913) to create email activity records based on the user's role in the conversation. All three top-level parse functions (`parse-from`, `parse-to`, `parse-cc`) are thin shells that iterate over the participant arrays and delegate the per-participant identity + logging work to the shared helper **`mvp/activity/log-participant-activity` (#12773)**.

The historical pipeline (`nylas-parse-sent_threads` #2140 and `nylas-parse-received_threads` #2152) calls the same helper, so realtime and historical share one canonical implementation.

---

## log-participant-activity — #12773

```text
mvp/activity/log-participant-activity — #12773
```

The shared per-participant routine. Resolves identity for one email address, optionally logs an `email_activity`, updates `my_person` counters, and triggers signature parsing the first time an address is seen. All callers (parse-from, parse-to, parse-cc, nylas-parse-sent_threads, nylas-parse-received_threads) delegate to this function.

### Input

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email_address` | text | Yes | Participant email (trimmed). Lowercased internally. |
| `name` | text | No | Display name. Caller pre-applies its preferred fallback (e.g. `"Anonymous"` or email-prefix). |
| `user_id` | int | Yes | Owning user. |
| `grant_id` | text | Yes | Nylas grant_id. Used for signature parse. |
| `activity_type_id` | int | Yes | `1` from, `2` to, `3` cc, `4` bcc. |
| `participant_count` | int | No | Total participants in the message. |
| `thread_id` | text | Yes | Nylas thread id. |
| `message_id` | text | Yes | Nylas message id. |
| `subject` | text | No | Email subject. |
| `snippet` | text | No | Email snippet. |
| `activity_timestamp` | int | Yes | Epoch ms. |
| `unread` | bool | No | Unread flag. |
| `temp_email_body` | text | No | Raw body. Stored on `email_activity` so the signature parser can read it on first contact. |
| `data_source_id` | int | Yes | Audit source for the `email_activity` row. `14` realtime, `24` historical. |
| `master_person_data_source_id` | int | No | Audit source for newly-created `master_person`. `47` realtime, `24` historical. Defaults to `data_source_id`. |
| `log_activity` | bool | No | When `false`, identity is still resolved (and brand-new master_persons are still created) but no `email_activity` row is written and `my_person` counters are not bumped. Used by `parse-to` to suppress recipient logging when the sender is brand new. Defaults to `true`. |
| `signature_mode` | text | No | Pass `"skip"` to disable signature parsing. Defaults to enabled. |

### Output

| Field | Type | Description |
|-------|------|-------------|
| `my_person_id` | int / null | Resolved or newly-created my_person id. |
| `master_person_id` | int / null | Resolved or newly-created master_person id. |
| `master_email_id` | int / null | master_email id for the address. |
| `activity_id` | int / null | New email_activity id, if logged. |
| `created_new_email` | bool | `true` only when this call created a brand-new master_email (i.e. first time seeing this address for this user). |

### Behavior

<Steps>
  <Step title="Marketing skip">
    Calls `check-identity-email-to`. If it returns `marketing: true` the function returns immediately with the empty result.
  </Step>
  <Step title="Branch A — existing master_person + my_person">
    Builds the card descriptor for the activity_type, inserts `email_activity`, increments `total_email_activity` on `my_person`, and calls `set-primary-email_v2`. All three are gated on `log_activity`.
  </Step>
  <Step title="Branch B — master_person exists, no my_person yet">
    Calls `copy-to-my-person` (always — keeps the contact list in sync), then logs the activity and bumps counters under the `log_activity` gate.
  </Step>
  <Step title="Branch C — brand new email address">
    Sets `created_new_email: true` on the result, runs `name-format` and `get-add/master-person` (always — even when logging is suppressed, so the person still exists), then under `log_activity` calls `copy-to-my-person`, inserts the activity, bumps counters, and calls `set-primary-email_v2`. After Branch C, fires `email-signature-parse-process` (async-shared) unless `signature_mode == "skip"`. Signature parse only runs in Branch C, so each address is parsed at most once per user — by design.
  </Step>
</Steps>

````text
function "mvp/activity/log-participant-activity" {
  input {
    text email_address filters=trim
    text name?
    int user_id
    text grant_id filters=trim
    int activity_type_id
    int participant_count?
    text thread_id filters=trim
    text message_id filters=trim
    text subject?
    text snippet?
    int activity_timestamp
    bool unread?
    text temp_email_body?
    int data_source_id
    int master_person_data_source_id?
    bool log_activity?
    text signature_mode?
  }

  stack {
    var $shouldLog {
      value = true
    }

    conditional {
      if ($input.log_activity == false) {
        var.update $shouldLog {
          value = false
        }
      }
    }

    var $mpDataSource {
      value = $input.master_person_data_source_id ?: $input.data_source_id
    }

    var $emailLower {
      value = $input.email_address|to_lower
    }

    var $result {
      value = {
        my_person_id     : null
        master_person_id : null
        master_email_id  : null
        activity_id      : null
        created_new_email: false
      }
    }

    function.run "mvp/identity/check-identity-email-to" {
      input = {email_address: $emailLower, user_id: $input.user_id}
    } as $identity

    conditional {
      if (($identity|get:"marketing":null) != true) {

        // ── Branch B: existing master_person, no my_person → copy-to-my-person
        // ── Branch C: brand-new → get-add/master-person, then signature-parse-process

      }
    }
  }
````

## parse-from — #2126

```text
mvp/activity/parse-from — #2126
```

Processes outbound emails (sent FROM the user's account). Iterates `to`/`cc`/`bcc` and delegates each participant to `log-participant-activity` with `activity_type_id` 2/3/4. Uses `data_source_id: 14` (realtime email_activity) and `master_person_data_source_id: 47` (realtime master_person).

<Accordion title="Full XanoScript — parse-from #2126">
````text
// parse & log email when it is sent FROM user's account
function "mvp/activity/parse-from" {
  input {
    json participants?
    int user_id?
  }

  stack {
    var $grantID {
      value = $input.participants|get:"grant_id":null
    }

    var $to {
      value = $input.participants.object|get:"to":[]
    }

    var $cc {
      value = $input.participants.object|get:"cc":[]
    }

    var $bcc {
      value = $input.participants.object|get:"bcc":[]
    }

    var $messageDate {
      value = $input.participants.object.date|multiply:1000
    }

    function.run "mvp/activity/calculate_participant" {
      input = {
        to  : $to
        from: $input.participants.object.from
        cc  : $cc
        bcc : $bcc
      }
    } as $participant_count

    foreach ($to) {
      each as $toItem {
        function.run "mvp/activity/log-participant-activity" {
          input = {
            email_address               : $toItem.email
            name                        : $toItem.name ?: "Anonymous"
            user_id                     : $input.user_id
            grant_id                    : $grantID
            activity_type_id            : 2
            participant_count           : $participant_count
            thread_id                   : $input.participants.object.thread_id
            message_id                  : $input.participants.object.id
            subject                     : $input.participants.object.subject
            snippet                     : $input.participants|get:"object.snippet":null
            activity_timestamp          : $messageDate
            unread                      : $input.participants.object.unread
            temp_email_body             : $input.participants|get:"object.body":null
            data_source_id              : 14
            master_person_data_source_id: 47
            log_activity                : true
          }
        }
      }
    }

    foreach ($cc) {
      each as $ccItem {
        function.run "mvp/activity/log-participant-activity" {
          input = {
            email_address               : $ccItem.email
            name                        : $ccItem.name ?: "Anonymous"
            user_id                     : $input.user_id
            grant_id                    : $grantID
            activity_type_id            : 3
            participant_count           : $participant_count
            thread_id                   : $input.participants.object.thread_id
            message_id                  : $input.participants.object.id
            subject                     : $input.participants.object.subject
            snippet                     : $input.participants|get:"object.snippet":null
            activity_timestamp          : $messageDate
            unread                      : $input.participants.object.unread
            temp_email_body             : $input.participants|get:"object.body":null
            data_source_id              : 14
            master_person_data_source_id: 47
            log_activity                : true
          }
        }
      }
    }

    foreach ($bcc) {
      each as $bccItem {
        function.run "mvp/activity/log-participant-activity" {
          input = {
            email_address               : $bccItem.email
            name                        : $bccItem.name ?: "Anonymous"
            user_id                     : $input.user_id
            grant_id                    : $grantID
            activity_type_id            : 4
            participant_count           : $participant_count
            thread_id                   : $input.participants.object.thread_id
            message_id                  : $input.participants.object.id
            subject                     : $input.participants.object.subject
            snippet                     : $input.participants|get:"object.snippet":null
            activity_timestamp          : $messageDate
            unread                      : $input.participants.object.unread
            temp_email_body             : $input.participants|get:"object.body":null
            data_source_id              : 14
            master_person_data_source_id: 47
            log_activity                : true

        }
      }
````

## parse-to — #2129

```text
mvp/activity/parse-to — #2129
```

Processes inbound emails (sent TO the user's account). Looks up the `nylas_grant` so it can self-skip when iterating recipients. Logs the FROM participant first, then iterates `to`/`cc`/`bcc`.

**`$logActvity` suppression** — when the FROM participant returns `created_new_email: true` from the helper, `$logActvity` is flipped to `false` and passed to every subsequent recipient call. This means recipients still get their `master_person` created (so the contact graph stays accurate), but no `email_activity` rows or `my_person` counter bumps happen. This avoids spamming the activity feed with one-shot first-contact threads.

<Accordion title="Full XanoScript — parse-to #2129">
````text
// parse & log email when it is sent TO user's account
function "mvp/activity/parse-to" {
  input {
    json participants?
    int user_id?
  }

  stack {
    db.get nylas_grant {
      field_name = "grant_id"
      field_value = $input.participants.object.grant_id
      output = ["id", "user_id", "grant_id", "email_account", "ai_enabled"]
      addon = [
        {
          name  : "user:user_id"
          output: ["master_person_id"]
          input : {user_id: $input.user_id}
        }
      ]
    } as $nylasGrant

    var $grantID {
      value = $input.participants|get:"grant_id":null
    }

    var $logActvity {
      value = true
    }

    var $to {
      value = $input.participants.object|get:"to":[]
    }

    var $cc {
      value = $input.participants.object|get:"cc":[]
    }

    var $bcc {
      value = $input.participants.object|get:"bcc":[]
    }

    var $messageDate {
      value = $input.participants.object.date|multiply:1000
    }

    function.run "mvp/activity/calculate_participant" {
      input = {
        to  : $input.participants.object.to
        from: $input.participants.object.from
        cc  : $input.participants.object.cc ?: []
        bcc : $input.participants.object.bcc ?: []
      }
    } as $participant_count

    // ─── FROM (activity_type 1) ───
    var $fromEmail {
      value = $input.participants.object.from|first|get:"email":null
    }

    var $fromName {
      value = $input.participants.object.from|first|get:"name":null
    }

    function.run "mvp/activity/log-participant-activity" {
      input = {
        email_address               : $fromEmail
        name                        : $fromName
        user_id                     : $input.user_id
        grant_id                    : $grantID
        activity_type_id            : 1
        participant_count           : $participant_count
        thread_id                   : $input.participants.object.thread_id
        message_id                  : $input.participants.object.id
        subject                     : $input.participants.object.subject
        snippet                     : $input.participants|get:"object.snippet":null
        activity_timestamp          : $messageDate
        unread                      : $input.participants.object.unread
        temp_email_body             : $input.participants|get:"object.body":null
        data_source_id              : 14
        master_person_data_source_id: 47
        log_activity                : true
      }
    } as $fromResult

    // suppress recipient activity logging when sender is brand new
    conditional {
      if ($fromResult.created_new_email) {
        var.update $logActvity {
          value = false
        }
      }
    }

    // ─── TO / CC / BCC (foreach pattern, self-skip via $nylasGrant.email_account) ───
    // each iteration calls log-participant-activity with log_activity: $logActvity
    // (full xanoscript in Xano #2129)
  }

  response = "success"
  tags = ["message.created webhook"]
}
````
</Accordion>

## parse-cc — #2132

```text
mvp/activity/parse-cc — #2132
```

## Shared sub-functions

| Function | Description |
|----------|-------------|
| `calculate_participant` | Counts total participants across to/from/cc/bcc arrays |
| `check-identity-email-to` | Resolves an email address to `master_person_id`, `my_person_id`, and `master_email_id`. Filters marketing senders and handles plus-addressing via `starts_with` to avoid false-positives. Used by the helper. |
| `copy-to-my-person` | Copies a `master_person` into the user's `my_person` table |
| `name-format` | Parses a full name into first/middle/last/suffix |
| `get-add/master-person` | Gets or creates a `master_person` record |
| `set-primary-email_v2` | Sets the primary email pointer on a `master_person` |
| `email-signature-parse-process` | (async-shared) Parses email signatures for contact enrichment. Fired by the helper only on Branch C (brand-new email). |
