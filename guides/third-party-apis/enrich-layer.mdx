## What it is

[Enrich Layer](https://enrichlayer.com) is a data enrichment API that provides LinkedIn-sourced company and person data. It returns structured profiles including specialties, staff counts, follower counts, descriptions, work history, education, and headquarters addresses.

## Why we use it

- **LinkedIn data** — company and person profiles sourced from LinkedIn
- **Person enrichment** — work history, education, skills, certifications, social links
- **Company specialties** — structured list of company specialties that feed into the expertise taxonomy via SPECIALIZES_IN edges
- **Complementary to PDL** — fills in data gaps that PDL doesn't cover (LinkedIn follower counts, background images, universal name IDs)

## How we use it at Orbiter

Enrich Layer is a key data source in both the **person** and **company** enrichment pipelines.

### Company pipeline

Raw responses are stored in `company_enrich_data.enrich_layer` in Xano, then processed by `run-base-company-process` (Section 2) to extract:

- **Industries and categories** → `industry_join` table
- **Specialties** → `speciality_join` table → resolved to `SPECIALIZES_IN` edges via vector matching
- **Description and tagline** → `company_about` table
- **Staff count and range** → employee count tracking
- **Follower count** → `linkedin_follower` table
- **Headquarters address** → `company_address` table
- **Social links** → LinkedIn, Crunchbase, Facebook, Twitter
- **Founded year, background image** → `master_company` fields

### Person pipeline

Raw responses are stored in `person_enrich_data.enrich_layer` in Xano, then processed by the person enrichment pipeline to extract:

- **Work history** — job titles, companies, dates → work edges (WORKS_AT, FOUNDED, etc.)
- **Education** — schools, degrees, dates → ATTENDED edges
- **Skills** — professional skills list → HAS_EXPERTISE edges via expertise pipeline
- **Certifications** — professional certifications → HAS_CERTIFICATION edges
- **Social links** — LinkedIn, Twitter, personal sites → `person_link` table
- **Location** — city, region, country → LOCATED_IN edges
- **Profile image** — avatar URL for the person record
