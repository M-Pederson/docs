## nylas-calendar-webhook-update-event — #294

Triggered by the Nylas `event.updated` webhook when a calendar event is modified. Updates the `event` record with new title, description, timing, and conferencing details. If the event doesn't exist yet, delegates to `calendar-webhook-parse-event` (#2477) to create it. Before iterating participants the function deduplicates the organizer: it checks whether the organizer email already appears in the participant list and only appends it when absent, preventing duplicate processing. Each participant is resolved via `check-identity-email` (which now also filters marketing emails -- if the `marketing` flag is set the participant is skipped via `continue`). Resolved participants are copied to `my_person` via `copy-to-my-person` (re-enabled for the organizer path as well) and an activity is logged. Removed participants are detected by diffing the old vs. new participant lists; for each removed participant the function now correctly references `$participant` (a plain email string, not `$participant.email`), uses activity type 17, and includes `interlocutor_email` in the activity record.
