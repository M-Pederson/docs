**Notes for downstream agents**

- `outcome_summary` is the human-readable one-liner; `target_spec` is the structured truth.
  Score on `target_spec`, render on `outcome_summary`.
- `must_haves` are hard filters — a company failing any one is excluded from results.
  `nice_to_haves` are ranking signals only.
- `strategic_rationale.rationale_excluded` matters: don't surface acqui-hire or IP-only
  plays when the user wants a product+customer acquisition.
- `ownership_structure.excluded` is a critical filter — PE-owned companies or banker-run
  processes should be excluded before any thesis scoring runs.
- `deal_structure.valuation_ceiling_usd` and `valuation_ceiling_multiple` should be used
  to pre-filter companies whose last-known valuation or implied ARR multiple is above threshold.
- `process_posture.sourcing_mode: quiet_outbound` means suppress any company
  currently listed on a sell-side marketplace or known to be running a banker process.
- `format_of_results` is one of `suggestion | csv | pdf`. `suggestion` returns enriched
  cards into the copilot UI; `csv`/`pdf` are operator-facing exports for board memo or banker handoff.
- `exclusion_list.master_person_ids` is populated when the user has already actioned
  founders ("don't show me anyone I've already reached out to or passed on").

## Description

This outcome type surfaces companies for M&A — either side. The User is acquiring (sourcing targets) or selling (sourcing acquirers). Includes platform acquisitions, tuck-ins, acqui-hires, and exit conversations. Control transactions, not minority investments.

**Best for:** corp-dev teams, PE platform-building, founders quietly exploring exit options.

**Adjacent outcomes:** [Find investment opportunities](/guides/open-work/suggestion-core-concepts/find-investment-opportunities) for minority deals instead of control, [Find investors](/guides/open-work/suggestion-core-concepts/find-investors) when the move is equity raise instead of M&A.
