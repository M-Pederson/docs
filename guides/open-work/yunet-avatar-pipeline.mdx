## Goal

Turn an arbitrary enrichment-sourced image into a square (1:1) headshot avatar, composed on the correct person's face, at standard sizes. Deterministic geometry, commercially-clean licensing, no GPU. Gemini stays in the loop only for identity selection/verification. Fully GCP-native — no Xano.

<Note>
  **Non-goals (v1):** face-alignment rotation, background replacement/segmentation, on-the-fly generation. All deferrable to v2.
</Note>

<Note>
  **Rekognition option:** the default plan below stays YuNet/OpenCV-first. The [AWS Rekognition path](#optional-aws-rekognition-path) is selective only: fallback when YuNet is weak, plus a film/music route where `RecognizeCelebrities` may identify the target directly.
</Note>

## 1. Inputs / Outputs

**Input**

- Source image: URL or bytes (from the Serper fallback resolver, or the primary PDL/Enrich Layer avatar).
- Contact identity: `name`, optional descriptors, optional known-good reference face.
- Routing hints: optional `industry`, `occupation`, `known_for`, and profile/source URLs (IMDb, MusicBrainz, Spotify, Wikipedia/Wikidata, etc.).
- `verification_required: bool` — `false` for trusted single-source avatars (e.g. LinkedIn), `true` for scraped candidates.

**Output**

- Avatar set: `800x800` master + `400x400` + `128x128`, WebP with JPEG fallback, metadata stripped.
- Record written to **AlloyDB** (avatar table): `source_url`, `det_provider`, `det_route`, `fallback_reason`, `face_box`, `det_confidence`, `gemini_verified`, `gemini_confidence`, `compare_similarity`, `celebrity_match`, `celebrity_match_confidence`, `crop_params`, `sizes[]`, `quality_flag`, `safety_flags`.

## 2. Pipeline Flow

1. **Fetch & normalize** — URL to bytes, decode, fix EXIF orientation, flatten transparency/animation.
2. **Route + detect** — default to YuNet (`FaceDetectorYN`) for local boxes + 5 landmarks + scores. Use Rekognition only as a fallback when YuNet is weak, or first-pass for likely film/music people where `RecognizeCelebrities` may identify the target directly.
3. **Select** the target face:
   - 0 faces: reject, fall through to next candidate source, then monogram fallback.
   - 1 face: use it (Gemini-verify only if `verification_required`).
   - More than 1: cheap heuristic first (largest by centrality); escalate to Gemini ("which face is `{name}`") when ambiguous or identity matters.
4. **Verify** (identity-gated) — Gemini Flash confirms the selected face is plausibly the target. Folds into the call you already make.
5. **Compose & crop** — expand box, square it, eye-line placement, clamp, quality gate.
6. **Resize & encode** — multi-size, WebP + JPEG, strip metadata.
7. **Store** — GCS bucket + signed/CDN URL; write metadata to AlloyDB.
8. **Telemetry** — system signals to Cloud Logging/Monitoring/Trace; Gemini calls to Phoenix; feed the avatar audit (see [§6](#6-telemetry-observability)).

## 3. Crop / Composition Spec

Quality lives here, not in detection. Use the **eye midpoint** as the anchor, not the box center — it gives consistent headroom across varied source framings.

```python
import cv2, numpy as np

# YuNet row layout (15 cols):
# [x, y, w, h, x_reye, y_reye, x_leye, y_leye, x_nose, y_nose,
#  x_rmouth, y_rmouth, x_lmouth, y_lmouth, score]

detector = cv2.FaceDetectorYN.create(
    model="face_detection_yunet_2023mar.onnx",  # OpenCV Zoo
    config="", input_size=(320, 320),
    score_threshold=0.7, nms_threshold=0.3, top_k=5000,
)

def detect_faces(img_bgr):
    h, w = img_bgr.shape[:2]
    detector.setInputSize((w, h))
    _, faces = detector.detect(img_bgr)
    return faces if faces is not None else np.empty((0, 15))

def compose_square(img, face, k=2.0, eye_top_frac=0.40):
    """k = face-height multiplier (1.7 tight avatar, 2.4 head+shoulders).
       eye_top_frac = where the eye-line sits from the top (~0.40)."""
    x, y, w, h = face[:4]
    eye_cx = (face[4] + face[6]) / 2.0
    eye_cy = (face[5] + face[7]) / 2.0
    H, W = img.shape[:2]

    S    = min(k * h, W, H)            # square side, capped to image
    left = eye_cx - 0.5 * S            # center horizontally on eyes
    top  = eye_cy - eye_top_frac * S   # eye-line in upper third
    left = max(0, min(left, W - S))    # clamp inside image
    top  = max(0, min(top,  H - S))
    left, top, S = map(lambda v: int(round(v)), (left, top, S))
    return img[top:top+S, left:left+S]
```

<Warning>
  **Quality gate (QA-critical):** if `face_h < 90px`, the source is too low-res for a crisp avatar — flag `quality_flag = "low_res"` and prefer another candidate rather than upscaling a blurry crop. This is the gate that keeps junk out of the avatar audit.
</Warning>

<Tip>
  Spike `k` and `eye_top_frac` against **real** enrichment images before locking them — media/press photography frames faces very differently from headshots.
</Tip>

## 4. Gemini Selection / Verification Contract

One structured call does select + verify + locate, routed through OpenRouter and logged to Phoenix.

**Request:** image + `"The target person is {name} {descriptors}."`

**Response — JSON only**, box normalized 0 to 1000:

```json
{ "match": true, "confidence": 0.0, "face_box": [0, 0, 0, 0], "reason": "" }
```

- **Reconciliation:** convert Gemini's box to pixels, then **IoU-match it against YuNet's boxes** and crop the *YuNet* box. Gemini decides *which* person and *whether* it's them; YuNet supplies the precise geometry. Best of both, no encumbered recognition weights.
- **Skip the call** entirely on single-face trusted sources and on re-crops.
- **Known-good reference face:** if the contact already has a trusted face, AWS Rekognition `CompareFaces` can pre-gate identity by similarity. It does **not** replace Gemini's name/descriptors path when no reference face exists.
- **Likely film/music person:** if enrichment says the person is in film or music, route the candidate through AWS Rekognition `RecognizeCelebrities` first. If the returned celebrity name/URLs match the target aliases, use that face and store the match metadata. If it does not match, continue through YuNet/Gemini as usual.

## 5. Architecture — Full GCP, No Xano

Event-driven and GCP-native end to end. Nothing orchestrates from Xano.

- **Ingest / trigger** — the enrichment step (candidates via the Serper resolver) publishes to **Pub/Sub** (`avatar-requests`) with `{contact_id, candidate_urls, name, verification_required}`.
- **Queue & backpressure** — Pub/Sub subscription with a **dead-letter topic** after N delivery attempts, or **Cloud Tasks** for explicit per-queue rate limiting.
- **Worker** — the avatar service on **Cloud Run** (push subscription delivers the message; or a Cloud Run job pulls for batch). CPU-only, scales to zero, model loaded once per warm instance.
- **Storage** — originals + rendered avatars to **GCS** (lifecycle: keep master, expire/regenerate derivatives); serve via CDN/signed URLs.
- **Source of truth** — avatar record + crop params + confidence + flags to **AlloyDB**. pgvector is right there if you later want to persist the face embedding for dedup/matching.
- **LLM calls** — Gemini via OpenRouter, traced to Phoenix ([§6](#6-telemetry-observability)).

<Warning>
  **Pub/Sub is at-least-once** — the worker **must be idempotent**. Dedupe key `hash(source) + contact_id`; check AlloyDB/GCS before recomputing. A DLQ with bounded retries (or Cloud Tasks rate limits) is also the structural fix for the runaway queue-saturation failure mode.
</Warning>

### CV language decision

YuNet is trivial in Python (`cv2.FaceDetectorYN`), but the platform is consolidating on a Go API server. Three paths:

| Path | Tradeoff |
|------|----------|
| **Python CV microservice** the Go service calls | Fastest to ship; separate deploy |
| **onnxruntime-go** | Run the YuNet ONNX directly; pre/post-processing in Go |
| **GoCV** | Wraps OpenCV `FaceDetectorYN` for a truly in-process Go path (verify the binding in your GoCV version) |

GoCV is cleanest if you want it native; a Python sidecar is cleanest if you want it fast.

### Optional AWS Rekognition path

Use Rekognition as a managed fallback, or as the first detector for likely film/music people. This changes the posture from "Full GCP" to "GCP-orchestrated, AWS-managed vision," so keep it behind explicit routing metadata: `det_provider = "yunet" | "rekognition"` and `det_route = "default" | "fallback" | "film_music"`.

| Rekognition API | Where it fits | Notes |
|-----------------|---------------|-------|
| [`RecognizeCelebrities`](https://docs.aws.amazon.com/rekognition/latest/APIReference/API_RecognizeCelebrities.html) | Rekognition-first route for film/music people | Returns recognized celebrity faces plus unrecognized faces. Use only as a routing/identity assist; not every film/music person will be in the celebrity set. |
| [`DetectFaces`](https://docs.aws.amazon.com/rekognition/latest/APIReference/API_DetectFaces.html) | Fallback face geometry when YuNet is weak | Returns up to the 100 largest faces, including `BoundingBox`, `Confidence`, `Quality`, `Pose`, and `Landmarks` in the default attributes. |
| [`CompareFaces`](https://docs.aws.amazon.com/rekognition/latest/APIReference/API_CompareFaces.html) | Optional identity pre-gate when a trusted reference face exists | Compares one source face to faces in the target image and returns matches above `SimilarityThreshold`; keep human/manual review for rights/privacy-impacting decisions. |
| [`DetectModerationLabels`](https://docs.aws.amazon.com/rekognition/latest/APIReference/API_DetectModerationLabels.html) | Optional safety gate before storage | Replaces or complements the current "optional Gemini safety flag" for rejecting inappropriate source images. |

**Detector router**

1. **Film/music route:** if `industry`, `occupation`, `known_for`, or source URLs indicate film or music context, call `RecognizeCelebrities` first.
2. If `RecognizeCelebrities` returns a celebrity whose `Name` or returned URLs match the target aliases, use the Rekognition celebrity face geometry and skip generic face selection unless Gemini verification is still required by policy.
3. If the celebrity API returns no target match, fall back to the normal YuNet-first path.
4. **Default route:** run YuNet locally. If YuNet returns a usable single target face, stay local.
5. **Fallback route:** call Rekognition `DetectFaces` when YuNet returns 0 faces, low confidence, bad/missing eye landmarks, an edge-clipped box, or ambiguous multi-face geometry.
6. If a trusted reference face exists, `CompareFaces` can run after either route as an additional identity pre-gate.

<Note>
  Treat film/music industry as a **routing hint**, not proof that Rekognition will know the person. Many working artists, producers, operators, and executives will not be in the celebrity index.
</Note>

**Flow**

1. Cloud Run fetches the candidate URL, normalizes EXIF/animation/transparency, and re-encodes a JPEG/PNG byte buffer.
2. Choose the detector route:
   - `film_music`: call `RecognizeCelebrities` first for likely film/music targets.
   - `default`: call YuNet locally.
   - `fallback`: call Rekognition `DetectFaces` only when YuNet cannot produce a trustworthy crop.
3. Convert Rekognition's normalized `BoundingBox` ratios into the same pixel crop contract used by YuNet. Clamp the box: Rekognition can return negative or >1 coordinates for partial edge faces.
4. Run the existing select/compose/quality path. Use Rekognition `Quality.Sharpness`, `Quality.Brightness`, `Pose`, and `FaceOccluded` as additional quality flags, but keep the `face_h < 90px` crispness gate.
5. If `known_good_reference_face` exists, call `CompareFaces` before Gemini. Store `compare_similarity`; accept only above the tuned threshold. If no reference exists and celebrity recognition did not identify the target, keep the Gemini select/verify contract.
6. Optionally call `DetectModerationLabels` with a tuned `MinConfidence` before writing to GCS.
7. Persist provider-specific metadata in AlloyDB: `det_provider`, `det_route`, `fallback_reason`, `det_confidence`, `rekognition_quality`, `rekognition_pose`, `face_occluded`, `compare_similarity`, `celebrity_match`, `celebrity_match_confidence`, `moderation_labels`.

```python
import boto3

rekognition = boto3.client("rekognition", region_name="us-east-1")

def clamp(value, low, high):
    return max(low, min(high, value))

def landmark_px(face, kind, image_w, image_h):
    for point in face.get("Landmarks", []):
        if point.get("Type") == kind:
            return [point["X"] * image_w, point["Y"] * image_h]
    return None

def to_crop_contract(face, image_w, image_h):
    box = face["BoundingBox"]
    x1 = clamp(box["Left"] * image_w, 0, image_w)
    y1 = clamp(box["Top"] * image_h, 0, image_h)
    x2 = clamp((box["Left"] + box["Width"]) * image_w, 0, image_w)
    y2 = clamp((box["Top"] + box["Height"]) * image_h, 0, image_h)

    return {
        "provider": "rekognition",
        "box": [round(x1), round(y1), round(x2 - x1), round(y2 - y1)],
        "left_eye": landmark_px(face, "eyeLeft", image_w, image_h),
        "right_eye": landmark_px(face, "eyeRight", image_w, image_h),
        "confidence": face["Confidence"],
        "quality": face.get("Quality", {}),
        "pose": face.get("Pose", {}),
        "face_occluded": face.get("FaceOccluded", {}),
    }

def detect_faces_rekognition(jpeg_bytes, image_w, image_h):
    response = rekognition.detect_faces(
        Image={"Bytes": jpeg_bytes},
        Attributes=["DEFAULT", "FACE_OCCLUDED"],
    )
    return [
        to_crop_contract(face, image_w, image_h)
        for face in response.get("FaceDetails", [])
    ]

def normalize_name(value):
    return "".join(ch.lower() for ch in value if ch.isalnum())

def recognize_target_celebrity(jpeg_bytes, image_w, image_h, target_aliases):
    aliases = {normalize_name(alias) for alias in target_aliases}
    response = rekognition.recognize_celebrities(Image={"Bytes": jpeg_bytes})

    for celebrity in response.get("CelebrityFaces", []):
        if normalize_name(celebrity.get("Name", "")) not in aliases:
            continue

        contract = to_crop_contract(celebrity["Face"], image_w, image_h)
        contract.update({
            "route": "film_music",
            "celebrity_match": celebrity.get("Name"),
            "celebrity_match_confidence": celebrity.get("MatchConfidence"),
            "celebrity_id": celebrity.get("Id"),
            "celebrity_urls": celebrity.get("Urls", []),
        })
        return contract

    return None

def compare_to_reference(reference_jpeg_bytes, candidate_jpeg_bytes):
    response = rekognition.compare_faces(
        SourceImage={"Bytes": reference_jpeg_bytes},
        TargetImage={"Bytes": candidate_jpeg_bytes},
        SimilarityThreshold=85,
        QualityFilter="AUTO",
    )
    matches = response.get("FaceMatches", [])
    return matches[0] if matches else None
```

**Auth / infra**

- Prefer GCP Cloud Run service account → AWS IAM role via Workload Identity Federation / STS. For a spike, a short-lived AWS key in Secret Manager is acceptable, but do not ship static long-lived credentials.
- IAM actions can stay narrow: `rekognition:RecognizeCelebrities`, `rekognition:DetectFaces`, `rekognition:CompareFaces`, `rekognition:DetectModerationLabels`.
- Keep GCS + AlloyDB as source of truth. Only add S3 if we later want AWS-side image caching; passing bytes avoids cross-cloud object replication.
- Add retries/backoff for Rekognition throttling and throughput errors; isolate AWS latency from the Pub/Sub ack deadline with bounded per-image timeouts.

<Warning>
  Rekognition is attractive for speed-to-build, but it is not "free" architecturally: cross-cloud egress, AWS IAM, data-processing review, service quotas, and per-image API cost become part of the production surface. Choose it deliberately, not accidentally.
</Warning>

## 6. Telemetry & Observability

Telemetry is the data the pipeline emits about its own behavior so you can see it run in production. Three pillars — logs, metrics, traces — instrumented once via **OpenTelemetry**, exported to GCP, with Phoenix as the lens on the LLM layer.

**What to track**

- **Detection hit-rate** — share of sources yielding a usable face. A continuous read on the Phase 0 risk; a drop signals a source-mix shift or model regression.
- **Detection route split** — `default`, `fallback`, and `film_music`, with hit-rate/latency/quality broken out by provider and route.
- **Escalation rate** — share of images sent to Gemini. The LLM cost driver.
- **Verification** — Gemini match rate, confidence histogram, manual-override rate (proxy for false accept/reject).
- **Quality** — share flagged `low_res`, share falling back to monogram.
- **Rekognition signals** *(if enabled)* — face count, `Quality.Sharpness`, `Quality.Brightness`, `FaceOccluded`, `compare_similarity`, `moderation_blocked`.
- **Latency** — per-stage p50/p95 (fetch / detect / gemini / crop / store).
- **Throughput / queue** — Pub/Sub backlog, DLQ count, processing rate.
- **Cost** — Gemini calls + Rekognition image operations per day times price.
- **Errors** — fetch / decode / timeout counts by type.

**Stack (full GCP)**

| Layer | Tool | Role |
|-------|------|------|
| Logs | **Cloud Logging** | Structured JSON, auto-captured from Cloud Run stdout — one line per processed image |
| Metrics | **Cloud Monitoring** | Custom metrics via OTel / Managed Prometheus; dashboards; alert policies (e.g. `hit-rate < X`, `DLQ > 0`, `p95 > Y`) |
| Traces | **Cloud Trace** | Spans across Pub/Sub to Cloud Run to stages |
| Exceptions | **Error Reporting** | Aggregated stack traces |
| LLM | **Arize Phoenix** | Prompt + image + response + tokens per Gemini call, plus evals on whether selection was correct |

Phoenix's OpenInference spans are OTel-compatible, so it sits **alongside** Cloud Trace, not instead of it.

<Note>
  **Split:** system/infra to Cloud Monitoring/Logging/Trace; LLM-decision quality to Phoenix.
</Note>

## 8. Edge Cases

- **No face** — next candidate source, then a generated monogram avatar.
- **Multi-face, Gemini uncertain** — store best-effort, flag for avatar audit, never block.
- **Tiny/low-res face** — reject for crispness (quality gate), prefer alternate source.
- **Profile / sunglasses / occlusion** — YuNet may still box it; if landmarks look unreliable, fall back to box-center vertical placement; Gemini guards identity.
- **EXIF rotation / animated / transparent** — normalize before detection.
- **Inappropriate source image** — optional Gemini safety flag or Rekognition `DetectModerationLabels` before use as an avatar.
- **Rekognition partial edge boxes** — clamp normalized boxes before composing; do not assume `Left`/`Top` always sit between 0 and 1.
- **At-least-once redelivery** — idempotent worker (key `hash(source) + contact_id`); cache crops so re-runs are free.

## 9. Phased Delivery

| Phase | Scope | Est. |
|-------|-------|------|
| **0 — Spike** | YuNet detect + naive crop on a sample of **real** enrichment images (Python is fine here regardless of final language). Validate hit-rate on the actual distribution; tune `k` / `eye_top_frac`. De-risks the whole build. | ~0.5d |
| **0R — Rekognition routing spike** *(optional)* | On the same sample set, run Rekognition only for YuNet failures/weak detections and film/music targets. Measure fallback recovery rate, celebrity match precision, false accepts/rejects, latency, cost, and privacy/IAM burden. | ~0.5–1d |
| **1 — Crop module** | Production compose (eye-based), multi-size output, EXIF/format handling, quality gate. Single-face path end-to-end. Pick the CV language. | ~1–2d |
| **2 — Gemini integration** | Selection + verification + IoU reconciliation; multi-face handling; confidence/flagging. | ~1–2d |
| **3 — Service + GCP wiring** | Cloud Run worker, Pub/Sub trigger + DLQ (or Cloud Tasks), GCS storage, AlloyDB metadata write-back, idempotency. | ~1–1.5d |
| **4 — Telemetry** | OTel instrumentation to Cloud Monitoring/Logging/Trace + Error Reporting; Phoenix on Gemini calls; dashboards + alert policies; wire into the avatar audit. | ~0.5–1d |

**MVP (Phases 0–1, single-face):** ~2–3 days (human). **Polished v1:** ~1–1.5 eng weeks (human). See Developer Notes for AI-agent build estimates.

## 10. Open Decisions

- **Rekognition routing thresholds** — exact YuNet confidence/landmark/edge-clip triggers for fallback, plus the film/music classification rules that call `RecognizeCelebrities`.
- **CV language** — Python microservice vs GoCV vs onnxruntime-go for the primary YuNet path. Decide before Phase 3; Python is fine for the Phase 0 spike either way.
- **Delivery model** — Pub/Sub push subscription (low latency) vs Cloud Run job pulling batches (cheaper at volume).
- **Embedding persistence** — store the face embedding in AlloyDB/pgvector for future dedup/matching, or skip for v1.
- `k` and `eye_top_frac` defaults — settle empirically in Phase 0.
- Multi-face heuristic threshold before escalating to Gemini (cost vs. accuracy).
- Monogram fallback styling — owned by design or generated?
