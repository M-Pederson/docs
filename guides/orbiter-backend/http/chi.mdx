## Overview

The Orbiter Go backend uses [**Chi**](https://github.com/go-chi/chi) as its HTTP router. Chi is a lightweight, composable router built entirely on Go's standard `net/http` package — no custom request wrappers, no framework-specific interfaces.

---

## Why Chi

| Criteria | Chi |
|---|---|
| **`net/http` compatibility** | 100% — handlers are `http.HandlerFunc`, middleware is `func(http.Handler) http.Handler` |
| **Middleware composition** | `r.Use(...)` for global, `r.With(...)` for per-route, `r.Group(...)` for scoped groups |
| **URL parameters** | `chi.URLParam(r, "id")` — no struct binding, just strings |
| **Sub-routers** | `r.Mount(...)` and `r.Route(...)` for modular route registration |
| **Framework lock-in** | None — handlers work with any `net/http`-compatible library |
| **Dependencies** | Zero external dependencies |

### Why not Gin or Fiber

- **Gin** wraps `http.Request` in a `gin.Context`, which couples handler signatures to the framework. Chi avoids this.
- **Fiber** is built on `fasthttp` rather than `net/http`, which means stdlib middleware, `httptest`, and many third-party libraries are incompatible without adapters.

Chi gives the benefits of a router (parameterised routes, middleware chaining, route grouping) without any of the trade-offs of a full framework.

## Route Registration

Routes are registered using Chi's grouping and sub-router patterns:

```go
package main

import (
    "net/http"

    "github.com/go-chi/chi/v5"
    "github.com/go-chi/chi/v5/middleware"
    "github.com/go-chi/cors"
)

func main() {
    r := chi.NewRouter()

    // Global middleware
    r.Use(middleware.Logger)
    r.Use(middleware.Recoverer)
    r.Use(middleware.RequestID)
    r.Use(cors.Handler(cors.Options{
        AllowedOrigins: []string{"https://app.orbiter.io"},
        AllowedMethods: []string{"GET", "POST", "PUT", "DELETE", "OPTIONS"},
        AllowedHeaders: []string{"Authorization", "Content-Type"},
    }))

    // Public routes
    r.Get("/health", healthCheck)

    // Authenticated API routes
    r.Route("/api/v1", func(r chi.Router) {
        r.Use(authMiddleware)

        r.Route("/persons", func(r chi.Router) {
            r.Get("/", listPersons)
            r.Post("/", createPerson)
            r.Get("/{id}", getPerson)
            r.Put("/{id}", updatePerson)
            r.Delete("/{id}", deletePerson)
        })

        r.Route("/companies", func(r chi.Router) {
            r.Get("/", listCompanies)
            r.Get("/{id}", getCompany)
        })
    })

    http.ListenAndServe(":8080", r)
}
```

## Middleware

Chi middleware follows the standard `net/http` middleware signature:

```go
func(next http.Handler) http.Handler
```

### Built-in middleware used

| Middleware | Purpose |
|---|---|
| `middleware.Logger` | Structured request/response logging |
| `middleware.Recoverer` | Panic recovery — returns 500 instead of crashing |
| `middleware.RequestID` | Attaches a unique request ID to each request context |
| `middleware.RealIP` | Extracts the real client IP from proxy headers |
| `middleware.Timeout` | Enforces a maximum request duration |

### Custom middleware example

```go
func authMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        token := r.Header.Get("Authorization")
        if token == "" {
            http.Error(w, "unauthorized", http.StatusUnauthorized)
            return
        }

        // Validate token, set user in context...
        ctx := context.WithValue(r.Context(), userContextKey, user)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}
```

### Scoped middleware

Apply middleware to specific route groups using `r.With(...)`:

```go
r.Route("/api/v1", func(r chi.Router) {
    // Admin-only routes
    r.With(adminOnlyMiddleware).Route("/admin", func(r chi.Router) {
        r.Get("/users", listAllUsers)
    })

    // Standard authenticated routes
    r.Route("/persons", func(r chi.Router) {
        r.Get("/", listPersons)
    })
})
```

## URL Parameters

Chi uses `chi.URLParam` to extract route parameters:

```go
func getPerson(w http.ResponseWriter, r *http.Request) {
    id := chi.URLParam(r, "id")
    // ...
}
```

## Mounting Sub-Routers

For modular codebases, each domain package can export its own router and be mounted at a prefix:

```go
// internal/handlers/persons/routes.go
func Routes() chi.Router {
    r := chi.NewRouter()
    r.Get("/", listPersons)
    r.Post("/", createPerson)
    r.Get("/{id}", getPerson)
    return r
}

// cmd/server/main.go
r.Mount("/api/v1/persons", persons.Routes())
```

## Testing

Because Chi handlers are standard `http.HandlerFunc`, they can be tested with `net/http/httptest` without any framework-specific test utilities:

```go
func TestGetPerson(t *testing.T) {
    r := chi.NewRouter()
    r.Get("/{id}", getPerson)

    req := httptest.NewRequest("GET", "/123", nil)
    w := httptest.NewRecorder()

    r.ServeHTTP(w, req)

    if w.Code != http.StatusOK {
        t.Errorf("expected 200, got %d", w.Code)
    }
}
```
