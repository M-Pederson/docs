The IMDB (Film & TV) Waterfall uses the canonical graph node definitions in [Ontology ŌåÆ Node Types](/guides/ontology/nodes). Titles and awards get dedicated Film & TV node types; people reuse the shared `Person` node, extended with IMDB roles, biography, and avatar.

This page is the **pipeline-scoped catalog**: which node types this pipeline writes, keyed how, by which function. Node labels, properties, vector fields, and examples should be edited only in the ontology source of truth.

<Note>
  Ground truth: [Node Types](/guides/ontology/nodes). The waterfall implementation should link to those sections rather than redefining node schemas. Write-side Cypher for the Film & TV nodes lives on [Cyphers ŌåÆ Film & TV](/guides/cyphers/film-tv).
</Note>

## Titles

### Film\_TV

Canonical section: [Film\_TV](/guides/ontology/nodes#film_tv). The title node ŌĆö `:Entity:Film_TV` plus a dynamic subtype label (`:Movie`, `:TV_Series`, `:TV_Movie`, `:TV_Mini_Series`, `:Short_Film`, `:Documentary`, `:TV_Special`, `:Video_Game`, `:Music_Video`). MERGE-keyed on `imdb_url` by [`add-film-tv-node` #2320](/guides/cyphers/film-tv) ŌĆö and by `add-node-edge-imdb-award` #2334 in award context, which since v1.10 writes the graph uuid back to `film_tv.node_uuid` and gives not-yet-enriched award-created titles a deterministic description fallback. Carries a `name_embedding` semantic vector (guarded splice ŌĆö a null/timeout embedding is skipped rather than corrupting the Cypher). Both writers register stamped UUIDs in `node_registry` (2026-07-09, `entitiy_type: "film_tv"` ŌĆö the live column name carries that typo).

## Awards

### Film\_TV\_Award

Canonical section: [Film\_TV\_Award](/guides/ontology/nodes#film_tv_award). The award node ŌĆö `:Entity:Film_TV_Award` plus a `:Winner` / `:Nominee` subtype label. MERGE-keyed on `(year, award_type, award_description, film_tv_imdb_url)`, or on `person_uuid` for person-only honoraries. Written by [`add-node-edge-imdb-award` #2334](/guides/cyphers/film-tv) ŌĆö since v1.7 also for title-level award rows with no linked person, and since v1.8 stamping display metadata (`name`, `title`, `display_label`, `source`, `updated_at`) on create and match. Carries a `name_embedding` semantic vector. UUIDs registered in `node_registry` (v1.12).

## People

### Person

Canonical section: [Person](/guides/ontology/nodes#person). IMDB does **not** create a person type ŌĆö it enriches the **shared** `Person` node with IMDB roles, short/long biography, and avatar, and stamps `visibility = true` in Phase 4a. The `imdb_person.master_person_id` \+ `node_uuid` link the Xano row to the graph node (`add-person-node` #2601 / `update-person-node` #2758 ŌĆö both dual-write `name_embedding` (OpenAI) **and** `embeddings` (Gemini ŌĆö 1536-dim since 2026-07-06) since 2026-06-15).

## Expertise

### SubDomainExpertise

Canonical section: [SubDomainExpertise](/guides/ontology/nodes#subdomainexpertise). Phase 4b runs `llm-identify-person-expertise` (#12666 v2) ŌåÆ `resolve-person-expertise-v2` (#12926), which MERGEs `SubDomainExpertise` nodes and connects them to the `Person` via `HAS_EXPERTISE` edges ŌĆö see [IMDB Edges](/guides/enrichment/waterfall/imdb-edges).

## Timestamp coverage

Every node create/update reachable from the IMDB waterfall stamps `created_at` on create and `updated_at` on update. The per-node-type record is audited in [Node Types ŌåÆ Timestamp coverage audit](/guides/ontology/nodes#timestamp-coverage-audit).
