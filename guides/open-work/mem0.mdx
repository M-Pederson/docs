### Checked near-matches

These endpoints look memory-related, but they use Xano tables instead of Mem0.

#### POST `memory` (#8223)

Step-by-step behavior:

1. Accepts `key`, `value`, optional `category`, `scope`, `team_id`, and `source`.
2. Queries `copilot_memory` for an active row for the authenticated user with the same `key` and `scope`.
3. If an active memory exists, updates that row's `value`, `category`, and `source`.
4. If no active memory exists, creates a new `copilot_memory` row.
5. Writes a create/update entry into `copilot_memory_log`.
6. Returns `{ memory, action }`.

Mem0 status: **no external Mem0 call**; this is local Xano memory storage.

#### GET `memory` (#8222)

Step-by-step behavior:

1. Accepts optional `scope`, `category`, and `team_id` inputs.
2. Queries active `copilot_memory` rows for the authenticated user.
3. Returns `{ memories, total }`.

Mem0 status: **no external Mem0 call**; this reads local Xano memory rows.

#### PATCH `memory/{memory_id}` (#8229)

Step-by-step behavior:

1. Loads the requested `copilot_memory` row by `memory_id`.
2. Confirms the row exists.
3. Confirms the row belongs to the authenticated user.
4. Updates `value`, `active`, `category`, and `confidence`.
5. Writes an update entry to `copilot_memory_log`.
6. Returns `{ memory }`.

Mem0 status: **no external Mem0 call**; this edits a local Xano memory row.

#### DELETE `memory/{memory_id}` (#8230)

Step-by-step behavior:

1. Loads the requested `copilot_memory` row by `memory_id`.
2. Confirms the row exists.
3. Confirms the row belongs to the authenticated user.
4. Soft-deletes the row by setting `active: false`.
5. Writes a delete entry to `copilot_memory_log`.
6. Returns `{ success: true }`.

Mem0 status: **no external Mem0 call**; this soft-deletes local Xano memory.

#### POST `chat` (#8064)

Step-by-step behavior:

1. Loads the selected copilot mode config by `input.mode`.
2. Builds the base system prompt from `copilot_config.prompt`.
3. Runs `mvp/copilot/build-memory-context` (#12665).
4. Appends the returned `[USER MEMORY]` block to the system prompt when memory exists.
5. Appends active person context and network data when provided.
6. Builds the OpenRouter chat-completions message array.
7. Calls `https://openrouter.ai/api/v1/chat/completions`.
8. Parses the LLM response into the frontend response shape.

Mem0 status: **no external Mem0 call**. The memory function reads `copilot_memory`; it does not call Mem0.
