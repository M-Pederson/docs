## mvp/funding/add-fundable-deal-node — #12701

```text
mvp/funding/add-fundable-deal-node — #12701
```

<Note>
  **v1.9 (2026-07-02) — write-safety, applied above the templates shown.** #12701 now **JSON-encodes/double-quotes every Cypher string property** before it reaches these Twig templates, so an apostrophe (`Deel's`, `company's`) no longer terminates a string literal and silently kills the node CREATE — the proven cause of 2 of 6 Deel `Funding_Round`s failing while #12701 still returned `"success"`. The `name_embedding` splice is guarded (an unavailable vector omits the property → vector-less node, never a crash), a missing `node_uuid` in the response logs a `log_crash` and leaves `funding_round_node_uuid` falsy (the signal #12856 v1.8's `processed_at` gate reads), and the RAISED `created_at` is **floored to true UTC midnight** (no time-of-day leak). The single-quoted `'{{$var.…}}'` interpolations rendered below reflect the pre-v1.9 template shape; the live v1.9 body wraps each string via `JSON.stringify`. **Xano artifact — do not reimplement:** parameterize the query in a Go/AlloyDB port.
</Note>

### MERGE Company RAISED Edge

```cypher
// Version v2 — JSON-encoded (quote-safe) string props
MATCH (c:Company {uuid: '{{$var.organization.master_company_node_uuid}}'})
MATCH (fr:Funding_Round {uuid: '{{$var.dealId}}'})
MERGE (c)-[raised:RAISED]->(fr)
SET raised.created_at = coalesce(raised.created_at, {% if $var.announcedMillis is not empty %}{{$var.announcedMillis}}{% else %}timestamp(){% endif %}),
    raised.updated_at = timestamp(),
    raised.uuid = coalesce(raised.uuid, '{{$var.organization.master_company_node_uuid}}-{{$var.dealId}}'),
    raised.weight = 10
  {% if $var.cyAnnouncedDate is not null %},
    raised.announced_date = {{$var.cyAnnouncedDate}}{% endif %}
  {% if $var.cyCompanyName is not null %},
    raised.company_name = {{$var.cyCompanyName}}{% endif %}
  {% if $var.cyShortDescription is not null %},
    raised.description = {{$var.cyShortDescription}}{% endif %}
RETURN c.name AS company, fr.uuid AS node_uuid
```

## mvp/fundable/resolve-investors-edges — #12702

Resolves all investor nodes for a deal, then creates investor-to-funding-round edges. Handles institutional investors (VC firms), angel investors, and investment partners.

<Note>
  **Repaired v1.12 (2026-07-02, sandbox-verified):** `cascade_depth` is now a nullable input read via `first_notnull`, so the explicit seed `0` its caller (#12856 v1.6) passes survives and the seed-only branch fires — **the Cypher below is the live behavior**. A 2026-07-02 sandbox seed deal produced `Funding_Round` \+ `RAISED` w10 \+ `LEAD_INVESTED_IN` w15 \+ `INVESTED_IN` w20, with idempotent MERGEs on re-run and zero SKIP rows. From v1.11 (2026-06-23) until the fix, a `first_notempty` depth coercion made #12702 self-skip on every invocation — deals processed in that window carry `Funding_Round` \+ `RAISED` only and have `processed_at` stamped; re-cascading them would require nulling `fundable_deals.processed_at` — backfill **declined 2026-07-02** (accepted as historical, per Mark). See [Company Edges → Investor edge set](/guides/enrichment/waterfall/company-edges).
</Note>

```text
mvp/fundable/resolve-investors-edges — #12702
```

### Fix Domain-Slug Company Name

When an investor company node has a domain-slug name (e.g. `acme_com`), update it to the proper name from Fundable.

```cypher
// Version v1
MATCH (c:Company {uuid: '{{$var.investorNodeUuid}}'})
SET c.company_name = '{{$var.safeInvestorOrgName}}',
    c.name = '{{$var.safeInvestorOrgName}}'
RETURN c.company_name
```

### Institutional Investor — Lead (LEAD_INVESTED_IN)

```cypher
// Version v1
MATCH (vc:Company {uuid: '{{$var.investorNodeUuid}}'})
SET vc:VC_Firm,
    vc.updated_at = timestamp()
WITH vc
MATCH (fr:Funding_Round {uuid: '{{$var.deal.funding_round_node_uuid}}'})
MERGE (vc)-[r:LEAD_INVESTED_IN]->(fr)
SET r.created_at = coalesce(r.created_at, timestamp()),
    r.updated_at = timestamp(),
    r.uuid = '{{$var.investorNodeUuid}}-{{$var.deal.funding_round_node_uuid}}',
    r.name = NULL,
    r.description = '{{$var.safeInvestorOrgName}} led the {{$var.safeFundedCoName}} {{$var.safeFinancingType}} funding round.',
    r.is_lead_investor = true,
    r.weight = 15
RETURN vc.name AS investor, fr.uuid AS funding_round, 'LEAD_INVESTED_IN' AS rel_type
```

### Institutional Investor — INVESTED_IN / FOLLOW_ON_INVESTED_IN

Checks if the VC has invested in earlier rounds of the same company. If so, creates a FOLLOW_ON_INVESTED_IN edge instead of INVESTED_IN. **Correct classification depends on chronological processing:** `add-all-fundable-deals` #12703 v2.3 (2026-07-02) dispatches a firm's rounds `date asc` so earlier rounds are already in the graph when a later one resolves — out-of-order processing previously misclassified genuine follow-ons as first-time `INVESTED_IN`.

```cypher
// Version v1
MATCH (vc:Company {uuid: '{{$var.investorNodeUuid}}'})
SET vc:VC_Firm,
    vc.updated_at = timestamp()
WITH vc
MATCH (fr:Funding_Round {uuid: '{{$var.deal.funding_round_node_uuid}}'})
With vc, fr
MATCH (fundedCompany:Company {uuid: '{{$var.fundedCompanyOrg.master_company_node_uuid}}'})
OPTIONAL MATCH (fundedCompany)-[:RAISED]->(earlierRound:Funding_Round)
  <-[earlierInvestment]-(vc)
WHERE earlierRound.announced_date < '{{$var.formattedDate}}'
  AND type(earlierInvestment) IN
    ['LEAD_INVESTED_IN', 'INVESTED_IN', 'FOLLOW_ON_INVESTED_IN']
WITH vc, fr, COUNT(earlierRound) > 0 AS hasEarlierInvestment

FOREACH (_ IN CASE WHEN NOT hasEarlierInvestment THEN [1] ELSE [] END |
  MERGE (vc)-[r:INVESTED_IN]->(fr)
  SET r.created_at = coalesce(r.created_at, timestamp()),
      r.updated_at = timestamp(),
      r.uuid = '{{$var.investorNodeUuid}}-{{$var.deal.funding_round_node_uuid}}',
      r.name = NULL,
      r.description = '{{$var.safeInvestorOrgName}} invested in the {{$var.safeFundedCoName}} {{$var.safeFinancingType}} funding round.',
      r.is_lead_investor = false,
      r.weight = 20
)

FOREACH (_ IN CASE WHEN hasEarlierInvestment THEN [1] ELSE [] END |
  MERGE (vc)-[r:FOLLOW_ON_INVESTED_IN]->(fr)
  SET r.created_at = coalesce(r.created_at, timestamp()),
      r.updated_at = timestamp(),
      r.uuid = '{{$var.investorNodeUuid}}-{{$var.deal.funding_round_node_uuid}}',
      r.name = NULL,
      r.description = '{{$var.safeInvestorOrgName}} made a follow-on investment in the {{$var.safeFundedCoName}} {{$var.safeFinancingType}} funding round.',
      r.is_lead_investor = false,
      r.weight = 20
)

With vc, fr
MATCH (vc)-[r]->(fr)
WHERE type(r) IN ['INVESTED_IN', 'FOLLOW_ON_INVESTED_IN']
RETURN vc.name AS investor, fr.uuid AS funding_round, type(r) AS rel_type
```

### Investment Partner (INVESTMENT_PARTNER_IN + INVESTMENT_PARTNER_AT)

Links an individual person (partner at a VC firm) to both the funding round and their firm for a specific deal.

```cypher
// Version v1
MATCH (p:Person {uuid: '{{$var.partnerNodeUuid}}'})
MATCH (fr:Funding_Round {uuid: '{{$var.deal.funding_round_node_uuid}}'})
MATCH (vc:Company {uuid: '{{$var.investorNodeUuid}}'})

MERGE (p)-[dr:INVESTMENT_PARTNER_IN]->(fr)
SET dr.created_at = coalesce(dr.created_at, timestamp()),
    dr.updated_at = timestamp(),
    dr.uuid = '{{$var.partnerNodeUuid}}-{{$var.deal.funding_round_node_uuid}}',
    dr.description = '{{$var.safePartnerName}} was an investment partner in the {{$var.safeFundedCoName}} {{$var.safeFinancingType}} funding round, representing {{$var.safeInvestorOrgName}}.',
    dr.weight = 15

MERGE (p)-[dv:INVESTMENT_PARTNER_AT]->(vc)
SET dv.created_at = coalesce(dv.created_at, timestamp()),
    dv.updated_at = timestamp(),
    dv.uuid = '{{$var.partnerNodeUuid}}-{{$var.investorNodeUuid}}-deal_partner',
    dv.description = '{{$var.safePartnerName}} represented {{$var.safeInvestorOrgName}} as an investment partner in the {{$var.safeFundedCoName}} {{$var.safeFinancingType}} funding round.',
    dv.weight = 20

RETURN p.name AS partner, fr.uuid AS funding_round, vc.name AS vc_firm
```
