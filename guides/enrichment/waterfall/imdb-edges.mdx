The IMDB (Film & TV) Waterfall uses the canonical graph edge definitions in [Ontology → Edge Types](/guides/ontology/edges). Write-side Cypher for the credit \+ award edges lives on [Cyphers → Film & TV](/guides/cyphers/film-tv); canonical weights are on [Edge Weights → Film / TV & Awards](/guides/ontology/edge-weights#film--tv--awards) (**lower = stronger**; hub-node edges sit Weak or below; never sub-1 — the same shared scale as the [music edges](/guides/enrichment/waterfall/music-edges)).

This page is the **pipeline-scoped catalog**: which edges this pipeline emits, at what weight, from which writer. Edge patterns, properties, and examples should be edited only in the ontology source of truth.

<Note>
  Ground truth: [Edge Types](/guides/ontology/edges) \+ [Edge Weights](/guides/ontology/edge-weights). The waterfall implementation should link to those sections rather than redefining edge schemas.
</Note>

## Credit edges

### WORKED\_ON

`(Person)-[:WORKED_ON]->(Film_TV)` · weight **30**. Written for every credit whose film is already in the graph by `add-credit-edges-from-person` (#13013) and `add-credit-edges-from-title` (#13015), plus `#2334` in award context. Each side silently skips endpoints whose other side hasn't been enriched yet — symmetric, so the edge appears once both exist. See [Cyphers → Film & TV](/guides/cyphers/film-tv).

## Award edges

### HAS\_AWARD

`(Film_TV)-[:HAS_AWARD]->(Film_TV_Award)` · weight **40**. Written by `add-node-edge-imdb-award` (#2334), gated on `hasFilm` (honorary person-only awards have no `Film_TV` anchor).

### WON\_AWARD

`(Person)-[:WON_AWARD]->(Film_TV_Award:Winner)` · weight **30**. Written by #2334 (Query B), always fires so person-only awards still connect.

### NOMINATED\_FOR

`(Person)-[:NOMINATED_FOR]->(Film_TV_Award:Nominee)` · weight **35**. Written by #2334 (Query B).

## Expertise edges

### HAS\_EXPERTISE

`(Person)-[:HAS_EXPERTISE]->(SubDomainExpertise)` · weight **22–50** (confidence formula `min(round(10 + score × 80), 50)`). Written by `resolve-person-expertise-v2` (#12926), invoked from Phase 4b via `llm-identify-person-expertise` (#12666 v2). Refreshed idempotently — existing edges are deleted and rewritten per run.

## Edge descriptions

The `WORKED_ON` edges get LLM-generated descriptions via the #13016 → #13017 → #13018 trio, keyed on `(person_uuid, film_uuid)` because the credit writers don't stamp `e.uuid` (the [music trio](/guides/enrichment/waterfall/music-edges#edge-descriptions) keys on `e.uuid` instead). Model: `deepseek/deepseek-v4-flash` — migrated per [Refactor Edge Descriptions](/guides/robert-mark/refactor-edge-descriptions), verified live 2026-07-01.
