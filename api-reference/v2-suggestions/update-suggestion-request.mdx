---

## Overview

Update an existing suggestion request record by ID, including its status and context.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login or signup.
</ParamField>

## Path Parameters

<ParamField body="suggestion_request_id" type="integer" required>
  The ID of the suggestion request to update.
</ParamField>

## Request Body

<ParamField body="request_context" type="string">
  Updated context for the suggestion request.
</ParamField>

<ParamField body="status" type="string" required>
  The new status. One of `suggestion`, `processing`, `draft`, `submitted`, or `archived`.
</ParamField>

## Response

<ResponseField name="id" type="integer">
  The ID of the updated suggestion request.
</ResponseField>

<RequestExample>
```bash cURL
curl -X PUT https://xh2o-yths-38lt.n7c.xano.io/api:MkA4QsNh/suggestion-requests/123 \
  -H "Content-Type: application/json" \
  -H "X-Data-Source: YOUR_DATA_SOURCE" \
  -H "X-Branch: v1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "request_context": "Updated context",
    "status": "processing"
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": 123
}
```
</ResponseExample>
