## What it is

[Sigma.js](https://www.sigmajs.org) is a WebGL-based JavaScript library for rendering large interactive graphs (network diagrams) in the browser. It is a **renderer only** — the graph data model lives in a companion library called [Graphology](https://graphology.github.io). Sigma subscribes to Graphology events and re-renders on mutation.

- **Current version:** `sigma@3.0.2` (v3 was a ground-up rewrite released March 2024 by [OuestWare](https://www.ouestware.com/2024/03/21/sigma-js-3-0-en/))
- **Repo:** [jacomyal/sigma.js](https://github.com/jacomyal/sigma.js) — monorepo (Lerna + Preconstruct), ~94% TypeScript, MIT
- **Maintainers:** [@jacomyal](https://github.com/jacomyal) (Alexis Jacomy), [@Yomguithereal](https://github.com/Yomguithereal) (Guillaume Plique), [@sim51](https://github.com/sim51) (Benoît Simard) — all OuestWare
- **Flagship app:** [Gephi Lite](https://github.com/gephi/gephi-lite) — the canonical real-world v3 reference

## Why Sigma

- **WebGL renderer** — only mainstream JS graph library that stays smooth past ~5k nodes. Practical ceiling: ~50k–100k edges with default styles.
- **Separation of concerns** — graph algorithms live in Graphology (usable in Node, workers, scripts); Sigma is just the renderer.
- **Framework-agnostic core** with first-class React bindings ([@react-sigma/core](https://sim51.github.io/react-sigma/)).
- **GPU picking** for hit detection on both nodes and edges (v3).
- **TypeScript generics** flow through `Sigma<NodeAttrs, EdgeAttrs, GraphAttrs>`, reducers, and events.

<Note>
v3 is a breaking change from v2. The size API was rewritten so sizes live in the same reference frame as positions and scale linearly with zoom. Porting v2 code directly will make sizes look wrong. See [the release post](https://www.ouestware.com/2024/03/21/sigma-js-3-0-en/).
</Note>

## Minimal example (vanilla)

```ts
import Graph from "graphology";
import Sigma from "sigma";

const graph = new Graph();
graph.addNode("a", { x: 0, y: 0, size: 10, label: "Alice", color: "#3b82f6" });
graph.addNode("b", { x: 1, y: 1, size: 15, label: "Bob",   color: "#ef4444" });
graph.addEdge("a", "b", { size: 2, color: "#999" });

const container = document.getElementById("container")!;
const renderer = new Sigma(graph, container);

// Cleanup when done
// renderer.kill();
```

<Warning>
The container must have **non-zero width and height** or Sigma throws at init (unless you pass `allowInvalidContainer: true`). "Graph doesn't render" is almost always a zero-sized container.
</Warning>

## Architecture: Sigma + Graphology

Sigma doesn't store graph data. Graphology does. Sigma listens to `nodeAdded`, `edgeAdded`, `nodeAttributesUpdated`, etc. and re-renders automatically on mutation. For visual-only state (hover, selection, filters) you use **reducers** — pure functions that compute display attributes without mutating the graph.

```
┌────────────────────┐        subscribes        ┌────────────┐
│  graphology Graph  │ ──────────────────────▶  │   Sigma    │
│  (data + events)   │                          │ (renderer) │
└────────────────────┘        renders on        └────────────┘
        ▲                     mutation
        │
        │ setNodeAttribute, addEdge, import()
        │
   ┌────┴──────┐
   │ your code │
   └───────────┘
```

## Graph data (Graphology)

Every node must have numeric `x`, `y`, `size`. `label` and `color` are strong conventions Sigma reads by default.

### Graph types

```ts
import Graph, {
  DirectedGraph,
  UndirectedGraph,
  MultiGraph,
  MultiDirectedGraph,
} from "graphology";

// Default Graph: mixed, self-loops allowed, parallel edges REJECTED
// Use MultiGraph / MultiDirectedGraph if you need parallel edges
const graph = new MultiDirectedGraph();
```

### Common API

```ts
// Mutate
graph.addNode("a", { x: 0, y: 0, size: 5, label: "A", color: "#f00" });
graph.addEdge("a", "b", { size: 1, weight: 0.5 });
graph.addEdgeWithKey("custom-id", "a", "b", { size: 1 });
graph.setNodeAttribute("a", "color", "#00f");
graph.mergeNodeAttributes("a", { size: 10, label: "Alice" });
graph.updateNodeAttribute("a", "size", (s) => (s ?? 1) * 2);
graph.dropNode("a");
graph.dropEdge("a", "b");

// Read
graph.hasNode("a");
graph.getNodeAttributes("a");
graph.getNodeAttribute("a", "label");
graph.neighbors("a");       // array of node ids
graph.inNeighbors("a");
graph.outNeighbors("a");
graph.areNeighbors("a", "b");
graph.extremities("edge-id"); // [source, target]
graph.hasExtremity("edge-id", "a");
graph.order;   // node count
graph.size;    // edge count

// Iterate
graph.forEachNode((id, attrs) => { /* ... */ });
graph.forEachEdge((id, attrs, src, tgt) => { /* ... */ });
graph.forEachNeighbor("a", (id, attrs) => { /* ... */ });
```

### Import / export

```ts
// Native JSON round-trip
graph.import({
  attributes: { name: "my graph" },
  nodes: [
    { key: "A", attributes: { x: 0, y: 0, size: 5, label: "A" } },
    { key: "B", attributes: { x: 1, y: 0, size: 5, label: "B" } },
  ],
  edges: [{ key: "e1", source: "A", target: "B", attributes: { size: 1 } }],
});
const serialized = graph.export();

// GEXF (Gephi)
import Graph from "graphology";
import { parse as parseGexf } from "graphology-gexf/browser";
const xml = await fetch("/data.gexf").then(r => r.text());
const g = parseGexf(Graph, xml, { addMissingNodes: true });

// GraphML
import { parse as parseGraphML } from "graphology-graphml/browser";
```

<Tip>
Bulk-load with `graph.import({ nodes, edges })` in one call rather than 500 individual `addNode`s. Each mutation fires events that Sigma listens to.
</Tip>

## Sigma instance API

### Constructor

```ts
new Sigma<N, E, G>(
  graph: Graph<N, E, G>,
  container: HTMLElement,
  settings?: Partial<Settings<N, E, G>>
)
```

### Lifecycle

```ts
renderer.refresh();                                    // reprocess + schedule render
renderer.refresh({ skipIndexation: true });            // skip quadtree rebuild (fast path)
renderer.refresh({ partialGraph: { nodes: ["a"] } });  // partial refresh for perf
renderer.render();                                     // force immediate render
renderer.scheduleRender();                             // RAF-schedule
renderer.resize();                                     // recompute dimensions
renderer.kill();                                       // destroy WebGL contexts, remove listeners
```

<Warning>
**Always call `renderer.kill()` on unmount.** WebGL contexts leak silently. In React, return it from `useEffect`.
</Warning>

## Coordinate systems

Three coordinate spaces. Most bugs come from mixing them.

| Space | Unit | Notes |
|---|---|---|
| **graph** | arbitrary (your data) | raw `node.x`, `node.y` |
| **framed graph** | [0, 1] normalized, y-flipped | what Camera operates on |
| **viewport** | pixels, y-down | what mouse events report |

```ts
renderer.graphToViewport({ x: 10, y: 20 });   // graph  → viewport
renderer.viewportToGraph({ x: 100, y: 200 }); // viewport → graph
renderer.framedGraphToViewport({ x, y });
renderer.viewportToFramedGraph({ x, y });
```

<Warning>
Viewport y is **flipped** (higher y = lower on screen). This bites when integrating with Leaflet/MapLibre overlays.
</Warning>

## Events

Sigma extends `TypedEventEmitter` — use `renderer.on`, `off`, `once`, `emit`.

### Full event list

| Category | Names |
|---|---|
| **Stage** (empty canvas) | `clickStage`, `doubleClickStage`, `rightClickStage`, `wheelStage`, `downStage`, `upStage`, `enterStage`, `leaveStage` |
| **Node** | `clickNode`, `doubleClickNode`, `rightClickNode`, `wheelNode`, `downNode`, `upNode`, `enterNode`, `leaveNode` |
| **Edge** (requires `enableEdgeEvents: true`) | `clickEdge`, `doubleClickEdge`, `rightClickEdge`, `wheelEdge`, `downEdge`, `upEdge`, `enterEdge`, `leaveEdge` |
| **Lifecycle** | `beforeClear`, `afterClear`, `beforeProcess`, `afterProcess`, `beforeRender`, `afterRender`, `resize`, `kill` |
| **Body capture** (for drag) | `moveBody` |
| **Camera** | fires `"updated"` on the camera instance, not the renderer |

### Payload shapes

```ts
interface SigmaEventPayload { event: MouseCoords; preventSigmaDefault(): void; }
interface SigmaNodeEventPayload extends SigmaEventPayload { node: string; }
interface SigmaEdgeEventPayload extends SigmaEventPayload { edge: string; }

interface MouseCoords extends Coordinates {
  sigmaDefaultPrevented: boolean;
  preventSigmaDefault(): void;
  original: MouseEvent | TouchEvent;
}
```

### Example

```ts
renderer.on("clickNode", ({ node, event }) => {
  console.log("clicked", node, "at", event.x, event.y);
});
renderer.on("enterNode", ({ node }) => setHovered(node));
renderer.on("leaveNode", () => setHovered(null));
renderer.on("clickStage", ({ event }) => {
  const g = renderer.viewportToGraph({ x: event.x, y: event.y });
  console.log("stage click at graph coord", g);
});
```

## Reducers — dynamic styling without mutating

Reducers run **every render** and let you compute display attributes from graph data plus external state. They never mutate the graph. If external state changes, call `renderer.refresh({ skipIndexation: true })`.

```ts
import type { NodeDisplayData, EdgeDisplayData } from "sigma/types";

let hoveredNode: string | null = null;
let hoveredNeighbors: Set<string> | null = null;

renderer.setSetting("nodeReducer", (node, data): Partial<NodeDisplayData> => {
  const res = { ...data };
  if (hoveredNode && node !== hoveredNode && !hoveredNeighbors?.has(node)) {
    res.label = "";
    res.color = "#f6f6f6";
  }
  if (node === hoveredNode) res.highlighted = true;
  return res;
});

renderer.setSetting("edgeReducer", (edge, data): Partial<EdgeDisplayData> => {
  const res = { ...data };
  const graph = renderer.getGraph();
  if (hoveredNode && !graph.extremities(edge).includes(hoveredNode)) {
    res.hidden = true;
  }
  return res;
});
```

<Warning>
Reducers run per-item per-frame. **Keep them O(1).** Pre-compute `Set`s of neighbors once in state; don't call `graph.forEachNeighbor` inside the reducer.
</Warning>

```ts
import { NodeCircleProgram } from "sigma/rendering";
import { NodeBorderProgram } from "@sigma/node-border";
import { NodeImageProgram } from "@sigma/node-image";

new Sigma(graph, container, {
  defaultNodeType: "circle",

    bordered: NodeBorderProgram,
```

## React integration (@react-sigma/core)

### SigmaContainer

Mounts a `<div className="sigma-container">`, instantiates Sigma, and exposes the instance to children via context. Every hook must live **inside** the container.

```tsx
import { SigmaContainer } from "@react-sigma/core";
import { MultiDirectedGraph } from "graphology";
import "@react-sigma/core/lib/style.css";

export function GraphApp() {
  return (
    <SigmaContainer
      style={{ height: "100vh", width: "100%" }}
      graph={MultiDirectedGraph}     // the CONSTRUCTOR, not an instance
      settings={{
        renderLabels: true,
        hideEdgesOnMove: true,
        hideLabelsOnMove: true,
        labelRenderedSizeThreshold: 6,
        defaultNodeColor: "#6c63ff",
      }}
    >
      <LoadGraph />
      <GraphEvents />
    </SigmaContainer>
  );
}
```

<Warning>
**Don't pass inline objects to `settings` or `graph`.** Any change re-instantiates Sigma. Memoize with `useMemo` or hoist to module scope.
</Warning>

### Hover highlight pattern

```tsx
function GraphEvents() {
  const sigma = useSigma<NodeAttrs, EdgeAttrs>();
  const registerEvents = useRegisterEvents<NodeAttrs, EdgeAttrs>();
  const setSettings = useSetSettings<NodeAttrs, EdgeAttrs>();
  const [hovered, setHovered] = useState<string | null>(null);

  useEffect(() => {
    registerEvents({
      enterNode: (e) => setHovered(e.node),
      leaveNode: () => setHovered(null),
    });
  }, [registerEvents]);

  useEffect(() => {
    const graph = sigma.getGraph();
    setSettings({
      nodeReducer: (node, data) => {
        if (!hovered) return data;
        if (node === hovered || graph.areNeighbors(hovered, node)) {
          return { ...data, zIndex: 1, highlighted: true };
        }
        return { ...data, color: "#eee", label: "", zIndex: 0 };
      },
      edgeReducer: (edge, data) => {
        if (!hovered) return data;
        return graph.hasExtremity(edge, hovered)
          ? { ...data, color: "#333", size: (data.size ?? 1) * 1.5 }
          : { ...data, hidden: true };
      },
    });
  }, [hovered, sigma, setSettings]);

  return null;
}
```

```ts
  size: number;
  weight: number;
  kind: "knows" | "invested_in";
  label?: string;
};

type GraphAttrs = { name: string };

const graph: Graph<NodeAttrs, EdgeAttrs, GraphAttrs> = new Graph();
graph.addNode("a", { x: 0, y: 0, size: 5, label: "A", color: "#f00", kind: "person" });
// ^ compile error if you omit `kind` or pass unknown fields

// React-sigma hooks flow the generics:
const sigma = useSigma<NodeAttrs, EdgeAttrs, GraphAttrs>();
const loadGraph = useLoadGraph<NodeAttrs, EdgeAttrs, GraphAttrs>();
```

`Settings` is typed as `Settings<N, E, G>` — `nodeReducer` / `edgeReducer` receive the correctly-typed attributes.

## Animation

Sigma's animation primitives are small and focused: camera tweening + a numeric-attribute tweening helper. Layout transitions and entrance effects compose on top.

### Camera animations

Covered above under [Camera API](#camera-api). TL;DR:

```ts
const camera = renderer.getCamera();
camera.animate({ x, y, ratio: 0.3 }, { duration: 600, easing: "cubicInOut" });
camera.animatedReset({ duration: 400 });
camera.animatedZoom(1.5);
```

<Warning>
`ratio` is **inverted**: lower ratio = zoomed in. `ratio: 0.5` ≈ 2× zoom.
</Warning>

### animateNodes — tween numeric attributes

```ts
import { animateNodes } from "sigma/utils/animate";

const cancel = animateNodes(
  graph,
  {
    a: { x: 0.5, y: 0.5, size: 20 },
    b: { x: 0.2, y: 0.8, size: 10 },
  },
  { duration: 800, easing: "cubicInOut" },
  () => console.log("done"),
);

// Interrupt
// cancel();
```

- **Numeric only.** Colors and strings don't tween — handle them with reducers or a manual RAF loop.
- Default `duration = 1000`, default `easing = "quadraticInOut"`.
- Sigma auto-rerenders on attribute change — no manual `refresh()` per step.

### Layout transitions

The canonical pattern: compute target positions, hand to `animateNodes`.

```ts
import circular from "graphology-layout/circular";
import forceAtlas2 from "graphology-layout-forceatlas2";
import { animateNodes } from "sigma/utils/animate";

circular.assign(graph);   // seed so FA2 has valid x/y

const targets = forceAtlas2(graph, {
  iterations: 200,
  settings: forceAtlas2.inferSettings(graph),
});

animateNodes(graph, targets, { duration: 1200, easing: "cubicInOut" });
```

### Hover / focus fades (reducer-driven)

Animate an external `fadeT` variable, drive reducers from it, call `refresh({ skipIndexation: true })` each frame.

```ts
let hovered: string | null = null;
let fadeT = 0;

renderer.setSetting("nodeReducer", (node, data) => {
  if (!hovered) return data;
  const isNeighbor = node === hovered || graph.areNeighbors(hovered, node);
  const alpha = isNeighbor ? 1 : 1 - fadeT * 0.85;
  return { ...data, color: withAlpha(data.color, alpha) };
});

function animateFade(dir: 1 | -1) {
  const t0 = performance.now();
  const start = fadeT;
  const step = (now: number) => {
    const p = Math.min(1, (now - t0) / 180);
    fadeT = Math.max(0, Math.min(1, start + dir * p));
    renderer.refresh({ skipIndexation: true });
    if (p < 1) requestAnimationFrame(step);
    else if (dir < 0) hovered = null;
  };
  requestAnimationFrame(step);
}

renderer.on("enterNode", ({ node }) => { hovered = node; animateFade(1); });
renderer.on("leaveNode", () => animateFade(-1));
```

### Entrance animations

**Fade + scale from zero:**

```ts
// Init hidden
graph.forEachNode((n) => graph.mergeNodeAttributes(n, { size: 0, _alpha: 0 }));

const targets: Record<string, { size: number; _alpha: number }> = {};
graph.forEachNode((n, a) => (targets[n] = { size: a.targetSize, _alpha: 1 }));
animateNodes(graph, targets, { duration: 900, easing: "cubicOut" });
// Read _alpha in nodeReducer to set color alpha
```

**Radial reveal** — delay each node by its distance to center:

```ts
const cx = 0.5, cy = 0.5;
graph.forEachNode((n, a) => {
  const d = Math.hypot(a.x - cx, a.y - cy);
  setTimeout(() => {
    animateNodes(graph, { [n]: { size: a.targetSize } }, {
      duration: 400, easing: "cubicOut",
    });
  }, d * 800);
});
```

### Animation performance

- **Never** call `refresh()` in a RAF loop without `skipIndexation: true` — full reindexation rebuilds the quadtree.
- Use `refresh({ partialGraph: { nodes: ["a", "b"] } })` when animating a small subset.
- Attribute changes (`setNodeAttribute`) auto-trigger a re-render — no manual refresh needed.
- Enable `hideEdgesOnMove` + `hideLabelsOnMove` during animation for large graphs.
- Prefer camera tweens (1 uniform update) over per-node tweens on >5k nodes.

## Layout arrangement

Sigma leans on the `graphology-layout-*` ecosystem. A layout is any function that writes numeric `x`/`y` to nodes — Sigma picks it up automatically.

### ForceAtlas2 — the workhorse

`graphology-layout-forceatlas2`. Two flavors: **sync** (blocks main thread) and **worker** (non-blocking, long-running).

**Sync — seed or small graphs:**

```ts
import forceAtlas2 from "graphology-layout-forceatlas2";

const settings = forceAtlas2.inferSettings(graph);  // sensible defaults from graph size
forceAtlas2.assign(graph, { iterations: 100, settings });

// Read-only variant
const positions = forceAtlas2(graph, { iterations: 100, settings });
```

**Worker — production:**

```ts
import FA2Layout from "graphology-layout-forceatlas2/worker";

const layout = new FA2Layout(graph, {
  settings: forceAtlas2.inferSettings(graph),
});
layout.start();
// later...
layout.stop();    // pause (can resume with start())
layout.kill();    // terminate worker — REQUIRED or it leaks
layout.isRunning();
```

**Settings tuning:**

| Setting | Default | When to change |
|---|---|---|
| `gravity` | 1 | Raise (5–20) to pull disconnected components inward |
| `scalingRatio` | 1 | Raise (5–10) to spread nodes — most common knob |
| `barnesHutOptimize` | false | **Enable for >1k nodes** — O(n²) → O(n log n) |
| `barnesHutTheta` | 0.5 | Lower = more accurate, slower |
| `slowDown` | 1 | Raise (5–20) to stabilise jittery layouts |
| `strongGravityMode` | false | Radial spring — good for single-component graphs |
| `adjustSizes` | false | Prevent node overlap using size attribute |
| `linLogMode` | false | Tighter clusters |
| `outboundAttractionDistribution` | false | Scale-free graphs — hubs repelled from hubs |
| `edgeWeightInfluence` | 0 | 1 = linear weight effect |

<Warning>
**All nodes must have numeric `x` and `y` before FA2 runs.** Seed with `random.assign(graph)` or `circular.assign(graph)` first or the solver emits NaN.
</Warning>

<Tip>
`barnesHutOptimize: true` is usually slower on graphs with fewer than 500 nodes due to tree-build overhead. Flip it on above ~1k nodes.
</Tip>

### Force (spring)

`graphology-layout-force`. Simpler spring model — smoother for drag-and-drop interactivity, struggles past ~500 nodes.

```ts
import forceLayout from "graphology-layout-force";
import ForceSupervisor from "graphology-layout-force/worker";

forceLayout.assign(graph, {
  maxMove: 200,
  settings: {
    attraction: 0.0005,
    repulsion: 0.1,
    gravity: 0.0001,
    inertia: 0.6,
    maxMove: 200,
  },
});

// Worker version
const supervisor = new ForceSupervisor(graph, {
  isNodeFixed: (_, a) => a.fixed,
});
supervisor.start();
```

### Circular

```ts
import circular from "graphology-layout/circular";

// center is applied as { x: center, y: center }
// Deterministic — nodes placed in insertion order
```

### Hierarchical / tree

Graphology doesn't ship a first-class dagre port. Options:

**dagre bridge (best for DAGs with cross edges):**

```ts
import dagre from "@dagrejs/dagre";

g.setGraph({ rankdir: "TB", nodesep: 50, ranksep: 80, edgesep: 10 });
g.setDefaultEdgeLabel(() => ({}));

  const { x, y } = g.node(n);

```

**d3-hierarchy (pure trees):** build `d3.hierarchy()`, run `tree()` or `cluster()`, copy `node.x`/`node.y` back.

### Group / clustered (by community)

Detect communities with Louvain, place cluster centroids in a circle, layout nodes within each cluster.

```ts
import louvain from "graphology-communities-louvain";
import forceAtlas2 from "graphology-layout-forceatlas2";

// 1) Detect communities — writes `community` attr
louvain.assign(graph, { resolution: 1 });

// 2) Place cluster centroids
const communities = new Set<number>();
graph.forEachNode((_, a) => communities.add(a.community));
const centroids: Record<number, { x: number; y: number }> = {};
const R = 500;
let i = 0;
for (const c of communities) {
  const angle = (2 * Math.PI * i++) / communities.size;
  centroids[c] = { x: Math.cos(angle) * R, y: Math.sin(angle) * R };
}

// 3) Seed nodes near their centroid with jitter
graph.forEachNode((n, a) => {
  const { x, y } = centroids[a.community];
  graph.mergeNodeAttributes(n, {
    x: x + (Math.random() - 0.5) * 50,
    y: y + (Math.random() - 0.5) * 50,
  });
});

// 4) Refine with FA2 (short run keeps clusters separate)
forceAtlas2.assign(graph, {
  iterations: 50,
  settings: { gravity: 5, scalingRatio: 2, barnesHutOptimize: true },
});
```

### Custom layouts

Any function writing numeric `x`/`y` to nodes is a valid layout.

```ts
function gridLayout(graph, cols = 10, spacing = 50) {
  let i = 0;
  graph.forEachNode((n) => {
    graph.mergeNodeAttributes(n, {
      x: (i % cols) * spacing,
      y: Math.floor(i / cols) * spacing,
    });
    i++;
  });
}
```

For per-frame iteration (live custom force), drive with `requestAnimationFrame` and mutate attributes in the loop — Sigma auto-renders.

### @react-sigma layout hooks

Sync hooks return `{ positions, assign }`. Worker hooks return `{ start, stop, kill, isRunning }`.

```tsx
import { useLayoutCircular } from "@react-sigma/layout-circular";
import { useLayoutRandom } from "@react-sigma/layout-random";
import {
  useLayoutForceAtlas2,
  useWorkerLayoutForceAtlas2,
  LayoutForceAtlas2Control,
} from "@react-sigma/layout-forceatlas2";
import {
  useLayoutNoverlap,
  useWorkerLayoutNoverlap,
} from "@react-sigma/layout-noverlap";

function LayoutRunner() {
  const { assign: circularAssign } = useLayoutCircular();

  // Sync FA2 — blocks main thread
  const { positions, assign: fa2Assign } = useLayoutForceAtlas2({
    iterations: 100,
    settings: { gravity: 1, scalingRatio: 10, barnesHutOptimize: true },
  });

  // Worker FA2 — non-blocking, use for anything non-trivial
  const { start, stop, kill, isRunning } = useWorkerLayoutForceAtlas2({
    settings: { gravity: 1, scalingRatio: 10 },
  });

  useEffect(() => {
    circularAssign();       // seed
    start();                // non-blocking refinement
    const t = setTimeout(stop, 5000);
    return () => {
      clearTimeout(t);
      kill();               // CRITICAL — worker leaks without this
    };
  }, []);

  return null;
}

// Built-in control widget
<ControlsContainer position="bottom-right">

</ControlsContainer>
```

<Warning>
React StrictMode double-invokes effects. Always return a cleanup that calls `kill()` on worker layouts or you leak workers.
</Warning>

### Canonical pipeline

```ts
import Graph from "graphology";
import Sigma from "sigma";
import circular from "graphology-layout/circular";
import forceAtlas2 from "graphology-layout-forceatlas2";
import FA2Layout from "graphology-layout-forceatlas2/worker";

// 1) Build
const graph = new Graph();
// ... addNode, addEdge ...

// 2) Seed — FA2 needs valid x/y
circular.assign(graph);

// 3) Render
const sigma = new Sigma(graph, container);

// 4) Refine with worker FA2
const layout = new FA2Layout(graph, {
  settings: forceAtlas2.inferSettings(graph),
});

// 5) Stop after stabilisation
const stopTimer = setTimeout(() => layout.stop(), 4000);

// 6) Clean up on teardown
function cleanup() {
  clearTimeout(stopTimer);
  layout.kill();
  sigma.kill();
}
```

### Click to focus camera

### Drag-and-drop nodes

```ts
let draggedNode: string | null = null;

renderer.on("downNode", (e) => {
  draggedNode = e.node;
  graph.setNodeAttribute(draggedNode, "highlighted", true);
  if (!renderer.getCustomBBox()) renderer.setCustomBBox(renderer.getBBox());
});

renderer.on("moveBody", ({ event }) => {
  if (!draggedNode) return;
  const pos = renderer.viewportToGraph(event);
  graph.setNodeAttribute(draggedNode, "x", pos.x);
  graph.setNodeAttribute(draggedNode, "y", pos.y);
  event.preventSigmaDefault();
  event.original.preventDefault();
  event.original.stopPropagation();
});

const handleUp = () => {
  if (draggedNode) graph.removeNodeAttribute(draggedNode, "highlighted");
  draggedNode = null;
};
renderer.on("upNode", handleUp);
renderer.on("upStage", handleUp);
```

Key bits: `setCustomBBox` on drag start prevents Sigma from auto-fitting while you drag out of bounds. `event.preventSigmaDefault()` blocks camera pan during the drag.

### Lasso / box select

No official plugin. Pattern:

1. On stage `mousedown`, show an overlay div, disable camera with `renderer.getMouseCaptor().disable()`.
2. On `mousemove`, update overlay geometry.
3. On `mouseup`, iterate `graph.nodes()`, convert each to viewport coords with `graphToViewport`, test point-in-polygon (lasso) or AABB (box).
4. Re-enable camera: `renderer.getMouseCaptor().enable()`.

### Search

Built-in `<SearchControl>` from `@react-sigma/core` handles label-based search out of the box. Custom search: iterate `graph.nodes()`, filter by label, then `camera.animate({ x, y, ratio: 0.3 })`.

### Filter

Drive `nodeReducer` / `edgeReducer` from React state; set `hidden: true` on excluded items. **Don't mutate the graph** for transient UI filtering.

### Context menus

Sigma emits `rightClickNode`, `rightClickEdge`, `rightClickStage`. Position a floating DOM menu at `event.event.x / y` and call `event.preventSigmaDefault()` to suppress default camera behavior.

## Performance playbook

For graphs >5k nodes:

1. **Worker layouts only.** Never run sync FA2 on the main thread.
2. **`hideEdgesOnMove: true`** and **`hideLabelsOnMove: true`**.
3. **Tune label density:**
   - `labelRenderedSizeThreshold: 6` or higher
   - `labelGridCellSize: 60–100`
   - `labelDensity: 0.07` for sparse labels
4. **Refresh smart:** `refresh({ skipIndexation: true })` for attribute-only changes, `refresh({ partialGraph: { nodes: [...] } })` for small subsets.
5. **Memoize `SigmaContainer` props** — inline `settings` or `graph` props re-instantiate Sigma.
6. **Bulk import** — `graph.import({ nodes, edges })` once, not 500 `addNode`s.
7. **Pure reducers** — pre-compute neighbor `Set`s outside; no iteration inside reducers.
8. **Keep node `size` attributes small** (1–20 range). Large values can exceed GPU `gl.POINTS` limits and silently break rendering (see [issue #1264](https://github.com/jacomyal/sigma.js/issues/1264)).
9. **Edge density:** if `edges > 10 × nodes`, show edges only on hover via `edgeReducer` (default `hidden: true`, false when related to hovered node).
10. **Avoid `zIndex: true`** unless you actually need layered drawing — it costs a per-frame sort.
11. **Cheap nodes:** prefer `NodePointProgram` (1 vertex/node) over `NodeCircleProgram` (3 vertices) when you don't need per-pixel anti-aliasing.

## SSR gotchas

Sigma imports WebGL context at module evaluation time — it **cannot** run on the server.

### Next.js App Router

`"use client"` is **not enough**. You must prevent server bundling entirely:

```tsx
// app/graph/page.tsx
import dynamic from "next/dynamic";

const GraphClient = dynamic(() => import("@/components/GraphClient"), {
  ssr: false,
});

export default function Page() {
  return <GraphClient />;
}
```

In `GraphClient.tsx`, put `"use client"` at the top and import normally.

### TanStack Start

Two options:

```tsx
// Route-level opt-out
export const Route = createFileRoute("/graph")({
  ssr: false,
  component: GraphPage,
});

// Or component-level with ClientOnly
import { ClientOnly } from "@tanstack/react-router";

<ClientOnly fallback={<div>Loading graph...</div>}>
  <GraphComponent />
</ClientOnly>
```

TanStack Start has no `next/dynamic` equivalent — use `React.lazy` + `Suspense` inside `ClientOnly` for dynamic imports.

### Vite / pure CSR

No special handling required.

## When to pick Sigma.js

| Nodes | Best choice |
|---|---|
| Under 3,000 | Any lib — Cytoscape, vis-network, Sigma, d3-force all fine |
| 3,000 – 5,000 | Canvas libs start degrading; Sigma or tuned vis-network |
| 5,000 – 50,000 | **Sigma.js** — the sweet spot |
| 50,000 – 500,000 | **Sigma.js** — only practical option without LOD tricks |
| 500,000+ | Consider Cosmograph / cosmos.gl (GPU force in shaders) |

**Trade-offs vs alternatives:**

- **vs Cytoscape.js** — Cytoscape has richer graph algorithms (shortest-path, centrality suite, BFS/DFS) and more out-of-box layouts, but Canvas-based rendering degrades past ~5k nodes. Pick Cytoscape for small bioinformatics-style graphs needing algorithms, Sigma for large visual graphs.
- **vs vis-network** — vis has smoother out-of-box interactivity (drag, physics, clustering); Sigma has dramatically better performance at scale.
- **vs D3-force** — D3 wins for under 500 nodes with heavy custom visuals and SVG/DOM needs. Sigma wins at scale.
- **vs Ogma (Linkurious)** — Ogma is commercial, has advanced layout and graph-simplification baked in. Sigma is free and open but you build those features yourself.

## Common gotchas — quick checklist

1. **Container needs explicit height** or Sigma silently renders nothing.
2. **In Next.js,** dynamic-import with `ssr: false`. `"use client"` alone isn't enough.
3. **Memoize `settings` and `graph` props** on `SigmaContainer` — inline objects cause re-instantiation.
4. **`refresh({ skipIndexation: true })`** when only attributes changed — full reindex is expensive.
5. **Keep node sizes small** (1–20 range). Large values can silently fail on GPU.
6. **Translate mouse events** with `sigma.viewportToGraph()`. Pixel != graph coords.
7. **Use worker FA2** (`graphology-layout-forceatlas2/worker`) for non-trivial graphs.
8. **Seed positions before FA2** — circular or random. FA2 emits NaN with missing x/y.
9. **Use reducers for visual-only state**; don't mutate graphology.
10. **Multi-edges need `MultiGraph` / `MultiDirectedGraph`** — default graph rejects parallels.
11. **Edge labels require node labels** on endpoints ([issue #1200](https://github.com/jacomyal/sigma.js/issues/1200)).
12. **Disable camera during drag** via `preventSigmaDefault()` + `setCustomBBox`.
13. **Always `sigma.kill()` and `workerLayout.kill()` on unmount.** WebGL + workers leak.
14. **Use `@sigma/export-image`** rather than hand-rolling canvas snapshots.
15. **v4 will break label rendering API** (Canvas → DOM). Keep label customization isolated.

## End-to-end React app

```tsx
// GraphApp.tsx
"use client";
import { useEffect, useMemo, useState } from "react";
import {
  SigmaContainer, useLoadGraph, useRegisterEvents, useSigma, useSetSettings,
  ControlsContainer, ZoomControl, SearchControl,
} from "@react-sigma/core";
import { useWorkerLayoutForceAtlas2 } from "@react-sigma/layout-forceatlas2";
import { useLayoutCircular } from "@react-sigma/layout-circular";
import { MultiDirectedGraph } from "graphology";
import "@react-sigma/core/lib/style.css";

type N = { x: number; y: number; size: number; label: string; color: string };
type E = { size: number; label?: string };

interface Data {
  nodes: Array<{ id: string } & N>;
  edges: Array<[string, string, E]>;
}

function LoadGraph({ data }: { data: Data }) {
  const loadGraph = useLoadGraph<N, E>();
  const { assign: assignCircular } = useLayoutCircular();

  useEffect(() => {
    const g = new MultiDirectedGraph<N, E>();
    data.nodes.forEach((n) => g.addNode(n.id, n));
    data.edges.forEach(([s, t, attrs]) => g.addEdge(s, t, attrs));
    loadGraph(g);
    assignCircular();
  }, [data, loadGraph, assignCircular]);

  return null;
}

function Fa2() {
  const { start, kill } = useWorkerLayoutForceAtlas2({
    settings: { slowDown: 10, barnesHutOptimize: true },
  });
  useEffect(() => {
    start();
    return () => kill();
  }, [start, kill]);
  return null;
}

function Hover() {
  const sigma = useSigma<N, E>();
  const register = useRegisterEvents<N, E>();
  const setSettings = useSetSettings<N, E>();
  const [h, setH] = useState<string | null>(null);

  useEffect(() => {
    register({
      enterNode: (e) => setH(e.node),
      leaveNode: () => setH(null),
    });
  }, [register]);

  useEffect(() => {
    const g = sigma.getGraph();
    setSettings({
      nodeReducer: (n, d) =>
        !h
          ? d
          : n === h || g.areNeighbors(h, n)
            ? { ...d, highlighted: true }
            : { ...d, color: "#eee", label: "" },
      edgeReducer: (e, d) =>
        !h ? d : g.hasExtremity(e, h) ? d : { ...d, hidden: true },
    });
  }, [h, sigma, setSettings]);

  return null;
}

export default function GraphApp({ data }: { data: Data }) {
  const settings = useMemo(
    () => ({
      renderLabels: true,
      hideEdgesOnMove: true,
      hideLabelsOnMove: true,
      labelRenderedSizeThreshold: 6,
      defaultNodeColor: "#6c63ff",
    }),
    [],
  );

  return (
    <SigmaContainer
      style={{ height: "100vh" }}
      graph={MultiDirectedGraph}
      settings={settings}
    >
      <LoadGraph data={data} />
      <Fa2 />
      <Hover />
      <ControlsContainer position="bottom-right">
        <ZoomControl />
      </ControlsContainer>
      <ControlsContainer position="top-right">
        <SearchControl style={{ width: 200 }} />
      </ControlsContainer>
    </SigmaContainer>
  );
}
```
