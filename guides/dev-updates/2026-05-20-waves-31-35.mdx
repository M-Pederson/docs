## Shipped

<Steps>
  <Step title="Wave 31 — Past Results tab restored">
    Charles's deleted legacy Outcomes list/detail UI (21 files) was recovered **byte-exact**
    from history and re-mounted as a new "Past Results" top-nav tab. The `fetchOutcomeById`
    page cap was raised 10 → 100 so the full back catalogue of `suggestion_request` rows
    deep-links. `self_company_yaml_context` is now wired into all three dispatch payloads, and
    a `[YOUR COMPANY]` block is appended to the backend memory context.
  </Step>
  <Step title="Wave 32 — Leverage Loops parity on OpenUI 0.5">
    Leverage Loops migrated off the old Crayon C1 framework onto OpenUI 0.5, pixel-matching the
    Outcomes chat. A new LL-framed interview endpoint covers 12 class branches; a turn-1 draft
    row is created on first engage, each turn persists immediately, and right-rail bridge events
    fire at every phase. Charles's response-section and list UI stay untouched per Mark's rule.
  </Step>
  <Step title="Wave 33 / 33a — Meeting Prep strategist prompt">
    Meeting Prep now runs Mark's verbatim strategist prompt: **landmines surfaced first**, plus
    level-set, path-to-outcome, connection points, questions to ask, and failure modes. The
    legacy talking-points view only renders when the new strategist sections are absent
    (back-compat). A one-line backend fix resolved a parse failure where a null company field
    was stripped before it was read.
  </Step>
  <Step title="Wave 34 — Voice bulletproof">
    The text-to-speech layer was made race-proof: a three-step abort window in the voice hook
    was replaced with a generation-counter pattern, `speak()` became promise-based, and a
    guard closes the "TTS reads the top line" bug. Seven new tests cover empty/short text,
    voice-disabled, rapid-fire cancel, and clean stop/resolve.
  </Step>
  <Step title="Wave 35 — How I Know X relationship modal">
    A new relationship interview lands end-to-end. A seventh PersonSheet tab opens a lifted
    dialog running an 8–14 turn warm interview, backed by a new relationship table and three
    new endpoints, with per-turn persistence and on-mount transcript replay. The composer uses
    a record-then-send two-button mic.
  </Step>
</Steps>
