## What it is

[Arize AI](https://arize.com) is an AI observability platform for monitoring, evaluating, and debugging LLM applications. It provides tracing, evaluation, and analytics for LLM calls — tracking latency, token usage, cost, and quality metrics across production workloads.

## Why we use it

- **LLM tracing** — end-to-end visibility into every OpenRouter call across enrichment pipelines
- **Token usage analytics** — track input/output token counts per function to compute accurate cost-per-call
- **Evaluation** — automated quality checks on LLM outputs (hallucination detection, schema compliance)
- **Cost monitoring** — real-time spend tracking across models and pipelines
- **Latency profiling** — identify slow LLM calls and timeout patterns

## How we use it at Orbiter

Arize is our LLM observability layer for the enrichment pipelines. Every OpenRouter call from Xano is traced and logged, giving us per-function token usage averages that feed into the `average-token-use` skill for cost documentation. We use it to monitor model performance after swaps (e.g., migrating Film & TV from Gemini 2.5 Flash to Gemini 3.1 Flash Lite) and catch regressions in output quality.
