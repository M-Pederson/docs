New work enters the IMDB (Film & TV) waterfall through the two **get-add gates** — one per entity type. Each gate normalizes the `imdb_url`, dedups, ensures a stub row, then branches: `cascade_depth == 0` \+ `queue == false` invokes the orchestrator synchronously and returns `action: "enriched_direct"`; anything else upserts a queue row for the [queue crons](/guides/enrichment/waterfall/imdb-crons) to drain. Existing `master_person` rows can also enter through the central link writer, the person pipeline's Phase 8 attach/verify path, or the cross-waterfall expertise bridge (#13185). Architecture spec: [IMDB Waterfall overview](/guides/enrichment/waterfall/imdb-waterfall) · function roster: [IMDB Waterfall Functions](/guides/enrichment/waterfall/imdb-functions) · queue schemas: [IMDB Waterfall Tables](/guides/enrichment/waterfall/imdb-tables).

  Gate for any IMDB person URL. Normalizes the URL, dedups on a 30-day `enriched_at` window, ensures a stub, then either sync-enriches (depth 0 \+ `queue: false` → invokes `run-base-imdb-person-enrich` directly) or upserts the queue row. Seed-id propagation (v1.3): when no caller seed is passed, the just-created stub IS the seed. **v1.4 (2026-07-08):** preserves an explicit zero `seed_imdb_person_id` for title-origin cascades — Xano's `first_notempty` treats `0` as empty, so missing person seeds only default to the current stub when there is no title seed and the source is not `film_tv`. **Primary entry point for new work.**

The existing-row contract is normalized before branching (v1.8). **v1.11 (2026-07-08):** rejects non-title IMDb URLs before `film_tv` creation or queueing — award pages can contain honorary person URLs, which must not become `Film_TV` rows.

Title URLs do not use this path. **v1.6–v1.8 (2026-07-08):** Deep Research Bio IMDb links are subordinate to the verified attach path (v1.6); the canonical `link_url` is lowercased before dedupe and writes (v1.7); duplicate detection is scoped by owner \+ `link_url` so the same URL can belong to different people (v1.8).
