## What it is

[Knock](https://knock.app) is a customer engagement platform for sending product notifications across channels — email, in-app, push, SMS, Slack, and chat. Notification logic is defined as workflows that can be managed in the dashboard or as code via the Knock CLI.

## Why we use it

- **Multi-channel** — one workflow definition fans out to email, in-app, push, SMS, and Slack
- **Workflows as code** — `knock workflow pull/push` for version-controlled notification logic
- **Environment promotion** — dev → staging → production flow with `knock commit promote`
- **Batching & throttling** — built-in dedup, batching windows, and rate limiting
- **Agent toolkit** — `@knocklabs/agent-toolkit` exposes Knock APIs as MCP tools for Claude Code

## How we use it at Orbiter

Knock handles all user-facing notifications in Orbiter — enrichment completion alerts, collaboration notifications, and system updates. Workflows are authored in the Knock dashboard, pulled to code for review, and promoted through environments.
