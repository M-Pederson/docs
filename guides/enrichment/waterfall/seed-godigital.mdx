A concrete seeding plan for the **GoDigital Media Group** demo, using the [Music Industry Waterfall](/guides/enrichment/waterfall/music-industry-waterfall). GoDigital's music arm is **Cinq Music Group** — a distributor/label-services company with a deep Latin-urban catalog (Daddy Yankee, Farruko, Ñengo Flow, De La Ghetto, Justin Quiles, Jowell & Randy, Ken-Y …). That scene collaborates constantly, so the graph comes out **dense and recognizable** — ideal for a demo.

<Note>
The Music Industry Waterfall is **design-stage** (not built). To populate a demo graph now, use the **[fast-path ingest](#fast-path-for-the-demo)** — a one-off script over the seed set below — or build the minimal artist+label slice ([build steps 1–3](/guides/enrichment/waterfall/music-industry-waterfall#build-order-when-we-proceed)). Either way, **pre-seed ahead of the demo** — the 1 req/s MusicBrainz limit means a full catalog takes time to graph.
</Note>

## Seeding the full catalog

To graph **all** of GoDigital's catalog (not just the marquee names), drive the cascade from the **label**:

```text
1. Seed Cinq Music Group LLC (label, depth 0)
   └─ run-base-label-enrich browses release?label=fb742ab4 (paginated, 100/page, throttled)
       └─ for each release: queue its Release_Group + every credited artist (depth 1, T2)
           └─ each artist enriches (depth 1) → their recordings, albums, band members,
              collaborators (depth 2, T1–T2 only)
               └─ each recording → its Work (composition) + performers (depth 2)
2. Add the marquee artists as depth-0 seeds   (guarantees density on the names that matter)
3. Let the unified cron (process-queue-musicbrainz) drain to depth 2
```

**Sequencing & timing** (1 req/s is the bottleneck):

| Step | Approx API calls | Wall-clock @ 1 req/s |
|---|--:|--:|
| Label browse (≈32 releases / 100 per page) | ~3 | ~3 s |
| Marquee artists (9 × ~3 lookups each) | ~27 | ~30 s |
| Depth-1 catalog artists (~50 × ~3) | ~150 | ~3 min |
| Depth-2 recordings + works (~300 × 1) | ~300 | ~5–8 min |
| **Full curated catalog to depth 2** | **~500** | **~10–15 min** |

<Warning>
**Pre-seed; don't build live.** ~10–15 minutes to graph the curated catalog means you cannot build it on stage. Run the seed the **day before**, demo the populated graph, and optionally do **one live seed** (add a single new artist on camera — finishes in seconds) to show the pipeline working in real time.
</Warning>

### Fast-path for the demo

Because the waterfall isn't built yet, the quickest route to a populated demo graph is a **one-off ingest script** that bypasses queues/crons:

```text
for each seed MBID:
    lookup with the entity's inc= set        (throttled to 1 req/s, retry on 503)
    MERGE the node (Person:Music_Artist | Music_Group | Recording | Work | …)
    write its edges (MEMBER_OF, PERFORMED, COMPOSED, RELEASED_ON, …)
    enqueue depth+1 neighbors (cap: 20 albums, 50 recordings, T1–T2 only)
repeat until depth 2 reached
then: generate node + edge `description`s in a batch LLM pass (makes the graph readable)
```

This reuses the exact [node](/guides/enrichment/waterfall/music-nodes) and [edge](/guides/enrichment/waterfall/music-edges) specs — it's just an un-queued, single-run version of the orchestrators. ~150 lines, runnable the night before.

  </Card>

  </Card>
  <Card title="Provenance & freshness" icon="clock">
    Every node and edge carries `created_at` / `updated_at` / `description` — so the demo can show **auditable, dated, human-readable** relationships ("Farruko collaborated with Daddy Yankee on …"), not anonymous dots.
  </Card>
  <Card title="Geographic & genre clusters" icon="earth-americas">

  </Card>

<Note>
**Demo flow that lands:** open on the **Cinq Music** node → expand the roster → click **Daddy Yankee** → fan out his collaborators and **works** → jump to a shared **Work** to show the rights chain (composer + every recording + PROs) → end on the **A&R radar** (unsigned, highly-connected collaborators). Label → network → rights → opportunity, in five clicks.
</Note>

## Checklist before the demo

1. Pick the seed set above (adjust marquee names to whatever GoDigital wants front-and-center).
2. Run the [fast-path ingest](#fast-path-for-the-demo) (or the minimal built slice) **the day before**, to depth 2.
3. Run the `description` generation pass so every node/edge has a readable sentence.
4. Spot-check density: Daddy Yankee should show dozens of `COLLABORATED_WITH` / `PERFORMED` edges and link to Cinq via `RELEASED_ON`.
5. Pre-stage the five-click flow above; keep one fresh artist MBID handy for a **live** seed.
