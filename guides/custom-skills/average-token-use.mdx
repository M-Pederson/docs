# Average Token Use — Arize AI Trace Averaging

## Purpose

Close the loop on cost accuracy. The \`check-models\` skill already recalculates
\`Cost/Call\` from \`Avg Input Tokens × input price + Avg Output Tokens ×
output price\`, but only when both averages are populated. Today those columns
are mostly \`TBD\` because nothing automatically measures real-world token
usage. This skill pulls actual trace data from [Arize AI](https://arize.com),
averages over a rolling window, and writes the numbers back into the per-pipeline
\`model-summary.mdx\` files. After a run, \`check-models\` picks up the new
averages and refreshes \`Cost/Call\` automatically.

## How It Works

### Phase 1 — Enumerate functions

Read all five \`guides/enrichment/**/llm-system-prompts.mdx\` files. For every
\`## N. Title\` section:

- Extract \`function_id\` from \`**#NNNN**\`
- Extract \`function_name\` from the heading or function path
- Record \`prompt_index\` = 1-based order within the function (1st system prompt
  → 1, 2nd → 2, etc. — same convention as \`sync-system-prompts\`)
- Record section title and file path

Output: list of \`(function_id, prompt_index, section_title, file_path)\` pairs.

### Phase 2 — Query Arize AI traces (TODO until MCP is live)

For each \`(function_id, prompt_index)\` pair:

1. Call the Arize MCP to fetch trace data for that function over the time window
   (proposed default: last 30 days).
2. Extract per-span **input token count** and **output token count**.
3. Filter to the time window, compute the mean.
4. Record sample size for later filtering.

**Open questions pending MCP install:**
- Exact Arize tool names and API shape
- Orbiter's project ID and space ID in Arize
- How traces are tagged with function identity (direct \`function_id\` attribute
  vs span name vs metadata key)
- Exact field names for token counts (input_tokens, output_tokens, prompt_tokens, completion_tokens, etc.)
- Critical: are multi-prompt functions like \`#2537\` (short bio + long bio)
  distinguishable in traces, or do they aggregate together? If aggregate, we
  need a fallback strategy (e.g., use the aggregate average for both rows).

### Phase 3 — Map averages to doc rows

Read all five per-pipeline \`model-summary.mdx\` files. For each function table row:

1. Extract \`function_id\` from the first column (usually a code block).
2. For single-prompt functions: match by \`function_id\` alone.
3. For multi-prompt functions (currently only \`#2537\`): match by
   \`(function_id, row_order)\` — the 1st table row for that function matches
   \`prompt_index: 1\`, the 2nd row matches \`prompt_index: 2\`, etc.
   
   **TODO:** Confirm the exact rendering convention for multi-prompt functions on
   \`model-summary.mdx\`. Are they in separate rows? Separate table rows with the
   same function_id? Sub-rows or footnotes?

4. Look up the \`(function_id, prompt_index)\` average from Phase 2.
5. If found and sample size ≥ threshold (proposed: 50 calls), pair the row with
   its average.

Output: map of \`(file_path, row_id) → (old_avg_in, old_avg_out, new_avg_in, new_avg_out, sample_size)\`.

### Phase 4 — Report

Display a diff table:

guides/enrichment/.../model-summary.mdx | #2537 | 1 | last 30d | 127 | TBD | 4200 | TBD | 8900 | write
... | ... | ... | ... | ... | ... | ... | ... | ... | ...
\`\`\`

Separately, list rows that were **skipped** (sample size below threshold):

\`\`\`

- guides/enrichment/.../model-summary.mdx #NNNN row X: 18 samples, stays TBD
- ...
\`\`\`

Ask: **"Apply these updates?"** Nothing is written until the user confirms.

### Phase 5 — Apply

On confirmation, for each row with a paired average:

1. Open the relevant \`model-summary.mdx\` file.
2. Find the row (by row ID or table position).
3. Replace the \`Avg Input Tokens\` cell with \`new_avg_in\` (formatted per rule
   below).
4. Replace the \`Avg Output Tokens\` cell with \`new_avg_out\`.
5. If the row has an \`Updated\` column, set it to today's date.
6. Preserve all other cells unchanged.
7. Never touch \`Cost/Call\` — that belongs to \`check-models\`.

**Formatting:** round to the nearest integer token count (proposed).

> refresh Cost/Call?"

## Rules

- **Sample size threshold:** never write an average when sample size < threshold
  (proposed: 50 calls). Leave the row as \`TBD\` and report it. Tiny samples
  produce misleading \`Cost/Call\`.
- **Never recompute Cost/Call directly.** That formula lives in \`check-models\`
  and is deliberately not duplicated here. Hand off to \`check-models\` after
  averages are updated.
- **Always show the diff and confirm before writing.** No silent updates.
- **Preserve manual overrides.** If a user has marked a row as manually locked
  (convention TBD), skip it and note in the report.
- **Time window is configurable.** Default proposed: last 30 days. User can
  request a different window (e.g., "last 7 days", "since last deployment").
- **One sample-size threshold for all rows.** Don't vary it per function.

---

## Edge Cases

### Multi-prompt function traces aggregate together

If Arize AI doesn't distinguish between \`#2537\` short bio and long bio traces:
- Use the aggregate average for both rows (they'll have the same averages)
- Flag this in the report so the user knows they're correlated
- If the user wants separate averages, they'll need to add trace tags or
  metadata to Xano functions to distinguish them

**TODO:** confirm this scenario and codify the fallback strategy.

### Time window has no traces

If a function has no traces in the requested window, leave the row as \`TBD\`
and report it.

### Sample size is exactly at threshold

If sample size == 50, write the average. The threshold is inclusive.

### Arize API returns an error

Warn and skip that function. Don't invent data. Report it in the final summary.

### Row doesn't exist on the doc page

E.g., the function was removed from Xano but the row is still in the doc (stale).
Report it but don't crash. The user can manually remove it if needed.

### Updated timestamp doesn't exist on that file

Some \`model-summary.mdx\` files may not have an \`Updated\` column. Only set it
if present.
