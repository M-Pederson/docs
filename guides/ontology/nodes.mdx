**18 node types** across 8 logical groups.

Every node in the Orbiter graph carries the base label **`Entity`** plus one or more type-specific labels. Nodes are created during the person and company get-add pipelines, funding ingestion, IMDB credit processing, and expertise identification.

---

## People

Nodes representing individual humans. A Person node may carry additional labels based on their role.

### Person

The core node for any individual in the graph. Created by `mvp/node/add-person-node` (#2601).

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier (UUID v4) |
| `name` | string | Full display name |
| `display_label` | string | Always `"Person"` |
| `visibility` | boolean | Whether the node is visible in the UI |
| `name_embedding` | vecf32 | Semantic embedding of the person's profile |
| `created_at` | string | ISO 8601 timestamp |

<Accordion title="Real example">
```json
{
  "uuid": "cf64fc2a-4b3e-4ff9-99dc-21d7cffebf01",
  "name": "Josh Diamond",
  "display_label": "Person",
  "visibility": true,
  "created_at": "2025-01-15T10:32:45.000Z"
}
```
</Accordion>

### Angel

An additional label applied to Person nodes identified as angel investors. Set by `mvp/node/update-person-node` (#2758) when `is_angel = true`.

**Labels:** `Entity:Person:Angel`

No additional properties beyond Person — `Angel` is a label-only modifier used for filtering and traversal.

<Accordion title="Real example">
```json
{
  "uuid": "a1b2c3d4-5678-9abc-def0-123456789abc",
  "name": "Jared Leto",
  "display_label": "Person",
  "visibility": true
}
```
_Node labels: `Entity`, `Person`, `Angel`_
</Accordion>

## Organizations

Nodes representing companies, schools, VC firms, and other organizations. All share the same creation function (`mvp/node/add-company-node` #2602) with a dynamic label based on type.

### Company

The default organization node type. Represents businesses, startups, and corporations.

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `name` | string | Company display name |
| `company_domain` | string | Primary web domain |
| `alternative_domains` | string | Comma-separated list of alternate domains |
| `display_label` | string | `"Company"` |
| `visibility` | boolean | UI visibility flag |
| `logo` | string | URL to company logo |
| `name_embedding` | vecf32 | Semantic embedding |
| `created_at` | string | ISO 8601 timestamp |

<Accordion title="Real example — Frame.io">
```json
{
  "uuid": "4845ea5b-5c8a-4ccd-aba8-f7ef0259c496",
  "name": "Frame.io",
  "company_domain": "frame.io",
  "display_label": "Company",
  "visibility": true,
  "logo": "https://img.logo.dev/frame.io?token=pk_Twvl42WzTX2-ySNbnZKlSA"
}
```
</Accordion>

### School

A Company node with the `School` label. Applied when `is_school = true` in the master company record.

**Labels:** `Entity:School`

Same properties as Company. Identified by `display_label: "School"`.

<Accordion title="Real example — CUNY / Hunter College">
```json
{
  "uuid": "9f3a2b1c-4d5e-6f7a-8b9c-0d1e2f3a4b5c",
  "name": "The City University of New York",
  "company_domain": "cuny.edu",
  "display_label": "School",
  "visibility": true
}
```
</Accordion>

### VC_Firm

A Company node with the `VC_Firm` label. Applied during funding ingestion when an institutional investor is identified (`mvp/fundable/resolve-investors-edges` #12702).

**Labels:** `Entity:Company:VC_Firm`

Same properties as Company. The `VC_Firm` label is additive — the node retains its `Company` label.

<Accordion title="Real example — Accel">
```json
{
  "uuid": "b2c3d4e5-6f7a-8b9c-0d1e-2f3a4b5c6d7e",
  "name": "Accel",
  "company_domain": "accel.com",
  "display_label": "Company",
  "visibility": true
}
```
_Node labels: `Entity`, `Company`, `VC_Firm`_
</Accordion>

### Organization

A generic organization node. Used for non-company entities like certification bodies, volunteer organizations, and honor issuers. Can be created standalone (`mvp/resolve/resolve-edges-volunteering` #4575) or applied as an additional label to an existing Company node (`mvp/resolve/resolve-edges-certifications` #12578).

**Labels:** `Entity:Organization:Company` (when added to existing Company) or `Entity:Organization:Company` (when created new by volunteering)

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `name` | string | Organization name |
| `createdAt` | integer | Unix timestamp (ms) |

<Accordion title="Real example — San Francisco Business Times">
```json
{
  "uuid": "d4e5f6a7-8b9c-0d1e-2f3a-4b5c6d7e8f9a",
  "name": "San Francisco Business Times",
  "createdAt": 1704067200000
}
```
</Accordion>

## Location

Geographic hierarchy nodes forming a `City -> Region -> Country` chain. Created by `mvp/node/primary-location-nodes-edges_v2` (#4690) and `mvp/node/location-nodes-edges-company` (#4528).

### Country

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `name` | string | Country name (MERGE key) |
| `display_label` | string | `"Country"` |
| `flag` | string | Emoji flag |
| `alpha_2_code` | string | ISO 3166-1 alpha-2 code |
| `population` | integer | Population count |
| `capital_city` | string | Capital city name |
| `currency_code` | string | ISO 4217 currency code |
| `currency_name` | string | Currency display name |
| `summary` | string | Brief description |
| `name_embedding` | vecf32 | Semantic embedding |

<Accordion title="Real example — United States">
```json
{
  "uuid": "f1a2b3c4-d5e6-7f8a-9b0c-1d2e3f4a5b6c",
  "name": "United States",
  "display_label": "Country",
  "alpha_2_code": "US",
  "capital_city": "Washington, D.C.",
  "currency_code": "USD",
  "currency_name": "United States Dollar"
}
```
</Accordion>

### Region

Represents a state, province, or administrative region. Uses a composite `region_key` for deduplication.

**Labels:** `Entity:Region` (person pipeline) or `Entity:State` / `Entity:Region` (company pipeline — dynamic via `{{$var.stateOrRegion|capitalize}}`)

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `name` | string | Region display name |
| `region_key` | string | Dedup key: `"region_name::country_name"` (lowercased) |
| `display_label` | string | `"State"` or `"Region"` |
| `country` | string | Parent country name |
| `summary` | string | Brief description |
| `name_embedding` | vecf32 | Semantic embedding |

<Accordion title="Real example — New York">
```json
{
  "uuid": "a2b3c4d5-e6f7-8a9b-0c1d-2e3f4a5b6c7d",
  "name": "New York",
  "region_key": "new york::united states",
  "display_label": "State",
  "country": "United States"
}
```
</Accordion>

### City

Uses a composite `city_key` for deduplication across regions and countries.

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `name` | string | City display name |
| `city_key` | string | Dedup key: `"city_name::region_name::country_name"` (lowercased) |
| `display_label` | string | `"City"` |
| `region` | string | Parent region name |
| `country` | string | Parent country name |
| `summary` | string | Brief description |
| `name_embedding` | vecf32 | Semantic embedding |

<Accordion title="Real example — New York City">
```json
{
  "uuid": "b3c4d5e6-f7a8-9b0c-1d2e-3f4a5b6c7d8e",
  "name": "New York",
  "city_key": "new york::new york::united states",
  "display_label": "City",
  "region": "New York",
  "country": "United States"
}
```
</Accordion>

## Film & TV

Nodes representing film/TV titles and their awards. Created by `mvp/node/add-film-tv-node` (#2320) and `mvp/edge/add-node-edge-imdb-award` (#2334).

### Film_TV

The base node for any IMDB title. Carries an additional dynamic type label based on the title type.

**Possible label combinations:**
- `Entity:Film_TV:Movie`
- `Entity:Film_TV:TV_Series`
- `Entity:Film_TV:TV_Movie`
- `Entity:Film_TV:TV_Mini_Series`
- `Entity:Film_TV:Short_Film`
- `Entity:Film_TV:Documentary`
- `Entity:Film_TV:TV_Special`

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `title` | string | Film/TV title |
| `imdb_url` | string | IMDB title URL (MERGE key for award flow) |
| `type` | string | Title type (`movie`, `tvSeries`, etc.) |
| `description` | string | Plot synopsis |
| `tagline` | string | Marketing tagline |
| `genres` | string | Comma-separated genre list |
| `year` | string | Release year |
| `imdb_rating` | string | IMDB user rating |
| `running_time` | string | Runtime in minutes |
| `mpaa_rating` | string | Content rating (PG-13, R, etc.) |
| `countries` | string | Comma-separated production countries |
| `languages` | string | Comma-separated languages |
| `image_url` | string | Poster image URL |
| `source` | string | Data source identifier |
| `name_embedding` | vecf32 | Semantic embedding |

<Accordion title="Real example — Edge of Tomorrow">
```json
{
  "uuid": "b7700ba7-7fb1-4fb7-a5d7-08fe61c96d57",
  "title": "Edge of Tomorrow",
  "imdb_url": "https://www.imdb.com/title/tt1631867",
  "type": "Movie",
  "description": "A man fighting in a war against aliens must relive the same day every time he dies, learning from each experience to become a better soldier.",
  "tagline": "Die. Repeat. Save humanity.",
  "genres": "Action,Adventure,Sci-Fi",
  "year": "2014",
  "imdb_rating": "7.9",
  "running_time": "113",
  "mpaa_rating": "PG-13",
  "countries": "United States,Canada,India,Australia",
  "languages": "English"
}
```
_Node labels: `Entity`, `Film_TV`, `Movie`_
</Accordion>

### Film_TV_Award

Represents a specific award nomination or win for a film/TV title. Carries an additional `Winner` or `Nominee` label.

**Label combinations:**
- `Film_TV_Award:Winner`
- `Film_TV_Award:Nominee`

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `year` | integer | Award year |
| `award_type` | string | Awarding body name |
| `award_description` | string | Specific category |
| `nominee_or_winner` | string | `"Winner"` or `"Nominee"` |
| `film_tv_imdb_url` | string | IMDB URL of the associated title |
| `project_title` | string | Title of the film/TV project |
| `description` | string | Full award citation text |
| `shared_with` | string | Names of co-recipients |
| `display_label` | string | `"Film_TV_Award"` |
| `name_embedding` | vecf32 | Semantic embedding |

<Accordion title="Real example — Edge of Tomorrow, Saturn Award nomination">

  "nominee_or_winner": "Nominee",

</Accordion>

<Accordion title="Real example — Cannes Film Festival">

## Expertise

Nodes representing domains and sub-domains of professional expertise. Created by `mvp/node/sync-domain-expertise-nodes` (#12663) and `mvp/expertise/resolve-person-expertise` (#12668).

### DomainExpertise

Top-level expertise categories. Synced from the `domain_expertise` table.

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `name` | string | Domain name |
| `description` | string | Domain description |
| `category` | string | Expertise category grouping |
| `display_label` | string | `"DomainExpertise"` |
| `name_embedding` | vecf32 | Semantic embedding |
| `expertise_embedding` | vecf32 | Expertise-specific embedding (same as name_embedding) |

<Accordion title="Real example — Software Engineering">
```json
{
  "uuid": "e2b4bd5c-1e5f-4781-be02-9b80916bbf5c",
  "name": "Software Engineering",
  "description": "Software Engineering: systematic design, coding, testing, DevOps, agile, cloud, microservices, algorithms, databases, APIs, CI/CD, version control, scalable reliable systems.",
  "category": "engineering and software development categories",
  "display_label": "DomainExpertise"
}
```
</Accordion>

### SubDomainExpertise

Specific expertise areas within a domain. Can be synced from the `sub_domain_expertise` table or created on-the-fly during person expertise resolution.

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier |
| `name` | string | Sub-domain name |
| `description` | string | LLM-generated description |
| `parent_domain` | string | Parent domain name (when synced from table) |
| `display_label` | string | `"SubDomainExpertise"` |
| `source` | string | `"llm_identified"` when created during resolution |
| `name_embedding` | vecf32 | Semantic embedding |
| `expertise_embedding` | vecf32 | Expertise-specific embedding |

<Accordion title="Real example — Web Development">
```json
{
  "uuid": "4414d987-a22e-438f-a450-21faeb440e8e",
  "name": "Web Development",
  "description": "Web development covers front-end & back-end coding, responsive design, APIs, databases, and deployment to create fast, secure, interactive websites and web apps using JavaScript, HTML, CSS, and modern frameworks.",
  "display_label": "SubDomainExpertise"
}
```
</Accordion>
