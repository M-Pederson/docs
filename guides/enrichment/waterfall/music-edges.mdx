The complete edge catalog for the [Music Industry Waterfall](/guides/enrichment/waterfall/music-industry-waterfall). Weights follow the canonical [Edge Weights](/guides/ontology/edge-weights) scale (**lower = stronger**; hub-node edges sit Weak or below; never sub-1). See [Music Nodes](/guides/enrichment/waterfall/music-nodes) for the node side.

<Note>
**`Artist` in a pattern below = `Person:Music_Artist` (a person) *or* `Music_Group` (a band).** Either can perform, release, or be credited. `MEMBER_OF` is the exception — it is always `(Person:Music_Artist)→(Music_Group)`. Person-only roles (`PLAYED_ON`, `PRODUCED`, `ENGINEERED`) use `Person:Music_Artist` explicitly.
</Note>

<Note>
**Every edge carries the standard envelope.** At a minimum, **all** edges below are created with `created_at`, `updated_at`, and `description` — plus `uuid` and `weight`. Edge-specific parameters are listed per edge. The natural-language `description` is populated by a post-write LLM pass (see [Edge descriptions](#edge-descriptions)); it is `null` between write and generation.
</Note>

## Standard edge envelope

Every music edge is written with this property set on the relationship variable `r`. `created_at` uses `coalesce` so re-enrichment never resets it; `updated_at` is stamped on every write (FalkorDB `timestamp()` = epoch-millis).

```cypher
MERGE (a)-[r:EDGE_TYPE]->(b)
SET r.uuid                    = coalesce(r.uuid, randomUUID()),
    r.weight                  = <band weight>,
    r.created_at              = coalesce(r.created_at, timestamp()),
    r.updated_at              = timestamp(),
    r.description             = <generated sentence | null>,
    r.description_generated_at = <timestamp | null>,
    // … edge-specific parameters …
```

| Field | Type | Required | Notes |
|---|---|:--:|---|
| `uuid` | string | ✅ | `randomUUID()`, stable across re-runs via `coalesce` |
| `weight` | int | ✅ | from the [edge-weights scale](/guides/ontology/edge-weights) |
| `created_at` | timestamp | ✅ | `coalesce(r.created_at, timestamp())` — set once |
| `updated_at` | timestamp | ✅ | `timestamp()` — bumped every write |
| `description` | string | ✅ | one-sentence NL summary; `null` until the description pass runs |
| `description_generated_at` | timestamp | ✅ | guard for the backfill (mirrors the IMDB `WORKED_ON` pattern); `null` until generated |

## Identity bridges

| Edge | Pattern | Weight | Edge-specific parameters | Sample `description` |
|---|---|:--:|---|---|
| `[:SAME_AS]` | `(Music_Label)→(Company)` | **1** | `match_method` (str) · `confidence` (float 0–1) | "Sub Pop Records is the same entity as the Orbiter company profile for Sub Pop." |

<Warning>
`SAME_AS` now applies to **labels only** (`Music_Label`→`Company`). **Person artists are not bridged** — a `Person:Music_Artist` *is* an Orbiter `Person` node, so it reconciles to an existing person by **node dedup/MERGE** (name + ISNI + URLs), not by an edge. `SAME_AS` is written by a **separate post-ingest reconciliation pass** — never at ingest (decided 2026-06-11) — and its `description` records *how* the match was made (`match_method`, `confidence`) for auditability. See [resolved design decisions](/guides/enrichment/waterfall/music-industry-waterfall#resolved-design-decisions-2026-06-11).
</Warning>

## Edge descriptions

The `description` + `description_generated_at` fields mirror the IMDB pipeline's `WORKED_ON` edge-description pattern ([#13016 / #13017 / #13018](/guides/enrichment/waterfall/imdb-waterfall-plan#tooling-operator-invoked-not-pipeline-wired)). The music equivalents were **built 2026-06-12** — and unlike the IMDB/person pipelines, which ship one trio per edge type, **a single generic trio covers all music edge types**, keyed on `e.uuid`:

<Card icon="wrench">

```
mb/edge/create-music-edge-description — 13095
mb/edge/write-music-edge-description — 13096
mb/tool/backfill-music-edge-descriptions — 13097
```

- **create** — given an edge's structured properties + both endpoint node names, `openai/gpt-oss-120b` (temp 0.3) returns a single factual 15–30-word sentence.
- **write** — sets `e.description` + `e.description_generated_at` (ISO) on the edge via Cypher, matched on `e.uuid`.
- **backfill** — drains edges where `description_generated_at IS NULL`, selected by edge type + `labels()` membership, in `try_catch` batches of 1–100 (default 10); idempotent via the guard. One generic drainer parameterised by edge type keeps the surface small.

</Card>

Driven by **Task 181** `backfill-music-edge-descriptions` (every 300 s, 20 edges/tick — shipped **INACTIVE**) and the operator endpoints `POST tool/backfill-edge-descriptions` (#8645) / `GET tool/peek-edge-descriptions` (#8646) in the `musicbrainz` API group — see [Music Waterfall Functions](/guides/enrichment/waterfall/music-functions#llm-edge-descriptions). Sandbox smoke test processed 11/11 edges with 0 pending; a real generated sample:

> "Kurt Cobain was an original member of the US grunge band Nirvana from 1987 until April 5 1994."

<Note>
Because `description` is generated *after* the structural write, every edge is created with `description = null` and `description_generated_at = null`, then filled by the backfill. The two fields are part of the [standard envelope](#standard-edge-envelope) so they exist on day one — the generator only populates them.
</Note>
