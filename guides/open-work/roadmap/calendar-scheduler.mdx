## Research: Open Source Scheduler Packages & Frameworks

The following open-source packages support the key features required for Orbiter's calendar and scheduling module:

### Top-tier Options

**FullCalendar** (Production-ready)
- **License:** Open source & commercial
- **Features:** Multiple view types (month, week, day, agenda), drag-and-drop event management, timezone support, recurring events
- **React Integration:** Official `@fullcalendar/react` package
- **Best for:** Feature-complete, enterprise-grade calendar with strong team collaboration capabilities
- **Team Scheduling:** Strong support via resource views and event management

**CalendarKit** (Feature-rich)
- **License:** Free & open source
- **Features:** Month, week, day views with agenda option, event creation, drag-and-drop (Pro), resource views (Pro)
- **React Integration:** Native React component
- **Best for:** Modern, lightweight alternative with beautiful UI
- **Team Scheduling:** Basic in Free tier, robust in Pro tier

**DayPilot Lite** (Specialized)
- **License:** Open source with commercial versions
- **Features:** Daily, weekly, monthly calendar views, resource scheduling, drag-and-drop
- **React Integration:** Dedicated React calendar component
- **Best for:** Enterprise resource scheduling and team calendars
- **Team Scheduling:** Built-in resource and team views

### Community & Lightweight Options

**React Big Schedule** (GitHub-based)
- **License:** Open source
- **Features:** Drag-and-drop scheduler, granular time views, resource planning
- **React Integration:** Native React component
- **Best for:** Custom resource scheduling workflows
- **Team Scheduling:** Resource-centric approach

**CalendarCN** (Modern UI)
- **License:** Open source (shadcn/ui + Tailwind CSS based)
- **Features:** Week view, dark mode, event colors, beautiful design
- **React Integration:** Built for React
- **Best for:** Design-first implementations aligned with modern UI frameworks
- **Team Scheduling:** Week-view optimized for viewing multiple calendars

## Key Features

### Month View
A traditional calendar grid showing all events for a month at a glance. Supports color-coding by category, event preview on hover, and drill-down to day/week detail.

**Recommended implementations:**
- FullCalendar's month view with drag-and-drop support
- CalendarKit's month view for lightweight alternative
- CalendarCN for design-first integration with Tailwind

### Agenda View
A list-based presentation of upcoming events, sorted chronologically. Ideal for quick scanning of what's coming up and identifying scheduling conflicts.

**Recommended implementations:**
- FullCalendar's list/agenda view (highly customizable)
- CalendarKit's agenda view with timezone awareness
- DayPilot's timeline view for resource-grouped agendas

### Robust Team Scheduling
Support for viewing and coordinating schedules across multiple team members, including:
- Multi-person calendar overlay
- Resource availability visualization
- Conflict detection across team calendars
- Group event creation and RSVP management

**Recommended implementations:**
- FullCalendar with resource views and drag-and-drop constraints
- DayPilot's dedicated team/resource scheduling views
- CalendarKit Pro for team-level features

## Recommendation

**FullCalendar** is the recommended primary choice because:
1. **Mature ecosystem** — battle-tested in production across thousands of apps
2. **React-native support** — official `@fullcalendar/react` package
3. **Complete feature set** — month, week, day, agenda views all built-in
4. **Team collaboration** — strong resource views and event constraints
5. **Extensibility** — plugin architecture for custom requirements
6. **Support** — active community and commercial support available

**CalendarKit** is a strong alternative if lightweight bundle size or design customization is a priority.
