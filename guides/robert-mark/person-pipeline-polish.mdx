## TL;DR

`#12553 mvp/get-add/master-person` is the canonical entry point for getting (or creating) a `master_person`. It calls People Data Labs first; if PDL returns a 404 it falls back to **EnrichLayer**. The fallback is intentionally lightweight ‚Äî only a LinkedIn URL and avatar are pulled into the initial flow. The full EnrichLayer payload is stashed on `person_enrich_data.enrich_layer_data` and processed downstream by `#12554 / run-base-person-enrich`.

RocketReach was removed entirely in v3.21.

## Why two EnrichLayer entry points

The two helpers return the **same profile shape**, so downstream code is identical ‚Äî only the lookup key differs.

## What #12553 extracts vs. defers

Only two fields are consumed by #12553 itself:

| EL field            | Used for                                          |
| ------------------- | ------------------------------------------------- |
| `public_identifier` | Construct `https://www.linkedin.com/in/{id}` URL  |
| `profile_pic_url`   | Avatar via `#237 mvp/avatar/replace-avatar`       |

Everything else (`experiences`, `education`, `summary`, `headline`, recommendations, activities, etc.) is **persisted but not processed**. It lands on:

</Card>

## Source attribution

The `source` text written to `master_link`, `master_email`, and `master_avatar` follows this routing inside #12553:

| Source value         | When                                                                                  |
| -------------------- | ------------------------------------------------------------------------------------- |
| `"People Data Labs"` | URL came from PDL `profiles[]`, OR caller-supplied URL is also in PDL response       |
| `"Enrich Layer"`     | URL or avatar came from EL fallback (`#12734` or `#12612`)                            |
| `$input.source`      | Caller-supplied URL not confirmed by any enrichment provider                          |
| `null`               | Cascade ran but nothing resolved (cascade_depth > 0 or both APIs returned nothing)    |

Order matters because `create-master-link` dedups on URL. EL links are written **first** in Sections 7 + 10, then `$inputLinks`, then `$pdlLinks` ‚Äî so EL wins attribution when a URL appears in multiple sources.

`$inputEmailSource` is the email-specific override:

- PDL initial status ‚âÝ 404 ‚Üí `"People Data Labs"`
- PDL was 404 + EL returned a linkedin ‚Üí `"Enrich Layer"`
- Otherwise ‚Üí `null`

This only affects email writes; link/phone writes still use `$input.source` (with the v3.20 per-URL override for links).

## hold_downstream for QA

Pass `hold_downstream: true` to skip both the direct `run-base-person-enrich` async dispatch and the queue path. A `qa_passed` row is logged to `log_crash` noting the skip. Use this to run #12553 in isolation while step-debugging ‚Äî the master_link triggers (Twitter / LinkedIn scrapers) will still fire though, since they're table-level.

## Where the work moves next

After #12553 returns, `#12554 / run-base-person-enrich` does the heavy lifting:

- Read `person_enrich_data.enrich_layer_data` and process experiences, education, biography, recommendations.
- Backfill work-experience joins, education joins.
- Run #13031 (Exa expand), #13033 (was RocketReach phase ‚Äî now retired), and downstream avatar / link enrichment phases.

The `#12554` migration to mirror this RR ‚Üí EL rewrite is pending and tracked separately.
