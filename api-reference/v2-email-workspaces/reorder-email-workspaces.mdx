## Overview

Bulk-update the display position of multiple email workspaces at once. Accepts an array of `{id, position}` pairs. Auth user must own every workspace in the list.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login or signup.
</ParamField>

## Request Body

<ParamField body="positions" type="json" required>
  Array of objects: `[{id: int, position: int}, ...]`. Each entry sets the display order for that workspace.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the reorder was successful.
</ResponseField>

<RequestExample>
```bash cURL
curl -X PATCH "https://xh2o-yths-38lt.n7c.xano.io/api:dUJtZDlP/email-workspaces/reorder" \
  -H "Content-Type: application/json" \
  -H "X-Data-Source: YOUR_DATA_SOURCE" \
  -H "X-Branch: v1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "positions": [
      {"id": 1, "position": 0},
      {"id": 2, "position": 1}
    ]
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "success": true
}
```
</ResponseExample>
