## Standard edge envelope

Every music edge is written with this property set on the relationship variable `r`. `created_at` uses `coalesce` so re-enrichment never resets it; `updated_at` is stamped on every write (FalkorDB `timestamp()` = epoch-millis).

```cypher
MERGE (a)-[r:EDGE_TYPE]->(b)
SET r.uuid                    = coalesce(r.uuid, randomUUID()),
    r.weight                  = <band weight>,
    r.created_at              = coalesce(r.created_at, timestamp()),
    r.updated_at              = timestamp(),
    r.description             = <generated sentence | null>,
    r.description_generated_at = <timestamp | null>,
    // â€¦ edge-specific parameters â€¦
```

|---|---|:--:|---|

## Edge descriptions

The `description` + `description_generated_at` fields mirror the IMDB pipeline's `WORKED_ON` edge-description pattern ([#13016 / #13017 / #13018](/guides/enrichment/waterfall/imdb-waterfall-plan#tooling-operator-invoked-not-pipeline-wired)). Proposed music equivalents:

<Card icon="wrench">

```
mb/enrich/create-music-edge-description-from-graph   (proposed)
mb/enrich/write-music-edge-description-to-graph       (proposed)
tool/backfill-music-edge-descriptions                 (proposed)
```

- **create** â€” given an edge's structured properties + both endpoint node names, an LLM (e.g. `google/gemini-3.1-flash-lite`) returns a single-sentence `description`. Keyed on `(source_uuid, target_uuid, type)`.
- **write** â€” sets `r.description` + `r.description_generated_at = timestamp()` on the edge via Cypher. No-op if either UUID is null.
- **backfill** â€” drains edges missing `description_generated_at` in `try_catch` batches; idempotent via the guard. One generic drainer parameterised by edge type (rather than one-per-type as IMDB did) keeps the surface small.

</Card>

<Note>
Because `description` is generated *after* the structural write, every edge is created with `description = null` and `description_generated_at = null`, then filled by the backfill. The two fields are part of the [standard envelope](#standard-edge-envelope) so they exist on day one â€” the generator only populates them.
</Note>
