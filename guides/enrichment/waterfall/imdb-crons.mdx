Two scheduled tasks drain the IMDB queue tables — one single-row tick per queue, every 600 s, priority-ordered `priority_tier asc, cascade_depth asc, id asc`. Two more inactive tasks exist around the waterfall: a one-row-per-tick seeder (#172) and the poster backfill wrapper (#178). Queue schemas: [IMDB Waterfall Tables](/guides/enrichment/waterfall/imdb-tables) · the orchestrators they dispatch: [IMDB Waterfall Functions](/guides/enrichment/waterfall/imdb-functions#orchestrators-full-enrichment-pipeline-per-entity).

<Card icon="clock-rotate-left">
  ```text
  process-queue-imdb-person — Task #159
  ```

  **v1.1 (2026-04-22):** single-row tick on `queue_imdb_person`. Priority-order pick: `sort = {priority_tier: "asc", cascade_depth: "asc", id: "asc"}`. Lock → dispatch `run-base-imdb-person-enrich` → delete on success, release lock \+ `log_crash` on failure. Schedule: every 600s (10 min).
</Card>

<Card icon="clock-rotate-left">
  ```text
  process-queue-imdb-title — Task #160
  ```

  **v1.1 (2026-04-22):** single-row tick on `queue_imdb_title`. Same priority ordering. Lock → dispatch `run-base-imdb-title-enrich` → delete on success, release lock \+ `log_crash` on failure. Schedule: every 600s (10 min).
</Card>

<Card icon="clock-rotate-left">
  ```text
  task/uta-cinematographers-ingest — Task #172
  ```

  **Inactive seeder.** Processes one `uta_cinematographers` row per run: picks the first `processing = false AND complete = false` row, marks it processing, calls the person gate `mvp/imdb/get-add/imdb-person` #12873 synchronously (`queue: false`, `cascade_depth: 0`, `source_function: "task/uta-cinematographers-ingest"`), then marks it complete. On error, resets `processing` so the row retries next run.
</Card>

<Card icon="clock-rotate-left">
  ```text
  backfill-film-tv-posters — Task #178
  ```

  **Inactive backfill wrapper.** Calls `mvp/imdb/backfill-film-tv-posters` #13047 with `batch_size: 10` every 120 s — walks `film_tv` rows missing a GCS poster, runs `save-imbd-poster` #13044 per row, and writes the result back to `image_url`.
</Card>

<Warning>
  **All four tasks are `active = false` live.** Flip #159 / #160 to `true` in the Xano UI before enabling production enrichment — they were built deliberately inert pending the full waterfall sign-off.
</Warning>
