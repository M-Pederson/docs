## parse-from — #2126

Parses and logs an email activity when the email is sent FROM the user's account. Iterates through every TO, CC, and BCC participant on the outbound message and, for each one, resolves or creates the contact's identity (`check-identity-email`, `get-add/master-person`), copies them into the user's address book (`copy-to-my-person`), writes an `email_activity` record with `activity_type_id = 2`, increments `total_email_activity` on the `my_person` row, and sets the primary email. Marketing-flagged contacts are skipped entirely. Participant count is calculated once up front via `calculate_participant`.

## parse-to — #2129

Parses and logs an email activity when the email is sent TO the user's account. First resolves the sender (FROM) and logs an inbound `email_activity` with `activity_type_id = 1`. Then iterates through additional TO recipients (skipping the user's own grant email) and CC/BCC participants, logging outbound activity records (`activity_type_id = 2` for TO, `3` for CC, `4` for BCC). For unknown senders, it also triggers `email-signature-parse-process` asynchronously to extract contact details. Uses `check-identity-email-to` for identity resolution and a `$logActvity` flag to conditionally suppress duplicate activity logging when the sender was newly created.

  input {
    json participants?
    int user_id?
  }

  stack {
    db.get nylas_grant {
      field_name = "grant_id"
      field_value = $input.participants.object.grant_id
      output = ["id", "user_id", "grant_id", "email_account", "ai_enabled"]
      addon = [
        {
          name  : "user:user_id"
          output: ["master_person_id"]
          input : {user_id: $input.user_id}
        }
      ]
    } as $nylasGrant

    // set up variables
    group {
      stack {
        var $grantID {
          value = $input.participants|get:"grant_id":null
        }

        var $logActvity {
          value = true
        }

        // create TO array variable
        var $to {
          value = $input.participants.object|get:"to":[]
        }

        var $cc {
          value = $input.participants.object|get:"cc":[]
        }

        var $bcc {
          value = $input.participants.object|get:"bcc":[]

## parse-cc — #2132

Parses and logs an email activity when the user's account is CC'd or BCC'd on the message. Looks up the Nylas grant to identify the user's own email account, then processes all four participant lists -- TO, FROM, CC, and BCC -- skipping the user's own address in each loop. For TO and CC participants it writes `activity_type_id = 2` (sent) or `3` (cc'd); for FROM it writes `activity_type_id = 1` (received); for BCC it writes `activity_type_id = 4`. Each participant is resolved or created via `check-identity-email` and `get-add/master-person`, copied into the user's address book, and triggers an async `email-signature-parse-process` call. Marketing-flagged contacts are skipped.

      output = ["id", "user_id", "grant_id", "email_account", "ai_enabled"]
      addon = [
        {
          name  : "user:user_id"
          output: ["id", "user_login_name"]

        }

    } as $nylasGrant

    // set up variables
    group {
      stack {
        var $grantID {
          value = $input.participants|get:"grant_id":null
        }

        var $to {

        }

        var $from {
          value = $input.participants.object|get:"from":[]
        }

        var $cc {
          value = $input.participants.object|get:"cc":[]
        }

        var $bcc {
          value = $input.participants.object|get:"bcc":[]
        }

    }

    function.run "mvp/activity/calculate_participant" {
      input = {
        to  : $input.participants.object.to
        from: $input.participants.object.from
        cc  : $input.participants.object.cc ?: []
        bcc : $input.participants.object.from ?: []
      }
    } as $participant_count

    // process TO
    foreach ($to) {
      each as $toItem {
        conditional {
          if ($toItem.email == $nylasGrant.email_account) {
            continue
          }

          else {

              stack {
                function.run "mvp/identity/check-identity-email" {
                  input = {email_address: $toItem.email, user_id: $input.user_id}
                } as $identityemail

                  stack {
                    conditional {
                      if ($identityemail|get:"marketing":null) {
                        continue
                      }
                    }
                  }
                }

                    group {

                          field_name = "id"
                          field_value = $identityemail.master_person_id
                          output = ["id", "name"]
                        } as $masterPerson

                        db.add email_activity {
                          data = {
                            created_at        : "now"
