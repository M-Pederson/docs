# Company Go/AlloyDB Rebuild Spec

## 1. Executive Summary

This page is the implementation contract for rebuilding the Company enrichment waterfall in Go on GCP, AlloyDB, and the standalone Orbiter Universe graph. It is extracted from:

- The current Company waterfall docs, especially [Company Waterfall](/guides/enrichment/waterfall/company-waterfall), [Entry Points](/guides/enrichment/waterfall/company-entry-points), [Functions](/guides/enrichment/waterfall/company-functions), [Tables](/guides/enrichment/waterfall/company-tables), [Nodes](/guides/enrichment/waterfall/company-nodes), [Edges](/guides/enrichment/waterfall/company-edges), and [Crons](/guides/enrichment/waterfall/company-crons).
- Read-only Xano workspace 3 exports for 61 Company-related functions under `/Users/markpederson/work/rebuild_specs/company/functions/`.
- Read-only Task #127 metadata from Xano MCP, confirming `process-company-queue` is `active: false`.

The Go rebuild must preserve the live data contract: procedural dedup, whole-envelope payload storage, queue tier/depth semantics, graph node/edge shape, provider gates, LLM parse/repair behavior, and non-recursive depth policy. Flow changes are allowed only after Mark approval.

This spec does not claim parity is proven. Golden fixtures have not been captured or run.

## 2. How to Use This Spec

Use this page as the authoritative build contract for the Go port. The existing Company waterfall pages remain canonical narrative reference and detailed source catalogs; they are not the implementation checklist.

Implement the shared Go foundations first, then the Company pipeline on top of them. Any proposed change to orchestration order, dedup keys, graph shape, queue semantics, provider request contracts, or historical bug/fix decisions needs Mark approval before implementation.

When in doubt, preserve current live behavior unless this page explicitly marks a behavior as `Fix in Go` or `Xano artifact - do not reimplement`.

## 7. End-to-End Worked Example

### Example: `resend.com` seed company

Input:

```json
{
  "domain": "resend.com",
  "profile_url": "https://www.linkedin.com/company/resend",
  "company_name": "Resend",
  "source": "User Input",
  "cascade_depth": 0,
  "priority_tier": 1,
  "queue": false,
  "hold_downstream": false,
  "return_fast": false
}
```

Expected control flow:

1. Entry gate rejects placeholders, normalizes domain, resolves redirects, and checks profile URL in `master_link`.
2. Kill switch reads `environment_variables.name = kill_switch_company`; if active and no local match, return a null shell with no provider calls.
3. Fanout governor is not active for this seed; create is allowed.
4. LLM preflight runs #12974 unless `return_fast` is true. It stores the whole response envelope, not only `data`.
5. Post-LLM dedup checks discovered profile links and domains.
6. Fundable lookup #12698 reads the local mirror; miss stores the literal JSON string `"no_data"`.
7. Fresh create writes `master_company`, `company_enrich_data`, relational fanout from #12989, initial Company graph node via #2602, and Fundable fanout if present.
8. Depth 0, not queue, not hold: dispatch #12992 asynchronously. No queue row is written.
9. Orchestrator applies its 60s debounce, repairs effective depth from queue context, and runs the 15-step chain.
10. At seed depth, Exa C-suite/founder people expansion, YC people, and seed-marker deal fanout are allowed. Downstream leaves are depth 1 and bounded.

Expected key AlloyDB writes:

| Table | Key | Important fields |
| --- | --- | --- |
| `company` | procedural dedup on `canonical_domain = resend.com` and links | `name`, `canonical_domain`, `source`, `visibility=false`, `is_vc` derived, `node_uuid` after graph create |
| `company_enrich_data` | `company_id` | `company_llm` full envelope, `fundable` payload or `"no_data"`, later `exa_c_suite`, `yc_data`, `signal_nfx_*` |
| `company_financial` | `company_id` | Preserve-existing writes from LLM/Fundable; `is_public` escalate-only |
| `company_enrichment_history` | `(company_id, source, created_at)` | Base row processing lifecycle, `Company_LLM`, `Fundable`, provider trace rows |
| `company_link` | `(company_id, normalized_url)` | One canonical URL per platform when identity-gated |
| `about_company` | `(company_id, md5(emoji_stripped_text))` | LLM/Fundable/YC text corpus |

Expected graph writes:

| Node/Edge | Key | Required properties |
| --- | --- | --- |
| `Company` | Created once; row stores `node_uuid` | `uuid`, `master_company_id`, `name`, `company_domain`, `display_label`, `created_at`, `name_embedding`, Gemini `embeddings` |
| `SPECIALIZES_IN` | `(company_uuid, expertise_uuid)` | Destructive rebuild; weight formula for matched nodes; fixed 10 for created nodes |
| `Funding_Round` | `uuid = fundable_deals.id` | Only when Fundable deals exist and seed-marker gate passes |
| `RAISED` | `(company_uuid, funding_round_uuid)` | Weight 10, deterministic uuid, created\_at from announced date when present |
| Investor edge set | `(investor_uuid, type, funding_round_uuid)` | Repaired live since 2026-07-02; seed depth only |
| `WORKS_AT` and title edges | Work experience rows from Exa loop | Written by #2800 after `master-person-from-exa` returns |
| Location edges | Address graph drain | `LOCATED_IN`, `IN_REGION`, `IN_COUNTRY` from already-geocoded addresses |

Queue/projection side effects:

- No `queue_enrich_company` row for this seed input.
- Depth-1 companies discovered later must queue through `upsert-enrich-company`.
- Projection writer must emit or refresh `company_card_snapshot` after create and again after enrichment completion.

Acceptance checks:

- Re-running the same seed does not create a second canonical company row when dedup keys match.
- Re-running within 60 seconds does not run a second orchestrator pass.
- The Company node is not created twice after `node_uuid` is already set.
- Empty/null optional graph properties are absent, not present as nulls.
- A duplicate queue request for the same company increments `count` and preserves best tier/depth.
- Explicit `priority_tier: 0` through the queue path behaves as documented: it becomes tier 4.

## 8. Function Contract Sheets

### `mvp/get-add/master-company-new` #12558

Role: synchronous entry gate and only sanctioned Company get-or-create path.

Reads:

- `master_link`, `master_company`, `company_enrich_data`, `environment_variables`, `fundable_organizations`, `log_enrichment_company`.

Writes:

- `log_enrichment_company`, `log_crash`, `enrich_history_company`, `master_company`, `company_enrich_data`, `company_financial`, identity/tag/about tables, `fundable_organizations` backlinks, initial Company graph node.

Calls:

- #4587, #12696, #12789, #12974, #12698, #12989, #2602, #12855, #12992, plus fanout helpers #2547, #2548, #2538, #2446, #2451, #281.

External providers:

- OpenRouter and Serper via #12974. Fundable lookup is local mirror only. logo.dev URL is constructed, never fetched.

Queue behavior:

- `hold_downstream` true: no queue and no orchestrator dispatch.
- `cascade_depth == 0 && queue == false && hold_downstream == false`: async orchestrator dispatch.
- Otherwise: sync `upsert-enrich-company` #12855.

Pseudo-code:

```go
func GetAddCompany(ctx context.Context, in CompanySeed) (*CompanyResult, error) {
    if placeholderCompany(in.CompanyName, in.ProfileURL) {
        logQAPass("placeholder blocked")
        return nil, nil
    }

    normalized := normalizeCompanySeed(in)
    if hit := findByProfileLink(ctx, normalized.ProfileURL); hit != nil {
        maybeQueueOrDispatch(ctx, hit.ID, in)
        return hit, nil
    }

    if killSwitchOn(ctx) {
        return localOnlyCompanyLookup(ctx, normalized)
    }

    if isLookupOnly(normalized) {
        return localOnlyCompanyLookup(ctx, normalized)
    }

    var llm CompanyLLMEnvelope
    if !in.ReturnFast && hasIdentityHint(normalized) {
        llm = providers.CompanyResearch(ctx, normalized)
    }

    if hit := postLLMDedup(ctx, normalized, llm); hit != nil {
        maybeQueueOrDispatch(ctx, hit.ID, in)
        return hit, nil
    }

    fundable := store.LookupFundableCompany(ctx, normalized)
    if hit := postFundableDedup(ctx, normalized, fundable); hit != nil {
        maybeQueueOrDispatch(ctx, hit.ID, in)
        return hit, nil
    }

    return store.WithTx(ctx, func(tx Tx) (*CompanyResult, error) {
        company := tx.CreateCompany(normalized, llm, fundable)
        tx.CreateCompanyEnrichData(company.ID, llm, fundableOrNoData(fundable))
        tx.ApplyCompanyLLMPayload(company.ID, "Company_LLM")
        graph.CreateCompanyNode(ctx, tx, company.ID)
        tx.ApplyFundableFanout(company.ID, fundable, in.HoldDownstream)
        maybeQueueOrDispatch(ctx, company.ID, in)
        projections.WriteCompanyCardSnapshot(ctx, tx, company.ID)
        return company, nil
    })
}
```

Go implementation notes:

- Use a transaction for row creation and relational fanout, but treat graph writes as idempotent side effects with recovery/backfill.
- Preserve procedural dedup keys, but intentionally add unique constraints in AlloyDB where this spec calls them out. The Go code must still mimic branch behavior on conflict.
- Do not trust `is_vc` input; derive from domain/profile/Fundable/LLM evidence.

Artifact decisions:

- Xano serializer workarounds (`{}|set`, double-prefix URL lookup, dead `$modelUsed`) are not ported.
- Preserve the load-bearing smoke-test typo only if external callers still send it: `seed-invetment-thesis-smoke-test`.

Acceptance checks:

- Placeholder company returns null and creates no row/node.
- URL-shaped `domain` does not become a bad domain.
- Kill switch blocks new creation and makes no provider calls.
- Lookup-only hit returns existing company without queueing.
- Fresh seed dispatches the orchestrator; depth-1 leaf queues.

Output shape:

```json
{
  "data": {
    "name": "Resend",
    "legal_name": null,
    "aliases": [],
    "domain": "resend.com",
    "alt_domains": [],
    "company_type": "Private Company",
    "profiles": [],
    "_meta": {
      "confidence": { "overall": 0.82 },
      "sources": [],
      "notes": []
    }
  },
  "model_used": "google/gemini-3.1-flash-lite",
  "escalated": false,
  "gemini_confidence": 0.82

```

Provider calls:

- OpenRouter tier 1: `google/gemini-3.1-flash-lite`, `openrouter:web_search`, temperature 0, max tokens 8000, timeout 180s, `provider.data_collection = deny`.
- Escalation: `anthropic/claude-sonnet-4.6`, `openrouter:web_search`, temperature 0.2, max tokens 8000, timeout 240s, when parse fails, confidence is missing/non-numeric, or confidence is below 0.7.
- Serper: `POST https://google.serper.dev/search`, batch of seven platform queries, timeout 30s, silent `[]` on failure.

Pseudo-code:

```go
func CompanyResearch(ctx context.Context, in ResearchInput) (CompanyLLMEnvelope, error) {
    requireIdentityHint(in.CompanyDomain, in.LinkedInURL, in.SourceURLs, in.SourceFacts)
    tier1 := openrouter.Chat(ctx, geminiCompanyPrompt(in), GeminiFlashLite, 180*time.Second)
    parsed := parseCompanyJSON(tier1)
    confidence := parsed.Meta.Confidence.Overall
    envelope := envelopeFrom(parsed, GeminiFlashLite)
    if !parsed.Valid || confidence == nil || *confidence < 0.7 {
        tier2 := openrouter.Chat(ctx, companyPrompt(in), ClaudeSonnet46, 240*time.Second)
        parsed2 := parseCompanyJSON(tier2)
        envelope = envelopeFrom(parsed2, ClaudeSonnet46)
        envelope.Escalated = true
    }
    serperURLs := serper.ProfileSearch(ctx, in)
    envelope.Data.Profiles = mergeAndIdentityGateProfiles(envelope.Data.Profiles, serperURLs, in)
    normalizeCompanyResearch(&envelope.Data, in.SourceFacts)
    return envelope, nil
}
```

Acceptance checks:

- A tier-1 response with missing `_meta.confidence.overall` escalates.
- A tier-2 parse failure returns an envelope with `data._error`; callers do not treat it as success.
- Serper failure does not fail the LLM stage.
- Profile merge emits at most one URL per platform and rejects person/content URLs.

### `mvp/enrich/apply-company-llm-payload` #12989

Role: deterministic mapping from `company_enrich_data.company_llm.data` into relational tables.

Writes:

- `master_company`: preserve-existing scalar fills, `is_school`/`is_vc` true-only escalation except documented repair paths outside this helper.
- `company_financial`: preserve-existing per field since v1.8, VC funding suppression after coalesce.
- `master_link`, `industry_join`, `speciality_join`, `about_company`, `master_address`, `master_email`, `master_phone`.

Pseudo-code:

```go
func ApplyCompanyLLMPayload(ctx context.Context, tx Tx, companyID int64) error {
    ced := tx.GetCompanyEnrichData(companyID)
    data := ced.CompanyLLM.Data
    if data.Empty() {
        return nil
    }
    tx.PreserveCompanyScalars(companyID, data)
    tx.UpsertCompanyFinancialPreserveExisting(companyID, data, ced.Fundable)
    tx.UpsertProfileLinks(companyID, identityGateProfiles(data.Profiles), "Company_LLM")
    tx.AddIndustryArray(companyID, industriesFrom(data))
    tx.AddSpecialtyArray(companyID, specialtiesFrom(data))
    tx.AddAboutRows(companyID, aboutTextsFrom(data), "Company_LLM")
    tx.AddAddresses(companyID, addressesFrom(data), "Company_LLM")
    tx.AddEmailAndPhones(companyID, data)
    return nil
}
```

Acceptance checks:

- Existing non-empty `company_financial` fields are not clobbered by sparse LLM output.
- `angel.co` and `wellfound.com` profiles pass the current v1.8 regex.
- Short aliases do not validate unrelated social handles.

### `mvp/enrich/enrich-master-company` #12992

Role: fifteen-step orchestrator.

State machine:

1. Debounce: if a `Base Company Enrich` history row exists for this company within 60s, log QA pass and return before base history row.
2. Add base `enrich_history_company` row with `processing = true`.
3. Load company and deepest queue row; compute effective depth.
4. Suppress funding-round-sourced VCs at depth greater than 0.
5. Guard missing `company_enrich_data`.
6. Step 1\+2: LLM preflight when `company_llm.data` empty; persist whole envelope.
7. Step 3: apply LLM payload only when LLM just ran.
8. Step 4/4.5: Fundable re-lookup and VC classification repair.
9. Step 5: Signal NFX when gated.
10. Step 6: investment thesis when gated.
11. Step 7: Exa C-suite; seed-depth people expansion only.

```go
func EnrichMasterCompany(ctx context.Context, companyID int64, depth int) error {
    if store.RecentBaseCompanyRun(ctx, companyID, 60*time.Second) {
        logs.QAPass(ctx, companyID, "DUPLICATE TRIGGER")
        return nil
    }
    hist := store.StartCompanyHistory(ctx, companyID, "Base Company Enrich")
    crash := false

    c := store.GetCompany(ctx, companyID)
    effectiveDepth := max(depth, store.DeepestQueuedDepth(ctx, companyID))
    if shouldSuppressFundingRoundVC(c, effectiveDepth) {
        store.FinishCompanyHistory(ctx, hist.ID, true)
        return nil
    }
    if !store.HasCompanyEnrichData(ctx, companyID) {
        store.FinishCompanyHistory(ctx, hist.ID, false)
        return nil
    }

    crash = runStep(ctx, &crash, "Company_LLM", func() error { return maybeRunCompanyLLM(ctx, companyID) })
    crash = runStep(ctx, &crash, "Apply LLM", func() error { return maybeApplyCompanyLLM(ctx, companyID) })
    crash = runStep(ctx, &crash, "Fundable", func() error { return fundableRelookupAndVCRepair(ctx, companyID) })
    crash = runStep(ctx, &crash, "Signal NFX", func() error { return maybeSignalNFX(ctx, companyID) })
    crash = runStep(ctx, &crash, "Thesis", func() error { return maybeBuildThesis(ctx, companyID) })
    crash = runStep(ctx, &crash, "Exa", func() error { return maybeExaCSuite(ctx, companyID, effectiveDepth) })
    crash = runStep(ctx, &crash, "YC", func() error { return processYC(ctx, companyID, effectiveDepth) })
    crash = runStep(ctx, &crash, "Fundable Backfill", func() error { return processFundableBackfill(ctx, companyID) })
    crash = runStep(ctx, &crash, "Financial", func() error { return processFinancialRepair(ctx, companyID) })
    crash = runStep(ctx, &crash, "Deals", func() error { return maybeDealFanout(ctx, companyID, effectiveDepth) })
    crash = runStep(ctx, &crash, "Specialties", func() error { return resolveCompanySpecialties(ctx, companyID) })
    crash = runStep(ctx, &crash, "About", func() error { return writeCompanyAbout(ctx, companyID) })
    crash = runStep(ctx, &crash, "Locations", func() error { return addCompanyLocations(ctx, companyID) })
    crash = runStep(ctx, &crash, "Node Sync", func() error { return graph.UpdateCompanyNode(ctx, companyID) })

}
```

- A duplicate run inside 60s writes only the duplicate marker.
- Effective depth cannot be lowered by a stale direct async call.
- Leaf companies at depth 1 save Exa results but do not create people.
- A failure in one step does not skip later isolated steps.

## 9. Orchestration State Machine

```text

  -> get-add gate
     -> placeholder/kill/governor short-circuit
     -> LLM + Fundable sourcing
     -> procedural dedup
     -> row + node create
     -> dispatch decision
        -> seed immediate async orchestrator
        -> depth leaf queue upsert
        -> hold_downstream no-op

queue drain or async dispatch

  -> effective-depth repair
```

## 13. Queue and Worker Contract

### `company_enrichment_queue` worker

Insert/upsert key: `company_id`.

Priority ordering: `priority_tier asc`, `cascade_depth asc`, `id asc`.

Locking SQL:

```sql
WITH picked AS (
  SELECT id
  FROM company_enrichment_queue
  WHERE processing = false
  ORDER BY priority_tier ASC, cascade_depth ASC, id ASC
  FOR UPDATE SKIP LOCKED
  LIMIT 1
)
UPDATE company_enrichment_queue q
SET processing = true, updated_at = now()
FROM picked
WHERE q.id = picked.id
RETURNING q.*;
```

Processing transaction boundary:

- Queue lock/update is its own short transaction.
- Worker runs orchestration outside the lock transaction.
- Success deletes row.
- Failure releases row and increments retry count; after configured max retries, move to DLQ.

Retry policy:

- Xano has no retry counter and can poison-block. Go must add bounded retries and DLQ for operational safety.
- Data-contract parity is table/graph idempotency, not infinite poison retry.

Explicit-zero behavior:

- `priority_tier: 0` becomes 4.
- `cascade_depth: 0` remains 0 in #12855 because fallback is also 0.

Acceptance checks:

- Duplicate queue hit increments `count`.
- Better tier/depth improves the existing row.
- Source metadata updates only on improvement.
- Worker uses row locks and cannot double-process a row concurrently.

## 16. Idempotency and Dedup Contract

Company dedup order:

1. Incoming/profile URL through `master_link`.
2. Resolved canonical domain.
3. Original pre-redirect domain.
4. LLM alt domains against existing canonical domains.
5. LLM-discovered and Fundable-known profile URLs.
6. Fundable resolved domain/original domain/known URLs.

Queue idempotency:

- One logical row per company in Go via unique `company_id`.
- Upsert must mimic #12855 update semantics.

Graph idempotency:

- Company create is row-gated by `node_uuid`, not graph MERGE.
- Funding\_Round and most edges are MERGE keyed.
- `SPECIALIZES_IN` is destructive rebuild.
- Location drain marks address processed even if endpoint MATCH no-ops in Xano; Go should make endpoint absence observable and retryable unless Mark wants exact no-op parity.

Provider idempotency:

- `company_llm.data` empty gates LLM.
- `exa_c_suite` empty gates Exa; zero-match envelope is non-empty and fetch-once.
- `yc_data == "no_data"` retries; `fundable == "no_data"` does not.
- `Signal NFX` 30-day history row gates retry, including failed attempts.
- Thesis rebuilds on no row, empty derived summary, or stale older than 30 days.

## 17. Error Handling and Observability

Structured logs:

- Replace Xano `log_crash` with structured event logs carrying company id, function/step name, severity, QA pass/fail, error, provider status, queue id, and fixture correlation id.
- Preserve important source names for parity comparisons.

Continuations:

- Most orchestrator steps continue after failure and mark final success false.
- Provider failures that write sentinel payloads must remain distinguishable from true misses.
- Graph writer failures should leave retryable state: e.g. no `node_uuid`, no `processed_at`.

Operational improvements allowed:

- Add queue retry counters and DLQ.
- Add stale processing reapers for queue and history.
- Add unique indexes and conflict handling.
- Add provider replay mode.

Do not hide these as parity. Document them as intentional Go reliability upgrades.
