## Welcome

Welcome to the Orbiter.io API documentation. This reference covers all available API endpoints for integrating with the Orbiter platform.

## Base URL

Each API group has its own base URL following the Xano format:

```
https://xh2o-yths-38lt.n7c.xano.io/api:{group_canonical}
```

For example, the Auth group's base URL is:

```
https://xh2o-yths-38lt.n7c.xano.io/api:qMCc0ojP
```

## Required Headers

All API requests must include the following headers:

| Header | Type | Required | Description |
|--------|------|----------|-------------|
| `Content-Type` | string | Yes | Always `application/json` |
| `X-Data-Source` | string | Yes | Identifies the data source for the request |
| `X-Branch` | string | Yes | The Xano branch to target (e.g. `v1`, `staging`, `dev`) |
| `Authorization` | string | Conditional | `Bearer YOUR_AUTH_TOKEN` — required for authenticated endpoints |

## Authentication

Most API endpoints require authentication using a Bearer token. You can obtain a token by calling the [Login](/api-reference/xano/auth/auth-login) or [Sign Up](/api-reference/xano/auth/auth-signup) endpoints.

Example headers for an authenticated request:

```bash
-H "Content-Type: application/json"
-H "X-Data-Source: YOUR_DATA_SOURCE"
-H "X-Branch: v1"
-H "Authorization: Bearer YOUR_AUTH_TOKEN"
```

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Success |
| 400 | Input Error - Check the request payload for issues |
| 401 | Unauthorized - Invalid or missing authentication token |
| 403 | Access Denied - Insufficient privileges |
| 404 | Not Found - The requested resource does not exist |
| 429 | Rate Limited - Too many requests |
| 500 | Unexpected server error |
