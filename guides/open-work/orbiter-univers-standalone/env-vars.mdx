## Rebuild Notes

- Keep env var names case-sensitive. The Xano code uses both uppercase and lowercase forms in a few places; the Go rebuild should normalize only after verifying compatibility.
- Treat `openRouter` as the shared LLM client key for both chat completions and embeddings unless the rebuild deliberately separates embedding credentials.
- Treat `LLM_MODEL_JSON_PARSE` as a deploy-time model setting. It belongs next to LLM config, not the secret store, unless the deployment platform only exposes one env mechanism.
- Keep `groq` for `mvp/founders/find-founder-linkedin` #12563. That function currently depends on Groq Compound's search behavior and is not an OpenRouter migration target for the rebuild.
- Move direct URL config such as `FULL_ENRICH_WEBHOOK_URL` and `xano_url` into non-secret config in the Go/GCP deployment.
- Do not commit raw secret values to Mintlify unless the repo is private, credential storage in docs is explicitly approved, and rotation/revocation expectations are clear.
