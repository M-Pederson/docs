# Parity QA Smoke Tests

<Note>
  **Last reviewed & updated: 2026-07-15** — the input shape below matches the live `master-person-new` #13039 input surface, and the deterministic preflight gates cover deep-bio payload integrity, terminal company dependencies, music probe lifecycles, and education replay ownership.

Forward-only QA cases for checking the person waterfall against live Xano behavior.

## Deterministic Preflight Gates

Run these sandbox workflow tests before spending a wipe on a full person smoke:

| Test | ID | Required result |
| --- | ---: | --- |
| WhyHow/Thomas verified candidate fixture | 87 | `status: ok` |
| Terminal current-company dependency barrier | 89 | Pending defers; explicit dependency failure fails closed |
| Music reverse-probe lifecycle | 90 | Stale lifecycle probes ignored; newest valid probe controls |
| Education relationship replay preservation | 92 | Row ID, school ID, and existing edge UUID survive a relationship-free replay |
| Thomas committed education parity | 93 | Zero unresolved school IDs and zero missing edge UUIDs |
| Deep-bio payload/receipt regression | Function 13210 | `passed: true`; all payload, retry-recovery, and provider-receipt checks are true |

The terminal education resolver must also log a committed audit with zero
incomplete rows, semantic duplicates, captured-value misses, and graph-date
misses. A CLI transport timeout is not a pass or failure by itself; inspect the
server-side terminal log and committed rows before classifying the run.

## End-to-End Pass Gate

A successful entry response is intake evidence, not the QA result. After the
waterfall settles, require all of the following:

- Three stable polls with no advancing related table, queue, job, log, or graph watermarks.
- The selected current company is fully committed at depth 0. A depth-1 queue row is not sufficient when `person_terminal_company_dependency` exists.
- The exact terminal company dependency is `success`, still points to the selected company, and is backed by company enrichment/history newer than `required_after`.
- `profile_enrichment_job`, the latest authoritative `enrich_history_person`, and `master_person.full_enrich` agree. A later failure overrides an earlier success response.
- The deep-bio job preserves the exact Cloud Run request and callback, validates the expected subject and complete `biography[]`, and has matching non-empty payload SHA-256 values in the job audit, successful `provider_payload_committed` receipt, and callback runner.
- `master_person.deep_bio` is a non-empty string array with at least 120 joined characters. Raw prose, truncated JSON, provider meta-commentary, JSON fragments, and a separate replacement job are failures, even when the entry response says success.
- Every materialized education row has both a positive school ID and a valid preserved `ATTENDED` edge UUID; neither field alone is sufficient.
- Expertise table rows and graph edges pass count, property, recency, date, tense, and description parity.
