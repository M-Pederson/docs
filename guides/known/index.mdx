**Known** is Orbiter's model of human relationships. It represents how two people are connected тАФ whether they know each other, how strong the bond is, what kind of relationship it is (professional, personal, familial, romantic), how that relationship has changed over time, and what evidence Orbiter has for any of it.

## Design philosophy

Known is built on **compositional hyperedges**: a single pair of people can have multiple relationship facets stacked between them, each capturing a different dimension of the relationship. Facets coexist, carry their own validity windows, and are created and invalidated independently.

This approach is more expressive than a single "relationship" edge with a flat set of fields. A pair of ex-co-founders who fell out over Series B strategy but still share a board seat, and used to be close friends, are represented by:

```
(Alice)-[:KNOWS]-(Bob)                    // baseline envelope
(Alice)-[:PROFESSIONAL_TIE]-(Bob)         // invalidated on fallout
(Alice)-[:BONDED]-(Bob)                   // invalidated on fallout
(Alice)-[:ESTRANGED]-(Bob)                // currently valid
```

Each facet's `valid_from`/`valid_to` fields capture the history. The combination answers questions a flat model can't: "were they ever close?" (yes, BONDED existed 2018-2022), "is the relationship currently strained?" (yes, ESTRANGED is valid), "what kind of relationship was it?" (professional AND personal, both facets present).

## Conventions

- **Temporal model:** `valid_from` + `valid_to` (nullable). `valid_to IS NULL` means currently valid. Matches the existing Orbiter graph ontology convention.
- **Canonicalization:** All undirected facet edges are stored with `a_id < b_id`. Only `MENTOR` is directed (mentor тЖТ mentee). Introductions are modeled as first-class `Introduction` nodes, not edges.
- **Evidence-driven:** Every `KNOWS` edge carries an evidence array. Facets are derived from evidence + interaction patterns. Nothing is created without a documented reason.
- **Compositional:** Facets stack. A relationship can have any combination of `BONDED`, `PROFESSIONAL_TIE`, `KINSHIP`, `ROMANTIC`, `MENTOR`, `ESTRANGED`.
- **Lifecycle-aware:** When a relationship state changes (falling out, divorce, reconciliation, death), the affected facets are invalidated with `valid_to` set, never deleted. History is preserved automatically.
