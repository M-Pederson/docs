## Shipped — Functional

### M9 / M10 — Submit writes `suggestion_request` only

The headline rule from the Thursday push: **Submit is fire-and-forget.** On Submit the app
writes the `suggestion_request` row (status → processing, result count + delivery format
persisted) and stops — it does **not** compute or render result cards inline. Mark's
complaint was that the canvas appeared to be "computing matches I told you not to compute";
now Submit fires instantly, a toast confirms, and the canvas resets to a clean compose
surface. Match generation is a separate backend lifecycle, not a synchronous wait.

### #4 — Draft restore

Mark's #1 functional complaint: clicking a draft showed "Loading conversation…" then flipped
back to the welcome screen and never loaded — the OpenUI-0.5-switchover regression. The fix
keys replay on `suggestion_request_id` (the thread key), so a draft now reopens its chat and
hydrates the right rail. Legacy empty drafts seed from their stored `request_context`.

### #5 / #26 / #59 — Clean rail on navigate

Flipping into Outcomes used to show the right panel populated with the **last** Outcomes
thing, and the panel content-flashed the previous page's design on arrival. The rule is now:
the panel is **empty on navigate** and only exists when there is a genuinely active thread.

### #8 — Tile dispatch fix

The `find_speakers` tile (and its siblings) did nothing on click and trapped the UI with no
escape. Tiles now dispatch their outcome class correctly and route into the interview.

### #27 — Dispatch-ready state stays armed

Once the interview reaches a submittable state, that state must **not** flip back off. Mark's
rule: once it's ON, the user can submit any time — don't un-arm it on a later free-text turn.

### Durable per-user memory reconnect (#30–#34)

The "can't recall drafts" symptom traced to the OpenUI 0.5 switchover dropping the memory
write path. This push restores a **stable per-conversation thread id** on the frontend and
provisions the user + thread on the backend so memory is durable and user-linked across
sessions. (The memory backend itself is mid-migration — see the 6/4 entry's scope notes.)

## Shipped — Cosmetic

### #24 / #60 / #61 — Show Options reveal

Mark liked the icons but the tiles were **too big** and there was **no way to close** the
reveal once open — it pushed everything down and trapped the user. Tiles are now compact
icon-plus-text, and the reveal has a collapse affordance. The welcome hero no longer clips,
and the center canvas no longer flashes the welcome page when switching between outcomes.
