**18 node types** across 8 logical groups.

Every node in the Orbiter graph carries the base label **`Entity`** plus one or more type-specific labels. Nodes are created during the person and company get-add pipelines, funding ingestion, IMDB credit processing, and expertise identification.

<Info>
Each node type below lists the Xano functions that **create** (<Icon icon="circle-plus" />) and **update** (<Icon icon="pen-to-square" />) it, in copy-paste call-out blocks (`function path ‚Äî ID`). Each block notes whether the function stamps a `created_at` (on create) / `updated_at` (on update) timestamp in its Cypher. As of **2026-06-01** every documented node type stamps both ‚Äî see the [Timestamp coverage audit](#timestamp-coverage-audit) at the bottom for the change record.
</Info>

## People

Nodes representing individual humans. A Person node may carry additional labels based on their role.

### Person

The core node for any individual in the graph.

<Card icon="circle-plus">

```
mvp/node/add-person-node ‚Äî 2601
```

**Creates** the Person node (`MERGE` on `master_person_id`, adds the `:Entity` label). Stamps `created_at` and `updated_at`. ‚úÖ

</Card>

<Card icon="pen-to-square">

```
mvp/node/update-person-node ‚Äî 2758
```

**Updates** the Person node ‚Äî refreshes properties and adds the `Angel` label when applicable. Stamps `updated_at`. ‚úÖ

</Card>

<Card icon="circle-check">

```
mvp/imdb/run-base-imdb-person-enrich ‚Äî 12880
```

Flips `visibility` to `true` during IMDB enrichment ‚Äî `SET p.visibility = true, p.updated_at = timestamp()`. ‚úÖ Bumps `updated_at` (fixed 2026-06-01).

</Card>

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier (UUID v4) |
| `name` | string | Full display name |
| `display_label` | string | Always `"Person"` |
| `visibility` | boolean | Whether the node is visible in the UI |
| `name_embedding` | vecf32 | Semantic embedding of the person's profile |
| `created_at` | string | ISO 8601 timestamp |

<Accordion title="Real example">
```json
{
  "uuid": "cf64fc2a-4b3e-4ff9-99dc-21d7cffebf01",
  "name": "Josh Diamond",
  "display_label": "Person",
  "visibility": true,
  "created_at": "2025-01-15T10:32:45.000Z"
}
```
</Accordion>

### Region

Represents a state, province, or administrative region. Uses a composite `region_key` for deduplication.

**Labels:** `Entity:Region` (person pipeline) or `Entity:State` / `Entity:Region` (company pipeline ‚Äî dynamic via `{{$var.stateOrRegion|capitalize}}`)

| `display_label` | string | `"State"` or `"Region"` |
| `country` | string | Parent country name |
| `summary` | string | Brief description |
| `name_embedding` | vecf32 | Semantic embedding |

<Accordion title="Real example ‚Äî New York">

## Live Events

### Live_Event

Represents a live industry event ‚Äî summits, conferences, trade shows, workshops, etc. Source of truth lives in Xano table `live_events` (#702).

<Card icon="circle-check">

```
mvp/node/add-live-event-node ‚Äî 12851
```

**Creates** the `Live_Event` node ‚Äî `CREATE (p:Entity:Live_Event:{event_type} {...})`, properties built dynamically from the `live_events` row. ‚úÖ Now stamps `created_at` + `updated_at`; the dynamic property loop skips any incoming timestamp keys (fixed 2026-06-01).

</Card>

The base node always carries `Entity:Live_Event` plus one dynamic sub-label drawn from a controlled vocabulary (`event_type`).

**Possible label combinations:**
- `Entity:Live_Event:Summit`
- `Entity:Live_Event:Conference`
- `Entity:Live_Event:Festival`
- `Entity:Live_Event:Trade_Show`
- `Entity:Live_Event:Workshop`
- `Entity:Live_Event:Webinar`
- `Entity:Live_Event:Hackathon`
- `Entity:Live_Event:Meetup`
- `Entity:Live_Event:Networking_Event`

<Info>
Sub-labels are Cypher identifiers ‚Äî **alphanumeric and underscore only, no spaces or slashes**. Never derive the label from the free-form `category` property. Use `event_type` only.
</Info>

<Note>
Edge types for connecting Live_Events to Person / Company / Organization nodes (hosting, speaking, sponsoring, attending, succeeding editions, etc.) are spec'd in [Live_Event Edges (proposed)](/guides/ontology/proposals/event-edges) ‚Äî not yet implemented.
</Note>

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier (matches Xano `node_uuid`) |
| `master_event_id` | integer | FK to Xano `live_events.id` |
| `name` | string | Official event name |
| `display_label` | string | Always `"Live_Event"` |
| `event_type` | string | Controlled sub-label (`Summit`, `Conference`, etc.) ‚Äî drives the Cypher label |
| `visibility` | boolean | UI visibility flag |
| `event_url` | string | Canonical event page URL (unique) |
| `event_key` | string | Stable slug for the specific edition (e.g. `lsi-usa-26-emerging-medtech-summit`) |
| `slug` | string | Short slug |
| `edition_year` | integer | Calendar year of this edition |
