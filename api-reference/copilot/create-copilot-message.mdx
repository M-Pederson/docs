---
title: 'Create Copilot Message'

description: 'Send a message to the copilot'
---

## Overview

Create a new copilot message associated with a suggestion request. This endpoint does not require authentication.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

## Request Body

<ParamField body="suggestion_request_id" type="integer" required>
  The ID of the suggestion request this message belongs to.
</ParamField>

<ParamField body="role" type="string" required>
  The role of the message sender. One of `user` or `system`.
</ParamField>

<ParamField body="context" type="string" required>
  The message content.
</ParamField>

<ParamField body="metadata" type="object">
  Optional metadata associated with the message.
</ParamField>

## Response

<ResponseField name="id" type="integer">
  The message ID.
</ResponseField>

<ResponseField name="suggestion_request_id" type="integer">
  The associated suggestion request ID.
</ResponseField>

<ResponseField name="created_at" type="timestamp">
  When the message was created.
</ResponseField>

<ResponseField name="role" type="string">
  The role of the message sender.
</ResponseField>

<ResponseField name="context" type="string">
  The message content.
</ResponseField>

<ResponseField name="metadata" type="object">
  Optional metadata associated with the message.
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://xh2o-yths-38lt.n7c.xano.io/api:6pGFSEtq/copilot/messages \
  -H "Content-Type: application/json" \
  -H "X-Data-Source: YOUR_DATA_SOURCE" \
  -H "X-Branch: v1" \
  -d '{
    "suggestion_request_id": 123,
    "role": "user",
    "context": "What are the key outcomes?"
  }'
```
</RequestExample>

</ResponseExample>
