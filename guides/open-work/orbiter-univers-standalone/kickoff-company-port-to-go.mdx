XanoScript semantics to honor per call site (verify each in the fetched body; never "clean up" during the port):
- first_notempty treats 0, false, "", [], {} as empty. #12558 alone runs three coercion regimes: cascade_depth|first_notempty:0 and priority_tier|first_notempty:4 (an explicit priority_tier: 0 becomes 4), bool flags on first_notnull (hold_downstream|first_notnull:false), and the $fanoutPolicy lambda's own num() helper treating only null/undefined/'' as missing. Enumerate each default's behavior at 0/false/""/missing and encode exactly that, bug included. The waterfall docs disagree with each other on this exact coercion — the fetched body wins.
- "", null, and missing-key are three distinct states. |get:"path":default fires on missing key (verify per call site whether explicit null triggers it); |is_empty collapses all three plus 0/false/[]/{}. Model tri-state fields explicitly in Go (pointers/presence flags); never map straight to Go zero values.
- Xano table columns are effectively non-nullable: "nullable" fields land with empty/zero defaults and DB reads never yield SQL NULL — map them to non-pointer Go types with zero defaults rather than inventing nil semantics the runtime never had. Sentinel strings also stand in for null: company_enrich_data.fundable stores the literal string "no_data" when Fundable found nothing and is gated later with != "no_data".
- All timestamps are epoch milliseconds (Date.now() math; 60s enrich debounce; 30-day thesis staleness). Use int64 epoch-ms end to end.
- Roughly half the logic lives in api.lambda blocks (embedded V8 JavaScript): regexes, URL parsing, date math. Port each lambda as a spec'd pure Go function. Inside lambdas, inputs are $input.x and stack vars are $var.x — a wrong prefix reads as silent undefined. A var declared then var.update'd before a try_catch reads back stale (pre-update) inside that try block's lambdas — trace such dataflow via run history.
- The stored source is not always the executed behavior: a bare filter-chain in if(...) never fires (live code uses == false comparisons); db.query with return {type:"list", paging} plus an output projection silently returns [] (and itemsTotal is always 0 unless the paging spec sets totals: true); two-arg |concat is NOT join-with-separator; inline {...}/[...] literals containing filter expressions may be stored as dead backtick template strings; function.run with an empty target string is a silent no-op. Grep every fetched body for these five patterns and confirm each hit against run history before porting it as live logic.
- Schema-layer filters=trim on table columns is invisible in function bodies. Fetch table XanoScript for every written table and replicate trim only where actually present.
- Normalization helpers are deliberately duplicated AND divergent: the VC-classification regex has 3 copies (#12992's adds an operatingCorporate exclusion), the host-normalization idiom has ~10 copies (some lowercase first, some trim first), and #12992 vs #12974 carry two DIFFERENT two-part public-suffix lists. Byte-diff every copy before centralizing; divergence is per-call-site behavior, not a bug to unify.
- Dead code exists inside live guards (e.g. lowPriorityLeaf in #12558's $fanoutPolicy is computed but never used). Port the decision actually taken; document dead branches, do not implement them.
- Known latent bug requiring an explicit decision: #12558's post-LLM link-dedupe builds lookup keys as "https://www." + item, so the full https:// profile URLs #12974 emits never match at that stage. Reproduce or fix — but decide explicitly and record it.
- runtime_mode = "async-shared" dispatches (enrich-master-company, enrich-master-person) are fire-and-forget with no result plumbing or ordering guarantee. Model them as async jobs, never synchronous calls.
- LLM system prompts are template-engine output and byte-sensitive. Port and parity-test the RENDERED prompt against captured run-history renders, not the template source.
- Failure handling is part of the contract: log_crash rows with qa_passed:true are the pipeline's breadcrumb/QA event stream and QA tooling depends on the exact note strings; real crashes flip a $hasCrash flag instead of aborting the chain; graph writes can no-op silently because send-cypher returns errors as strings. Port these semantics, including broken catch paths, and flag them.
- Port Cypher query text verbatim. FalkorDB rejects a:Label WHERE predicates (live code uses 'Label' IN labels(a)); do not normalize label predicates or loop guards to more idiomatic forms.
