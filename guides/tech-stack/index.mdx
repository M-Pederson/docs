Orbiter.io is built on a modern stack optimized for knowledge graph construction, enrichment pipelines, and AI-driven workflows. Each technology was chosen for a specific reason — here's what we use and why.

---

## Core Platform

<CardGroup cols={2}>
  <Card title="Xano" icon="server" href="/guides/tech-stack/xano">
    No-code backend platform powering all APIs, database operations, and enrichment pipeline orchestration. The backbone of Orbiter's server-side logic.
  </Card>
  <Card title="TanStack Start" icon="react" href="/guides/tech-stack/tanstack-start">
    Full-stack React framework for the Orbiter frontend. Server-side rendering, type-safe routing, and server functions with Cloudflare Workers deployment.
  </Card>
</CardGroup>

## Auth & Identity

<CardGroup cols={2}>
  <Card title="WorkOS" icon="shield-halved" href="/guides/tech-stack/workos">
    Authentication and identity platform powering login, SSO, directory sync, and user management for Orbiter.io.
  </Card>
</CardGroup>

## Editor & Notifications

<CardGroup cols={2}>
  <Card title="Tiptap" icon="pen-fancy" href="/guides/tech-stack/tiptap">
    Headless rich text editor framework for building collaborative editing experiences. Powers Orbiter's content editing surfaces.
  </Card>
  <Card title="Knock" icon="bell" href="/guides/tech-stack/knock">
    Customer engagement infrastructure for cross-channel notifications — email, in-app, push, SMS, and Slack. Workflows managed as code.
  </Card>
</CardGroup>
