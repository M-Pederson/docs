The complete node catalog for the [Music Industry Waterfall](/guides/enrichment/waterfall/music-industry-waterfall). Each node is keyed on its MusicBrainz `mbid` (`MERGE (n:Label {mbid})`) and is idempotent on re-enrich. Sample data is from live MusicBrainz responses.

<Note>
**Every node carries the standard envelope.** At a minimum, **all** nodes below are created with `created_at`, `updated_at`, and `description` — plus `uuid`, `mbid`, `display_label`, `visibility`, and `name_embedding`. The natural-language `description` is generated after the structural write (like [edge descriptions](/guides/enrichment/waterfall/music-edges#edge-descriptions)); it is `null` between create and generation.
</Note>

<Warning>
**Artist split — Person vs Group.** A MusicBrainz `artist` is **not** one node type:
- `type == "Person"` → **`Entity:Person:Music_Artist`** — a real human is stored as an Orbiter **`Person`** node with the `Music_Artist` label added, so it participates in **both** the people graph (`WORKS_AT`, `KNOWS`, …) **and** the music graph.
- `type ∈ {Group, Orchestra, Choir, Other}` → **`Entity:Music_Group`** — a **new, dedicated** node type. A band (Nirvana) is never a `Person`.

`MEMBER_OF` is therefore always `(Person:Music_Artist)→(Music_Group)`.
</Warning>

## Standard node envelope

```cypher
MERGE (n:<labels> {mbid: $mbid})
SET n.uuid           = coalesce(n.uuid, randomUUID()),
    n.display_label  = "<Label>",
    n.visibility     = true,
    n.description    = <generated summary | null>,
    n.name_embedding = vecf32($embedding),
    n.created_at     = coalesce(n.created_at, timestamp()),
    n.updated_at     = timestamp()
    // … node-specific properties …
```

| Field | Type | Required | Notes |
|---|---|:--:|---|
| `uuid` | string | ✅ | `randomUUID()`, stable via `coalesce` |
| `mbid` | string | ✅ | MusicBrainz ID — the MERGE key |
| `display_label` | string | ✅ | UI label |
| `visibility` | boolean | ✅ | UI flag |
| `description` | string | ✅ | one-paragraph NL summary; `null` until generated |
| `name_embedding` | vecf32 | ✅ | semantic embedding (name + disambiguation + description) |
| `created_at` | timestamp | ✅ | `coalesce(n.created_at, timestamp())` |
| `updated_at` | timestamp | ✅ | `timestamp()` on every write |

## People & groups

### Person:Music_Artist

A human musician. Stored as an Orbiter **`Person`** node (base `Entity:Person`) with the **`Music_Artist`** label, so music data enriches the same node the rest of Orbiter uses for that person. Reconciled to an existing `master_person`-derived `Person` by a dedup pass (name + ISNI + shared URLs) — not a separate node.

**Labels:** `Entity:Person:Music_Artist`

| Property | Type | Description |
|---|---|---|
| `uuid` | string | Primary identifier |
| `mbid` | string | MusicBrainz artist ID (MERGE key) |
| `name` / `sort_name` | string | `Kurt Cobain` / `Cobain, Kurt` |
| `display_label` | string | `"Person"` |
| `gender` | string | `Male` |
| `country` | string | `US` |
| `birth_date` / `death_date` | string | `1967-02-20` / `1994-04-05` (from `life-span`) |
| `begin_area` | string | `Aberdeen` (hometown) |
| `instruments` | string (csv) | aggregated from `MEMBER_OF` / credits — `guitar, vocals` |
| `isnis` / `ipis` | string (csv) | `0000000110685005` — rights-society IDs |
| `music_rating` / `music_rating_votes` | number / int | `5` / `1` |
| `genres` | string (csv) | `grunge, rock` |
| `disambiguation` | string | `lead singer and guitarist for Nirvana` |
| `description` | string | _generated_ |
| `visibility` · `name_embedding` · `created_at` · `updated_at` | | standard envelope |

### Music_Group

A band, orchestra, choir, or other non-person act — a **new** node type (never a `Person`). The `MEMBER_OF` edges connect its `Person:Music_Artist` members.

**Labels:** `Entity:Music_Group`

| Property | Type | Description |
|---|---|---|
| `uuid` | string | Primary identifier |
| `mbid` | string | MusicBrainz artist ID (MERGE key) |
| `name` / `sort_name` | string | `Nirvana` |
| `display_label` | string | `"Music_Group"` |
| `group_type` | string | `Group` (`Orchestra` / `Choir` / `Other`) |
| `country` / `area` | string | `US` / `United States` |
| `formed_year` / `dissolved_year` / `ended` | int / int / bool | from `life-span` |
| `music_rating` / `music_rating_votes` | number / int | `4.6` / `46` |
| `genres` | string (csv) | curated — `grunge, alternative rock, noise rock`; also written as `HAS_GENRE` edges to `Genre` nodes (weight 100) |
| `tags` | string (csv) | free-form folksonomy — **property only, never edges** ([decided](/guides/enrichment/waterfall/music-industry-waterfall#resolved-design-decisions-2026-06-11)) |
| `disambiguation` | string | `1980s–1990s US grunge band` |
| `description` | string | _generated_ |
| `visibility` · `name_embedding` · `created_at` · `updated_at` | | standard envelope |

</Accordion>

## Releases & labels

### Release_Group

The abstract album/single/EP — one creative work that, in MusicBrainz, may have many concrete releases. In the graph, only the **one canonical `Release`** is materialized per release-group (see below).

**Labels:** `Entity:Release_Group` (+ `:Album` / `:Single` / `:EP` / `:Broadcast` / `:Other`)

| Property | Type | Example |
|---|---|---|
| `mbid` | string | `1b022e01-…` |
| `title` | string | `Nevermind` |
| `primary_type` | string | `Album` |
| `secondary_types` | string (csv) | `Live, Compilation` |
| `first_release_date` | string | `1991-09-24` |
| `music_rating` / `genres` | number / csv | `4.45` / `grunge, alternative rock` |
| `description` | string | _generated_ |
| envelope | | |

### Release

One concrete edition — a specific pressing / country / barcode. **Canonical-release rule ([decided 2026-06-11](/guides/enrichment/waterfall/music-industry-waterfall#resolved-design-decisions-2026-06-11)):** exactly **one** `Release` node materializes per `Release_Group` — `status = Official`, earliest release date (any country). All other pressings stay in the raw JSON on the `mb_*` source tables. The example below is that single canonical release for *Nevermind*.

**Labels:** `Entity:Release`

| Property | Type | Example |
|---|---|---|
| `mbid` | string | `eccae410-…` |
| `title` | string | `Nevermind` |
| `status` | string | `Official` |
| `date` / `country` | string | `1991-09-24` / `AU` |
| `barcode` / `packaging` | string | `720642442524` / `Jewel Case` |
| `formats` / `track_count` | csv / int | `CD` / `12` |
| `description` | string | _generated_ |
| envelope | | |

## Summary — all node types

| Node | Labels | Seed? | `description` | Reconciles to |
|---|---|:--:|:--:|---|
| Person artist | `Entity:Person:Music_Artist` | ✅ | ✅ | existing `Person` (dedup) |
| Group / band | `Entity:Music_Group` | ✅ | ✅ | — |
| Recording | `Entity:Recording` | ✅ | ✅ | — |
| Work | `Entity:Work` (+`:Song`) | cascade | ✅ | — |
| Release group | `Entity:Release_Group` (+`:Album`…) | cascade | ✅ | — |
| Release | `Entity:Release` | cascade | ✅ | — |
| Label | `Entity:Music_Label` | ✅ | ✅ | `Company` (`SAME_AS`) |
| Event | `Entity:Music_Event` (+`:Festival`…) | ✅ | ✅ | `Live_Event` (future) |
| Place | `Entity:Place` (+`:Studio`/`:Venue`) | inline | ✅ | — |
| Genre | `Entity:Genre` | inline | ✅ | — |

**10 node types.** All 10 carry `created_at`, `updated_at`, and `description` (plus `uuid`, `mbid`, `display_label`, `visibility`, `name_embedding`) — the [standard envelope](#standard-node-envelope) — on top of their type-specific properties.
