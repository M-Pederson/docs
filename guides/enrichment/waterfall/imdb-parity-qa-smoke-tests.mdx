# Parity QA Smoke Tests

Forward-only QA cases for checking the IMDB waterfall against live Xano behavior.

## Smoke Tests

Use these fixtures against **workspace 3 sandbox only** after running the sandbox clear utility.

Run the seed through `mvp/imdb/get-add/imdb-person` (#12873) with `queue: false` and `cascade_depth: 0`, then paste the returned entry-point response into the `qa-the-sandbox` skill.

### Pass Criteria

- Waterfall settles with 3 consecutive stable polls.
- `log_crash` has 0 rows where `qa_passed = false`.
- Root `imdb_person` is enriched, has `master_person_id`, and stamps a `node_uuid`.
- Root `Person` node is visible and has a non-empty `description`.
- Top synced credits create `Film_TV` graph nodes; deferred lower-priority credits create `film_tv` stubs and `queue_imdb_title` rows with `processing = false`.
- `qa/falkor/connected-subgraph-readonly` (#13177) returns the sandbox graph, excludes vector properties, and shows `Film_TV.name`/`Film_TV.title` for connected title nodes.
- Every connected node has `description`.
- Every connected edge has `uuid` and `description`.
- `WORKED_ON` edges include useful role/title context, not blank placeholder text.
- Title enrichment should not fall back to `title_extract_failed_stub_node` for synced top credits when IMDb markdown contains usable title metadata.
- Awards pages with no awards may skip cleanly; they should not create warning rows when IMDb has no award data.
