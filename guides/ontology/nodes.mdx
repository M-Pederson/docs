Core graph node types plus enrichment-specific graph nodes, organized by logical group.

Most product graph nodes carry the base label **`Entity`** plus one or more type-specific labels. Legacy non-`Entity` labels are called out in the [Master Label List](#master-label-list). Nodes are created during the person and company get-add pipelines, funding ingestion, IMDB credit processing, expertise identification, and MusicBrainz waterfall enrichment.

<Info>
Each node type below lists the Xano functions that **create** (<Icon icon="circle-plus" />) and **update** (<Icon icon="pen-to-square" />) it, in copy-paste call-out blocks (`function path — ID`). Each block notes whether the function stamps a `created_at` (on create) / `updated_at` (on update) timestamp in its Cypher. As of **2026-06-01** every documented node type stamps both — see the [Timestamp coverage audit](#timestamp-coverage-audit) at the bottom for the change record.
</Info>

## Master Label List

This is the canonical label inventory for node label combinations used by the documented Cypher writers. It includes both full create-label shapes and subset / match-only label combinations such as `Person`, `Company`, `Film_TV`, or `Work`. Dynamic families are shown with the controlling source field.

<Note>
`Entity` is the intended base label for graph entities. A few older writers still create legacy non-`Entity` nodes (`Funding_Round`, `Project`, `Honor`, `Film_TV_Award:*`); they are listed here because they still appear in live or documented Cypher.
</Note>

## People

Nodes representing individual humans. A Person node may carry additional labels based on their role.

### Person

The core node for any individual in the graph.

<Card icon="circle-plus">

```
mvp/node/add-person-node — 2601
```

**Creates** the Person node (`MERGE` on `master_person_id`, adds the `:Entity` label). Stamps `created_at` and `updated_at`. ✅

</Card>

<Card icon="pen-to-square">

```
mvp/node/update-person-node — 2758
```

**Updates** the Person node — refreshes properties and adds the `Angel` label when applicable. Stamps `updated_at`. ✅

</Card>

<Card icon="circle-check">

```
mvp/imdb/run-base-imdb-person-enrich — 12880
```

Flips `visibility` to `true` during IMDB enrichment — `SET p.visibility = true, p.updated_at = timestamp()`. ✅ Bumps `updated_at` (fixed 2026-06-01).

</Card>

| Property | Type | Description |
|----------|------|-------------|
| `uuid` | string | Primary identifier (UUID v4) |
| `name` | string | Full display name |
| `display_label` | string | Always `"Person"` |
| `visibility` | boolean | Whether the node is visible in the UI |
| `name_embedding` | vecf32 | Semantic embedding of the person's profile |
| `created_at` | string | ISO 8601 timestamp |

<Accordion title="Real example">
```json
{
  "uuid": "cf64fc2a-4b3e-4ff9-99dc-21d7cffebf01",
  "name": "Josh Diamond",
  "display_label": "Person",
  "visibility": true,
  "created_at": "2025-01-15T10:32:45.000Z"
}
```
</Accordion>

### Angel

An additional label applied to Person nodes identified as angel investors.

<Card icon="tag">

```

```

**Updates** the Person node — adds the `Angel` label when `is_angel = true`. Stamps `updated_at`. ✅

</Card>

**Labels:** `Entity:Person:Angel`

No additional properties beyond Person — `Angel` is a label-only modifier used for filtering and traversal.

<Accordion title="Real example">

```

</Accordion>

## Organizations

Nodes representing companies, schools, VC firms, and other organizations. All share the same creation function (`mvp/node/add-company-node` #2602) with a dynamic label based on type.

### Company

The default organization node type. Represents businesses, startups, and corporations.

<Card icon="circle-plus">

```
mvp/node/add-company-node — 2602
```

**Creates** the Company node — `CREATE (p:{nodeLabel} {...})` with a dynamic label (`Company`, `School`, etc.). Stamps `created_at` (active v4 template). ✅

</Card>

<Card icon="pen-to-square">

```
mvp/node/update-company-node — 4659
```

**Updates** the Company node — refreshes enrich properties. Stamps `updated_at`. ✅

</Card>

<Card icon="pen-to-square">

tool/resync-company-node-domains — 13062   ·   tool/backfill-strip-domain-errors-nodes — 13060

Maintenance tools that rewrite `company_domain` / `logo` on the Company node. Both stamp `updated_at`. ✅

</Card>

<Card icon="circle-check">

</Card>

<Card icon="tag">

```

**Updates** an existing Company node — `MATCH (organization:Company) SET organization:Organization, organization.updated_at = timestamp()`. ✅ Bumps `updated_at` (fixed 2026-06-01).

</Card>

**Labels:** `Entity:Organization:Company` (when added to existing Company) or `Entity:Organization:Company` (when created new by volunteering)
