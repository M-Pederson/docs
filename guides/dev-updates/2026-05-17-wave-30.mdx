## Shipped — Functional

### YAML context flows to every dispatch

The user's context "ball" — a YAML document describing who they are and their company — now
rides along on every dispatch:

<Steps>
  <Step title="Person + company context wired">
    The Outcomes canvas ships `self_yaml_context` on all three dispatch sites; every dispatch
    verifiably carries the full user YAML.
  </Step>
  <Step title="Leverage Loops carries the target too">
    Leverage Loops ships `self_yaml_context` + `target_yaml_context` + `for_master_person_id`
    — the "help someone else" flow needs both sides of the relationship.
  </Step>
  <Step title="Backend reads it before classifying">
    The dispatch endpoint prepends `[YOU]` and `[HELPING TARGET]` blocks to the memory context
    before the classifier runs. Stress-tested across six payload shapes (empty, small, 10KB,
    injection, orphan target, full target) — all pass, and the classifier's reasoning cites
    the YAML content verbatim.
  </Step>
</Steps>

A 30KB cap is applied at each dispatch site so a large context ball can't blow the payload.

### find_investors moves to Opus 4.7

The `find_investors` synthesis swapped to **Claude Opus 4.7** (via OpenRouter) as the primary
model, with Opus 4.1 as fallback. It's cheaper, faster, and verifiably more honest — 4.7
follows the facts guardrail more faithfully and flags weak matches ("partial fit",
"thesis-level only") instead of overselling with pitch-deck language.

### Haiku synthesis silently-dropped-cards bug fixed

A class of "0 cards returned" bugs traced to a context-pressure failure: when fed 7+ graph
rows, the Haiku synthesis model returned a syntactically valid but **empty** array. The fix
caps the rows fed into synthesis and adds a fallback that emits raw graph rows as minimal
cards if synthesis comes back empty — applied across 11+ outcome-class endpoints. Several
classes that were demo-blocked now render cards again.

## Out of scope (not introduced here)

- Backend model-routing and Cypher-pattern rewrites are engine-side tuning; they change card
