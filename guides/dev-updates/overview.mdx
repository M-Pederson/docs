## How to read an entry

- **Shipped — Functional / Cosmetic** — the items that landed, each tagged with its
  punch-list number (e.g. `#39`) and the commit that closed it.
- **Dogfood proof** — every functional claim is backed by a live screenshot taken
  against `pnpm dev`, not a unit test alone.
- **Known open gaps** — anything the push surfaced but intentionally left to backend
  (clearly labelled so it is never mistaken for a frontend regression).
