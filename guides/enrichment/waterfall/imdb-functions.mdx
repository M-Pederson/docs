## Entry gates

The four functions that can start IMDB work — the person \+ title get-add gates (#12873 / #12874), the master-person attach path (#10509), and reverse IMDB-URL verification (#12677) — are documented on [IMDB Waterfall Entry Points](/guides/enrichment/waterfall/imdb-entry-points). As of 2026-07-08, #10509 creates/links the `imdb_person` stub before scraping, queues a retry row on optional no-markdown Firecrawl misses, stamps success metadata on direct attach, clears stale retry rows after success, and delegates credit fanout to `cascade-credits-from-person(sync_top_credits: 5)` so the top seed credits enrich inline while the rest queue with `seed_imdb_person_id` provenance.

## Graph writers (FalkorDB nodes \+ edges)

<Card icon="cloud-arrow-up">
  ```text
  mvp/edge/add-node-edge-imdb-award — #2334
  ```

  **v1.6 (2026-04-24):** Writes `Film_TV` \+ `Film_TV_Award` nodes, `HAS_AWARD` edge, and (when a Person UUID resolves) `WORKED_ON` \+ `WON_AWARD` / `NOMINATED_FOR` edges to FalkorDB for each `film_tv_award` row. Two-query split so a missing Person stub never kills the whole write. Three-tier bulletproof chain on Query A: MERGE → retry → MATCH-only fallback → `NULL_UUID_FINAL` hard-fail log. v1.5 honorary handling: `film_tv_id == null` rows persist a Person→Award edge without a `Film_TV` anchor. Stamps `film_tv_award.node_uuid` on success. Both `Film_TV` and `Film_TV_Award` nodes stamp `created_at` (`ON CREATE`) \+ `updated_at` (`ON MATCH`). Award/credit edge weights were re-set in the [§3.6 weight audit](/guides/ontology/edge-weights#weight-audit--applied-adjustments) (2026-05-31) — current values on [IMDB Edges](/guides/enrichment/waterfall/imdb-edges#summary--imdb-edges-at-a-glance). Full Cypher on [Cyphers → Film & TV](/guides/cyphers/film-tv).
</Card>

<Card icon="cloud-arrow-up">
  ```text
  mvp/imdb/add-credit-edges-from-person — #13013
  ```

  **v1.0 (2026-05-16) — NEW.** Writes `(Person)-[:WORKED_ON]->(Film_TV)` edges in FalkorDB for every credit on `imdb_person.json.credits[]` whose `imdb_url` resolves to an existing `film_tv` row with `node_uuid` stamped. Credits whose film hasn't been enriched yet are silently skipped — they get picked up when the title cron processes them via #13015. Idempotent MERGE-based; no role in MERGE key. Called from **#12880 Phase 4c**, after Phase 4a stamps `imdb_person.node_uuid`. Closes Bug 2: the original architecture had no writer for non-award `WORKED_ON` edges — `#2334` only fires when a `film_tv_award` row exists with both endpoints resolved. `WORKED_ON` weight per the [§3.6 audit](/guides/ontology/edge-weights#weight-audit--applied-adjustments).
</Card>

<Card icon="cloud-arrow-up">
  ```text
  mvp/imdb/add-credit-edges-from-title — #13015
  ```

  **v1.0 (2026-05-17) — NEW.** Mirror of #13013, sides flipped. Writes `(Person)-[:WORKED_ON {role}]->(Film_TV)` edges for every cast/crew member in `film_tv.json` (directors, writers, cinematographers, producers, cast) whose `imdb_url` resolves to an existing `imdb_person` row with `node_uuid` stamped. Role precedence: directors \> writers \> cinematographers \> producers \> cast (first match wins per MERGE key). Called from **#12881 Phase 3.5**, after Phase 3 stamps `film_tv.node_uuid`. Symmetric with #13013 — once both sides are enriched, edges appear regardless of which side processed first. `WORKED_ON` weight per the [§3.6 audit](/guides/ontology/edge-weights#weight-audit--applied-adjustments).
</Card>

<Card icon="cloud-arrow-up">
  ```text
  mvp/node/add-film-tv-node — #2320
  ```

  **v2 (2026-05-17):** switched `CREATE` → `MERGE` keyed on `imdb_url` so the function is idempotent against `Film_TV` nodes already created by **#2334** in award context (which fires before a title is fully enriched via #12881). Pre-v2 the row-level `node_uuid` guard wasn't enough because award-context node creation never wrote `node_uuid` back to the `film_tv` row — so a fresh #12881 run on an award-anchored title would `CREATE` a duplicate node. MERGE prevents that. Also adopted #2334's safer patterns: `JSON.stringify` literals \+ `api.lambda` cypher builder. Always re-stamps `node_uuid` back onto the `film_tv` row.
</Card>

<Card icon="cloud-arrow-up">
  ```text
  mvp/imdb/tagline-generator — #10516
  ```

  **2026-06-04:** LLM-generates a 96-char-or-less tagline from a film description using `google/gemini-3.1-flash-lite` with the `openrouter:web_search` server tool. The deprecated `:online` suffix was removed during the OpenRouter migration. Called from `run-base-imdb-title-enrich` Phase 2.
</Card>

<Card icon="cloud-arrow-up">
  ```text
  mvp/expertise/llm-identify-person-expertise — #12666
  ```

  **v2 (2026-05-17; chat model moved to `qwen/qwen3.7-plus` on 2026-07-03):** Analyzes person context (YAML) via LLM to identify domains of expertise. Calls `resolve-person-expertise-v2` (#12926) to match/create `SubDomainExpertise` nodes, connect via `HAS_EXPERTISE` edges (with `r.weight` persisted on the graph edge), AND dual-write to `person_has_expertise` (table 711). Logs all results to `expertise_identification_log` for QA. **Added to #12880 Phase 4b in v1.8 (2026-05-17)** — mirrors what `process-person-phase-8` (#12829) does for non-IMDB persons.
</Card>

## Tooling (operator-invoked, not pipeline-wired)

<Card icon="clock-rotate-left">
  ```text
  tool/backfill-award-edges-imdb — #13014
  ```

  **v1.0 (2026-05-16):** re-runs **#2334** for every `film_tv_award` row belonging to a given `imdb_person_id`. </Card>

<Card icon="wrench">
  ```text
  enrich/create-worked-on-edge-description-from-graph — #13016
  ```

  Generates a single-sentence description for a `(Person)-[:WORKED_ON]->(Film_TV)` edge using the edge's structured properties \+ endpoint node names. Keyed on `(person_uuid, film_uuid)` because credit-edge writers (#13013 \+ #13015) don't stamp `e.uuid` — unlike the music trio, which keys on `e.uuid`. Returns description text. **Model migrated to `deepseek/deepseek-v4-flash`** (temp 0.3, `max_tokens` 300, reasoning off — verified live 2026-07-01), completing the swap tracked in [Refactor Edge Descriptions](/guides/robert-mark/refactor-edge-descriptions).
</Card>

<Card icon="wrench">
  ```text
  enrich/write-worked-on-edge-description-to-graph — #13017
  ```

  **v1.0 (2026-05-17):** orchestrates a single `WORKED_ON` edge description: calls `#13016` for the LLM text, then writes `e.description` \+ `e.description_generated_at` back onto the edge via Cypher. No-op if either UUID is null/empty. Mirror of `#13008 write-work-edge-description-to-graph` for the `WORKED_ON` type.
</Card>

<Card icon="clock-rotate-left">
  ```text
  tool/backfill-worked-on-edge-descriptions — #13018
  ```

  **v1.0 (2026-05-17):** batch drainer for `(Person)-[:WORKED_ON]->(Film_TV)` edges missing `description_generated_at`. Fetches up to `limit` (default 10) undone edges via Cypher, calls `#13017` per pair inside `try_catch`, logs summary to `log_crash`. Safe to call repeatedly — idempotent via the `description_generated_at` guard. Mirror of `#13006 backfill-work-edge-descriptions-v5`.
</Card>
