---
title: 'Get Current User'
api: 'GET https://xh2o-yths-38lt.n7c.xano.io/api:qMCc0ojP/auth/me'
description: 'Get the user record belonging to the authentication token'
---

## Overview

Retrieve the profile and settings for the currently authenticated user. This endpoint requires a valid Bearer token.

## Headers

<ParamField header="X-Data-Source" type="string" required>
  Identifies the data source for the request.
</ParamField>

<ParamField header="X-Branch" type="string" required>
  The Xano branch to target (e.g. `v1`, `staging`, `dev`).
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from login or signup.
</ParamField>

## Response

<ResponseField name="id" type="integer">
  The user's unique identifier.
</ResponseField>

<ResponseField name="created_at" type="string">
  Timestamp of when the user account was created.
</ResponseField>

<ResponseField name="master_person_id" type="integer">
  The associated master person record ID. Can be `null`.
</ResponseField>

<ResponseField name="clerk_id" type="string">
  The user's Clerk authentication ID. Can be `null`.
</ResponseField>

<ResponseField name="tiptap_collab_jwt" type="string">
  JWT token for Tiptap collaboration features.
</ResponseField>

<ResponseField name="tiptap_ai_jwt" type="string">
  JWT token for Tiptap AI features. Can be `null`.
</ResponseField>

<ResponseField name="feature_base_jwt" type="string">
  JWT token for FeatureBase integration. Can be `null`.
</ResponseField>

<ResponseField name="knock_jwt" type="string">
  JWT token for Knock notification service. Can be `null`.
</ResponseField>

<ResponseField name="local_timezone" type="string">
  The user's local timezone.
</ResponseField>

<ResponseField name="copilot_enabled" type="boolean">
  Whether the AI copilot feature is enabled for this user. Defaults to `false`.
</ResponseField>
